# 为什么 Laplace 机制必须用「全局敏感度」来标定噪声——纯逻辑推导

> 任务材料：Dwork, McSherry, Nissim, Smith, *Calibrating Noise to Sensitivity in Private Data Analysis*, 2006（`laplacedp_material/laplacedp_dwork_et_al_2006.pdf`）。以下推导完全基于该论文的 Definition 1（ε-indistinguishability）与 Definition 2（L₁-sensitivity），不依赖任何额外实验。

---

## 1. 先把定义严格化（与论文一致）

**邻接数据集**：两个数据库 `x, x' ∈ D^n` 在 Hamming 距离上仅相差一行（"differ in only one entry"）。

**Definition 1（ε-差分隐私 / ε-indistinguishability）**。一个机制 `San` 是 ε-差分隐私的，当且仅当对**所有**邻接对 `x, x'`、**所有**对手 `A`、**所有**转录 `t`，都满足

$$\left|\ln\frac{\Pr[\mathcal T_A(x)=t]}{\Pr[\mathcal T_A(x')=t]}\right|\le \varepsilon.$$

这个定义里有两个量词在"外面"：
- "对**所有**邻接对 `x, x'`" — 即必须在**任意**一对可能的数据集上都成立；
- "对**所有** `t`" — 即必须在机制的**整个**输出空间上都成立。

**Definition 2（L₁-sensitivity）**。函数 `f: D^n → R^d` 的 L₁ 敏感度

$$S(f)=\max_{x,x'\,:\,d_H(x,x')=1}\|f(x)-f(x')\|_1.$$

外层 `max` 是在**所有**邻接对 `x, x'` 上取的；`S(f)` 只取决于函数 `f` 本身（以及域 `D`），**与具体的输入数据库无关**。

**Laplace 机制**（论文 Proposition 1）。

$$\mathcal S(x)=f(x)+(Y_1,\dots,Y_d),\qquad Y_i\overset{i.i.d.}{\sim}\mathrm{Lap}(S(f)/\varepsilon).$$

噪声尺度 `S(f)/ε` 同样只与 `f`、`ε` 有关，**与 `x` 无关**。

---

## 2. 为什么用「全局（最坏情形）敏感度」才能对**任意**数据集保证 ε-DP

这一步是纯粹的逻辑蕴含链，对应论文 Example 1 与 Proposition 1 的证明。

**Step 1. 先看 Laplace 分布本身的尾部比。**
对任意 `y, y' ∈ R`（一维情形），Lap(λ) 的密度 `h(y) ∝ exp(−|y|/λ)`，故

$$\frac{h(y)}{h(y')}=\exp\!\left(\frac{|y'|-|y|}{\lambda}\right)\le \exp\!\left(\frac{|y-y'|}{\lambda}\right).$$

推广到 `R^d`（逐坐标独立），对任意 `t ∈ R^d`：

$$\frac{\Pr[\mathcal T(x)=t]}{\Pr[\mathcal T(x')=t]}
   =\prod_{i=1}^d \frac{\Pr[f_i(x)+Y_i=t_i]}{\Pr[f_i(x')+Y_i=t_i]}
   \le\exp\!\left(\frac{\sum_i |f_i(x)-f_i(x')|}{\lambda}\right)
   =\exp\!\left(\frac{\|f(x)-f(x')\|_1}{\lambda}\right).$$

**Step 2. 用上界替换 `∥f(x)−f(x')∥₁`。**
关键的不等式是：

$$\|f(x)-f(x')\|_1 \;\le\; S(f)=\max_{x,x'\,:\,d_H(x,x')=1}\|f(x)-f(x')\|_1 .$$

`S(f)` 是在**所有**邻接对上取的最大值，所以对**任意**具体邻接对 `x, x'`，上面这个不等式都成立。

**Step 3. 选噪声尺度 `λ = S(f)/ε`。** 把 Step 1、Step 2 合在一起得到对**任意**邻接对 `x, x'`、**任意** `t`：

$$\left|\ln\frac{\Pr[\mathcal T(x)=t]}{\Pr[\mathcal T(x')=t]}\right|
   \le \frac{\|f(x)-f(x')\|_1}{\lambda}
   \le \frac{S(f)}{S(f)/\varepsilon}=\varepsilon.$$

这正是 Definition 1。整个推导中：

- Definition 1 的外层量词（"所有 `x, x'`"、"所有 `t`"）要求不等式必须对**任意**邻接对都成立；
- 因此 Step 2 中**必须**用覆盖所有邻接对的那个上界 `S(f)`；
- 而 `S(f)` 不含 `x`，是函数 `f` 的内禀属性——这正是"全局"二字的意思。

**结论**：只要噪声尺度是 `(关于 f 的、与 x 无关的常数)/ε`，无论服务器实际收到的是**哪一份**数据库 `x`，上面的差分隐私不等式都自动满足。这就是 Laplace 机制在论文里用 `S(f)` 而不是用 `f(x)−f(x')` 的具体值的根本原因：**全局上界是保证定义中"对所有"量词成立的最弱条件**。

论文 Example 1（Noisy Sum `f(x)=Σx_i`，`S(f)=1`）正是这个证明的具体实例：`f(x)` 与 `f(x')` 在一个加性扰动上最多差 1，因此 `Lap(1/ε)` 已经足够。

## 3. 若改用「局部敏感度」会出什么问题

**Definition（Local Sensitivity，Nissim–Raskhodnikova 思路的概念性版本）**。
给定数据库 `x`，函数 `f` 在 `x` 处的**局部敏感度**

$$LS_f(x)=\max_{x'\,:\,d_H(x,x')=1}\|f(x)-f(x')\|_1 .$$

直观上 `LS_f(x) ≤ S(f)`，并且**依赖具体的 `x`**。考虑一个"局部 Laplace 机制"：

$$\mathcal T_{\text{local}}(x)=f(x)+(Y_1,\dots,Y_d),\qquad Y_i\overset{i.i.d.}{\sim}\mathrm{Lap}(LS_f(x)/\varepsilon).$$

### 3.1 破坏 ε-DP 保证的逻辑根源

Definition 1 要求不等式对**所有**邻接对 `x, x'` 成立。`LS_f(x)` 是在以 `x` 为中心、半径 1 的邻域上取的最大值，而 `x` 与 `x'` 只差一行——也就是说 **`x` 的邻域与 `x'` 的邻域只差一格**。

具体看一个反例（论文 Example 2 的逻辑延伸）。设 `f` 是某具体函数（例如 `f(x)=` 字典查询），则对相邻 `x, x'` 有 `LS_f(x) ≠ LS_f(x')` 完全可以发生；最坏情况 `LS_f(x)≪S(f)`，于是从 `x` 看到的噪声尺度比从 `x'` 看到的要**小得多**。把两边的事件 `t` 放在一起看，**只要 `t` 在"小噪声"那条分布的典型值附近、又在"大噪声"那条分布的尾部附近**，两边概率之比就远远超过 `e^ε`。

形式上：取 `x, x'` 邻接，且 `LS_f(x)≪LS_f(x')`，则对同一 `t`：

$$\frac{\Pr[\mathcal T_{\text{local}}(x)=t]}{\Pr[\mathcal T_{\text{local}}(x')=t]}
   \approx \exp\!\left(\frac{\|f(x)-f(x')\|_1}{LS_f(x)/\varepsilon}-\frac{\|f(x)-f(x')\|_1}{LS_f(x')/\varepsilon}\right).$$

当 `LS_f(x')/LS_f(x)` 大到一定程度（例如 ×Ω(n) 都不罕见），这个比值会爆炸性超过 `e^ε`。也就是说 **ε-DP 在某对邻接 `x, x'` 上、某个 `t` 上被违反**——而 Definition 1 一旦在**任何**一对 `(x, x')` 与**任何** `t` 上不成立，整个 ε-DP 保证就直接失效。

直观类比：全局敏感度给出的是"在**所有**邻接对上方都成立的一张安全网"；局部敏感度只在当前那一对邻接对上方算得紧，一旦数据集换成 `x'`，安全网就漏了。

### 3.2 直接泄露数据本身——一个侧信道

更要紧的是 `LS_f(x)` **依赖** `x`。如果机制对每个 `x` 用不同的噪声尺度，那么 **"用多大的噪声"这件事本身就成了关于 `x` 的信息**。对手可以这样攻击：

- 观察到 `t` 后，对手可以（在很多情形下）**反推**出"为了产生 `t` 这样接近 `f(x)` 的输出，机制究竟用了多大尺度的噪声"；
- 或者更直接：若机制被规定成公布噪声参数 `σ = LS_f(x)/ε`（哪怕只是隐式），那么 `σ` 本身就是 `LS_f(x)` 的**函数**；
- 由 `LS_f(x) = max_{x':d_H(x,x')=1}∥f(x)−f(x')∥₁`，这相当于把 `f(x)` 在局部邻域中的"波纹"全部暴露。

经典反例：**中位数查询** `f(x)=median(x)`。`S(f)=1`（最大变化量 1，与 n 无关），但

- 当 `x` 中所有值都接近某个常数 `c` 时，`LS_f(x)≈0`；
- 当 `x` 中含一个远离其余的离群点时，`LS_f(x)=1`。

若用 `Lap(LS_f(x)/ε)`：
- 全相同的 `x` → 几乎不加噪声 → 输出几乎等于 `c` → 对手知道 `x≈(c,c,…,c)`；
- 含离群点的 `x` → 加大量噪声 → 对手能据此判定"这个 `x` 必然是异质的"。

这正是论文在 Section 3.2「Distance from a Predicted Set」一段中指出的风险：函数值在**某些**数据上变化很小、在**另一些**数据上变化很大，局部敏感度会把这个差异**编码到噪声尺度里**直接传出去，ε-DP 在形式上即便勉强维持，对手也已经从"用了多少噪声"这一额外信号中学到了关于 `x` 本身的信息——而这恰恰是差分隐私应该保护的对象。

更进一步：所谓 `LS_f(x)` 自身就是一个**敏感查询**。要让对手无法反推它，就必须用 Laplace 机制再去发布它，这就又需要"`LS_f` 的全局敏感度"`…`LS_f` 的全局敏感度又依赖 `x`……无限递归。所以局部敏感度路线在逻辑上既**破坏**了 Definition 1、又**绕不过**对自身做差分隐私的开销——这两条问题不是独立的，而是同一根因（`S(f)` 不是 `x` 的函数）派生出来的。

### 3.3 论文为这个问题留的"出口"

论文 Section 3.2 明确给出过这条出路：「**Distance from a Predicted Set**」。当一个函数 `f` 的全局敏感度太大（典型如中位数、最小割），但**对当前数据库**有"到某个固定集合 `S ⊆ D^n` 的 Hamming 距离"这个**上界** `f_S(x) = min_{x'∈S} d_H(x, x')`，则

$$S(f_S)\le 1,$$

即对**这个新函数** `f_S` 来说，全局敏感度又是常数 1。这是因为 `f_S` 的取值范围是 `{0,1,2,…,n}`，改一行最多改 1，**与具体 `x` 无关**。换言之：要避开局部敏感度的陷阱，只能去寻找或构造一个**与 `x` 无关的**上界，重新落回"全局"框架。

---

## 4. 一句话总结

- **逻辑上**：`S(f) = max` 是覆盖 Definition 1 中"所有邻接对"那个外层量词所需的**最紧常数**；用 `LS_f(x)` 等于把外层量词掐掉，换来对一个 `x` 紧、对另一个 `x'` 漏，于是 ε-DP 在某些 `(x, x', t)` 上失效。
- **信息上**：`S(f)` 只与 `f` 相关，机制对**所有** `x` 用**相同**的噪声尺度，输出分布的形态不携带 `x` 的特征；而 `LS_f(x)` 是 `x` 的函数，**噪声尺度本身就成为 `x` 的特征**，等于把本来要保护的东西当成了副产品泄露出去。

所以"全局敏感度"并不是为了 worst-case 而 worst-case 的保守选择——它恰恰是让"隐私保证不依赖真实数据库"这一性质在逻辑上能够成立的**最小条件**。
