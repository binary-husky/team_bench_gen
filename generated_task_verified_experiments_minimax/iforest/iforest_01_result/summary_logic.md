# Isolation Forest 子采样规模 ψ 选小的逻辑推导

> 任务：读 Liu 等人 2008 年论文《Isolation Forest》，仅通过逻辑推理回答：
> **为什么 Isolation Forest 故意让每棵树只在一个小规模 ψ（如 256）的随机子样本上构建，反而能提升异常检测？**
> 先定义两种失效模式，再论证小 ψ 如何同时缓解两者，并解释若改用全部数据（大 ψ）时异常会如何既干扰正常点的隔离、又彼此干扰。

---

## 0. 核心直觉（一句话先行）

Isolation Forest 的异常评分完全由"路径长度 h(x)"决定：路径越短 → 越像异常。在一棵 iTree 里，h(x) 由"把 x 与其他点隔开所需的随机切分数"决定——而这个数字又由"x 周围到底有多少点、点之间到底有多近"决定。**让一个 ψ ≈ 256 的小子样本进入每一棵树，本质上就是主动控制每棵 iTree 看到的"局部密度"和"邻域组成"，从而让"少而不同"的异常点在每一棵树里都更容易暴露出来**。下面把这个直觉拆成三种失效模式与缓解机制。

---

## 1. 两种失效模式的精确定义

论文 §3 明确给出了 swamping 与 masking 的概念。重新表述为数学/逻辑意义上的定义：

### 1.1 Swamping（淹没 / 误把正常当异常）

- **定义**：正常点被错误地识别为异常。
- **触发条件**：正常点离异常点太近，分布上几乎与异常点重叠。
- **iTree 中的表现形式**：当一个正常点 xi 的邻域里塞进了太多异常点（或者说 xi 处的"局部点密度"被异常点抬高），那么随机切分时，把 xi 单独切出去所需的次数 h(xi) 会被人为压低——它会和异常点一样被迅速隔离，路径长度看起来"很短"。于是 s(xi) 趋近 1，被误判为异常。
- **简言之**：异常点"污染"了正常点的邻域，把正常点的路径长度"拽"短了 → 正常点被淹没在异常的异常分数里。

### 1.2 Masking（遮蔽 / 异常彼此掩护）

- **定义**：异常点太多、彼此靠得太近，反而把自己藏起来。
- **触发条件**：异常簇本身又大又密（论文原话："When an anomaly cluster is large and dense"）。
- **iTree 中的表现形式**：当若干异常点聚成一团时，对它们中的任意一个 xi，随机切分必须先把这"一团"劈开很多次才能把它单独切出来。结果是 h(xi) 被异常簇的内部密度人为拉长。h(xi) 一旦变大，s(xi) 就掉到 0.5 附近，异常看起来"和正常点差不多"。
- **简言之**：异常点之间互相干扰隔离过程，把彼此的路径长度"撑"长了 → 异常的短路径特征被遮蔽。

### 1.3 一个统一的视角

无论是 swamping 还是 masking，**根因都是"树的输入里出现了过多与 x 类别无关或同类的近邻点"**。Swamping 是"被对方阵营的近邻拉短"，masking 是"被己方阵营的近邻拉长"。两者都表现为：异常点与正常点的 h(x) 之差被压缩，最终 s(x) 区分度下降，论文里所谓"longer path lengths making anomalies more difficult to detect"。

论文同时点明："both swamping and masking are a result of too many data for the purpose of anomaly detection." ——也就是说，**降低数据规模是缓解两者的共同杠杆**。

## 2. 为什么小 ψ 能同时缓解 swamping 与 masking

论文 §3 的关键论述是："the unique characteristic of isolation trees allows iForest to build a partial model by sub-sampling which incidentally alleviates the effects of swamping and masking. This is because: (1) sub-sampling controls data size, which helps iForest better isolate examples of anomalies and (2) each isolation tree can be specialised, as each sub-sample includes different set of anomalies or even no anomaly." 我们把这两条拆开，并与上面的两种失效模式对齐。

### 2.1 子采样"控规模"——对 swamping 的缓解

设原始数据有 N 个点，其中 n_a 个异常、n_n 个正常。整棵 iTree 的每次切分都在 N 量级的样本空间里"挑一个 attribute、挑一个 split value"。当 N 很大而 n_a / N 又不可忽略时：

- 对一个正常点 xn 而言，从 N-1 个邻居里抽 attribute 和切分点的过程，**会被 xn 旁边那批异常点反复干扰**：切分点常常恰好落在异常点和 xn 之间，xn 就被划到一个小叶子里，h(xn) 被人为缩短。
- 这正是 swamping 的形成机制。

当 ψ ≪ N 时：

1. 子样本里异常点的**绝对数量**骤降。如果异常在原数据中占比本身就小（比如 1%），ψ = 256 时一个子样本里平均只有 ~3 个异常点。
2. xn 周围的"被异常污染的近邻"密度随之下降，**xn 与异常点被随机切分到同一叶子的概率大幅降低**。
3. 因此 h(xn) 回到"由正常点内部密度决定"的正常长度，s(xn) 不再被异常拉成接近 1。

直观上：**swamping 的强度正比于"xi 邻域内异常点的密度"**；把 N 缩到 ψ，本质上是把每个 xi 看到的"局部异常密度"乘了一个因子 (ψ/N) 级别，swamping 被等比削弱。

### 2.2 子采样"控规模"——对 masking 的缓解

对 masking：原始数据中若异常本身形成一个又大又密的簇（设簇内有 m 个异常），则对簇内任一异常 xi，切分过程必须先把整个簇一分为二、再二分为四…… 直到 xi 单独成叶。h(xi) ≈ O(log m) 这个量级被异常簇的大小锁死。

把整棵树只看到 ψ 个点后：

1. 异常簇内**被采到的成员数 m' ≈ m · ψ/N**。当 ψ 比 m 小一个数量级以上时，m' 趋向 1~几个，**整个"互相掩护"的异常簇在子样本中常常被严重稀疏化甚至只留下单个孤立点**。
2. 一旦簇被打破，xi 在子样本里就是一个真正"少而不同"的孤立点，**一次切分就能把它隔离，h(xi) 回到 ~1~2 的极短路径**。
3. 路径长度的方差因此大幅释放，s(xi) 才能靠近 1，异常的"短路径"特征重新显现。

直观上：**masking 的强度正比于"xi 周围的同类异常点数"**；把 N 缩到 ψ，直接把这个同类近邻数砍掉，异常互相掩护的局面被瓦解。

### 2.3 子采样带来的"每棵树专一化"——对两类失效模式的二次缓解

论文强调的第二条是 "each isolation tree can be specialised, as each sub-sample includes different set of anomalies or even no anomaly"。这一条与上面的规模效应相互独立、却同样关键：

- **随机性 + 小 ψ ⇒ 每棵 iTree 的样本构成高度多样**。t 棵树是独立抽样得到的，每棵树看到的"异常子集"都不同——有些树甚至一个异常都没抽到（ψ = 256、异常占比 1% 时概率约 e^{-2.56} ≈ 7.7%，并非罕见）。
- 当一棵树里**完全没有异常**时，它的 h(x) 分布就成了"纯正常"的基准，可以反衬出哪些点更短。
- 当一棵树里**恰好抽到 1~几个异常**时，这些异常在该子树里就处在"少而不同"的位置，h(x) 自然很短。
- 多棵树做平均后，**正常点的 E[h(x)] 由"纯正常树"和"含少量异常树"共同决定，仍然很长且稳定；异常点的 E[h(x)] 在所有抽到它的树中都极短**，从而 s = 2^{-E[h]/c(ψ)} 把异常拉到接近 1。

这一步的逻辑是：**单棵树看到的越"专一"，异常/正常的 h(x) 差距越显著；多棵树看到的越"多样"，这种差距被平均后依然稳定**。小 ψ 同时满足了这两个相反方向的诉求——既允许每棵树足够"专一"，又因为集成效应（t 棵独立子树）保证结果的方差足够小。

### 2.4 ψ 的"甜区"：太小也不行

ψ 不能无限小：

- ψ 过小（如 < 64）时，单棵树的统计性下降，方差太大；同时正常点内部密度估计也被随机噪声主导，反而损害 s 的稳定性。
- ψ 过大（如逼近 N）时，又退回到 swamping + masking 的原始问题。
- 论文 §5.2 通过 Http 与 ForestCover 两个大数据集的实验观察到 AUC 在 ψ = 128~512 间达到最优、再往上几乎不变。这与上面的逻辑推导一致——**只要 ψ 大到足以让单棵 iTree 反映"正常点的典型局部密度"、小到足以把异常点数和异常簇规模压到不构成 swamping/masking 的水平，性能就收敛**。经验默认 ψ = 256 即处于这个甜区中心。

## 3. 反过来：用全部数据（大 ψ → ψ = N）会怎样

把上面的论证反过来推一遍，就能看清"为什么若改用全部数据，异常会既干扰正常点的隔离、又彼此干扰"。

### 3.1 异常干扰正常点的隔离（swamping 放大）

设一个正常点 xn，它的"h(xn) 的下界"由"距 xn 最近的那个近邻"决定。在 iTree 的递归切分过程中，**只要切分值落在"xn 与它最近邻之间"，xn 就会更早被单独隔离**。

- 若 xn 周围还散布着若干异常点 xa1, xa2, ...，这些异常点在每一层切分时都会增加"落在 xn 与近邻之间"的概率。
- 大 ψ 下这个概率与 (异常点数 / 总点数) · (异常点离 xn 的接近度) 成正比。**只要异常占比不是可以忽略的量级，整棵树的切分过程都被异常点反复搅乱**。
- 结果：xn 的 h(xn) 被压低，s(xn) → 1，xn 被错认为异常。**这就是异常"干扰"了正常点的隔离**。

换句话说，**异常干扰正常点隔离的本质，是异常点在切分空间里提供了大量"额外的、针对正常点的切分候选"**——把候选空间从"只在正常点之间切"扩大到"在正常-异常混合集里切"，xn 的隔离路径被人为加速。

### 3.2 异常彼此干扰（masking 放大）

考虑一个由 m 个异常组成的密集簇 C。簇内每个异常 xi 要"被单独隔离"必须穿过整个簇：

- 在大 ψ 下，iTree 在簇 C 的范围内每切一刀，最坏情况是把这 m 个点大致一分为二。
- 要把 xi 完全单独切出来，至少需要 ⌈log₂ m⌉ 次有效切分（且只有在某次切分恰好把 xi 与其他 m-1 个分开时才前进一格）。
- 因此 h(xi) ≈ O(log m) 而非"孤立异常"应有的 O(1)。
- 当 m 足够大时，**异常簇内的 xi 的 h(xi) 会长到与正常点一样甚至更长**，s(xi) → 0，xi 与正常点不可区分。**这就是异常"彼此干扰"**。

masking 的本质：**在一个共享的切分空间里，异常点之间互相成为彼此的"最近邻"，从而把彼此的隔离路径拉长**。这是密度问题的几何表现——它依赖于"异常之间彼此距离足够近"，而大 ψ 保证了异常之间的相对位置被完整保留。

### 3.3 联合效应：异常的"双向污染"

把上面两点合起来：

- 在大 ψ 下，异常簇 C 同时扮演两个角色：
  1. 对外部正常点 xn，C 是一个"密集的近邻源"——把 xn 的路径拉短（swamping）。
  2. 对内部异常点 xi，C 本身是一个"互相掩护的群体"——把 xi 的路径拉长（masking）。
- 结果是 **h(xn) 与 h(xi) 在两端被同时压缩**：xn 被拽短、xi 被撑长，**两类点的 h(x) 差距反而缩到最小**，s 的判别力崩溃。

这就是论文图 4(a) 展示的 4096 点原始样本里两个异常簇几乎看不见的原因——也是为什么论文 §5.2 把 ψ 从 32768 降到 128~512 时 AUC 从 0.67 飙升到 0.91 的原因。

### 3.4 为什么"少切几次/多用几个 attribute"也救不了大 ψ 的失败

一个常见的误解是"既然树太深/路径太长，那限制高度 l 不就行了"。

- 论文把 l 设为 ceiling(log₂ ψ)。这是"限制树的高度上限"，**但 iTree 本身已经天然不需要遍历到完全分离**——它用 ψ 控制平均树高、用 BST 的 c(ψ) 做归一化。
- 即便人为限制 l，**只要异常点的近邻还在（ψ 太大时必然如此），前几次切分就足以把异常点和正常点一起分到小叶子里**，h(x) 的统计性依然被污染。
- 也就是说，"深度"不是问题；"局部密度"才是问题。**唯一能从根源上同时缓解两种污染的杠杆，是 ψ**。

## 4. 小 ψ + 多棵树 = "少而不同"假设的最优落实

把上面的推导收束成一个完整的逻辑链：

1. Isolation Forest 的工作前提是"异常是少而不同的"（论文 §1 第 2 段明确写的两个量化前提）。
2. 这个前提要被 iTree 忠实反映，需要在每棵树的局部数据里"异常真的少、真的孤立"。
3. 大 ψ 把异常从"少而孤立"变成"多而抱团"，两个前提同时被破坏 → swamping + masking。
4. 小 ψ 把每棵树的局部数据强制回到"异常真的少、真的孤立"的状态 → 两个前提被恢复 → h(异常) ≪ h(正常) → s(异常) ≫ s(正常)。
5. 单棵树的小 ψ 会让结果方差大，但 t 棵独立小 ψ 子树做集成（论文默认 t = 100）后，**正常点的 E[h] 收敛到一个稳定的长值，异常点的 E[h] 在所有抽到它的树里都极短**，最终的 s 排序既稳定又具有强判别力。
6. 因此 ψ 小既是"暴露异常"的必要条件，又是"通过集成换稳定方差"的可行方案——这正是论文 §6 强调的"small ψ provides high AUC and low processing time, and a further increase of ψ is unnecessary"。

---

## 5. 总结

- **Swamping**：正常点因周围有异常点而被错误地快速隔离，路径被人为压短 → 正常被当异常。
- **Masking**：异常点因彼此抱团而难以被单独隔离，路径被人为拉长 → 异常被当正常。
- 二者都是"数据太多"在 iTree 切分机制下的副作用，根源都是"局部密度被无关点污染"。
- **小 ψ 通过两个独立机制同时缓解两者**：
  1. 控规模——把每个 xi 邻域里的"无关点数"乘以 ψ/N 的比例因子，从而压低 swamping 的强度；
  2. 控组成——让每棵 iTree 的样本"专一化"，异常簇在子样本里被稀疏化甚至打散，从而瓦解 masking；同时多棵独立子树的集成保证方差可控。
- **大 ψ 下异常既干扰正常点（swamping）、又彼此干扰（masking）**：异常既扮演正常点的"额外近邻"把后者路径压短，又扮演彼此的"最近邻"把自身路径拉长——h(异常) 与 h(正常) 的差距在两端被压缩到最小，s 的判别力崩溃。

因此 Isolation Forest 选择 ψ ≈ 256 不是"工程折中"，而是从"少而不同"的核心假设出发、用子采样同时对冲 swamping 与 masking 的必然设计。论文 §5.2 的实验（ψ 从 32768 降到 128~512 时 AUC 从 0.67 提升到 0.91）只是把这个逻辑推导用数字再确认了一遍。
