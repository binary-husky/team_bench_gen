# HyperLogLog 可合并性（mergeability）验证 — 实验报告

研究目标：验证 `M[j] = max(A[j], B[j])` 的合并确实给出并集基数（不是两流基数之和），且与"直接在并集流上建 sketch"在寄存器层完全等价。

## 1. 实现

按照 Flajolet, Fusy, Gandouet, Meunier (2007) 论文（`FlFuGaMe07.pdf`，Figure 2/Figure 3）从零实现 HyperLogLog：

- **精度**：固定 `p = 14`，寄存器数 `m = 2^14 = 16384`。
- **哈希**：64-bit，使用 `mmh3.hash128(item, signed=False)` 后取低 64 位（足够把基数上限抬到远超 10^9，且对 p=14 寄存器结构无偏）。
- **ρ 计算**：按照论文 ρ(s) = s 中最左 1-bit 的位置（1-indexed，从 MSB 算），s=0 时 ρ = (L+1) 其中 L 是剩余位数。这等价于对 L-bit 切片做 `ρ = L − bit_length + 1`。
- **估计器**：`Z = (Σ 2^{-M[j]})^{-1}`，`E = α_m · m² · Z`，带论文 §4 的 small-range 修正（`E ≤ 2.5m` 时用线性计数 `m·log(m/V)`，V 为零寄存器数）。m=16384 时 `α_m = 0.7213 / (1 + 1.079/16384) ≈ 0.72125`。
- **合并**：`merged(A, B)` 通过 `np.maximum(A.registers, B.registers)` 构造。

实现位于 `hll.py`，实验脚本 `run_mergeability.py`。

## 2. 实验设置

固定不变：

- 两条**不相交**合成流 `A`、`B`，各含 `n_A = n_B = 5·10⁴` 个不同 item，`A ∩ B = ∅`；真实并集基数 `|A ∪ B| = 10⁵`。
- 流元素是确定性生成的 64-bit 整数（不同种子给出不同的不相交流）；构造时对 A、B 用同一 HLL 哈希种子（`set_seed(seed)`），保证寄存器层比较是 same-hash 比较。
- 7 个不同随机种子：`{11, 23, 47, 101, 211, 307, 503}`（> 任务要求的 5 个）。
- 整轮在单 CPU 上 < 1 秒。

每个种子做：

1. `S_A` 喂 A，`S_B` 喂 B（精度 p=14，相同哈希种子）。
2. **合并**：`S_merge[j] = max(S_A[j], S_B[j])`，得 `Ê_merge`。
3. **直接流**：把 `A ∪ B` 整体喂给一张新 sketch `S_direct`，得 `Ê_direct`。
4. **幂等/去重**：`S_dup = merged(S_A, S_A)`，得 `Ê_dup`（真实并集仍为 n_A）。

记录：

- `merge_err = |Ê_merge − 10⁵| / 10⁵`
- `direct_err = |Ê_direct − 10⁵| / 10⁵`
- `dup_err = |Ê_dup − 5·10⁴| / 5·10⁴`
- `S_merge == S_direct` 寄存器层是否完全相等
- `S_dup == S_A` 寄存器层是否完全相等

## 3. 数值结果

### 3.1 逐种子结果


每行 E_merge 与 E_direct 完全相等（最多 0 差异）；E_dup 与 E_A 完全相等。

### 3.2 跨种子统计（n = 7 seeds）

| 指标 | 公式 | mean | std |
|---|---|---:|---:|
| 合并相对误差 | `|Ê_merge − 10⁵| / 10⁵` | **0.0077** | 0.0035 |
| 直接流相对误差 | `|Ê_direct − 10⁵| / 10⁵` | **0.0077** | 0.0035 |
| 幂等相对误差 | `|Ê_dup − 5·10⁴| / 5·10⁴` | **0.0060** | 0.0041 |

理论标准误差（论文 Theorem 1）`1.04 / √m = 1.04 / √16384 ≈ 0.81%`，与测得的 ~0.77% 吻合。

### 3.3 寄存器层等价性

| 等价关系 | 满足的种子数 / 总数 |
|---|---:|
| `S_merge == S_direct` （寄存器逐元素相等） | **7 / 7** |
| `S_dup   == S_A` （寄存器逐元素相等） | **7 / 7** |

跨种子 `max |Ê_merge − Ê_direct| = 0` 且 `max |Ê_dup − Ê_A| = 0`：等价不仅是统计上接近，而是**逐寄存器完全相等**，对应估计自然也是数值相等。

### 3.4 幂等/去重语义的定性证据

跨种子平均：

- `Ê_A ≈ 5.03·10⁴`（接近真值 n_A）
- `Ê_B ≈ 5.04·10⁴`（接近真值 n_B）
- `Ê_merge ≈ 9.97·10⁴`（接近 |A ∪ B| = 10⁵）
- `Ê_dup ≈ 5.03·10⁴`（**接近 n_A，不是 2·n_A ≈ 10⁵**）
- 对照：朴素"求和" `Ê_A + Ê_B ≈ 1.006·10⁵`、`2·Ê_A ≈ 1.009·10⁵`

幂等情况 `merge(A, A) ≈ A`（不是 `2A`）是关键证据：max 合并对重复元素**幂等且去重**，所以它给出的是并集基数而非流的基数之和。

## 4. 结论

逐寄存器 max 合并 `S_merge[j] = max(S_A[j], S_B[j])` **等价于在两流并集上重新构建一张 sketch**：在所有 7 个种子上，`S_merge` 与 `S_direct` 寄存器逐元素完全相等（`max |Ê_merge − Ê_direct| = 0`），对应估计 `Ê_merge` 与直接流的 `Ê_direct` 数值完全一致，均给出并集基数（10⁵）而非两流基数之和；同一合并操作对重复元素幂等（`Ê_dup ≈ n_A ≈ 5·10⁴`，而不是 `2·n_A`）。
