# HyperLogLog SE-Scaling 推理与代数推导

## 任务概述

不进行任何代码实验，仅通过逻辑推理与代数推导，从给定材料中的核心结论
`SE ≈ 1.04 / sqrt(m)`（其中 `m = 2^p` 是寄存器个数，`p` 为精度参数；总内存 ∝ m）
出发，逐条判断下列三条主张的正误，并改正错误的那条。

---

## 判定总览

| 主张 | 判定 | 关键事实 |
|------|------|---------|
| (i)  m 翻倍，SE 减半 | **FALSE** | 翻倍 m 仅使 SE 变为 `1/sqrt(2) ≈ 0.707` 倍（约降 29.3%） |
| (ii) SE 减半，内存需 4 倍 | **TRUE** | m → 4m ⇒ SE → SE/2，而内存 ∝ m |
| (iii) p +1 ⇒ SE 乘 1/sqrt(2) ≈ 0.707（约 -29%） | **TRUE** | p+1 ⇒ m → 2m ⇒ SE → SE/sqrt(2) |

---

## 逐条代数推导

### 基础关系
- `SE ∝ 1 / sqrt(m)`
- `m = 2^p`，故 `m = 2^p ⇒ sqrt(m) = 2^(p/2)`
- 每个寄存器占用相同内存 ⇒ 总内存 `∝ m`

---

### 主张 (i)：『把寄存器个数 m 翻倍，就能把标准差减半』 → **FALSE**

**推导：**

令 `m' = 2m`，则

```
SE' = 1.04 / sqrt(m')
    = 1.04 / sqrt(2m)
    = (1 / sqrt(2)) · (1.04 / sqrt(m))
    = SE / sqrt(2)
    ≈ 0.7071 · SE
```

因此 `SE' ≈ 0.7071 · SE`，**只降低了约 29.3%，而非减半**。

**正确版本（改正）：**
> 『把 m 翻倍，标准差变为原来的 `1/sqrt(2) ≈ 0.707`（约降低 29.3%）。』
>
> 也就是说，**仅靠 m 翻倍只能把 SE 缩小约 1.414 倍（即 1/sqrt(2) ≈ 0.707 倍）**，要真正『减半』需要 m 翻 4 倍（见下）。

---

### 主张 (ii)：『要把标准差减半，总内存代价必须是原来的 4 倍』 → **TRUE**

**推导：**

目标：`SE' = SE / 2`。

由 `SE ∝ 1 / sqrt(m)`，有

```
SE' / SE = sqrt(m) / sqrt(m') = 1/2
⇒ sqrt(m') = 2 · sqrt(m)
⇒ m'      = 4m
```

由于总内存 `∝ m`，故

```
Memory' = c · m' = c · 4m = 4 · (c · m) = 4 · Memory
```

也就是说，要使 SE 减半，**m 必须增大到原来的 4 倍，对应总内存也是原来的 4 倍**。该主张正确。

---

### 主张 (iii)：『精度参数 p 每增加 1，标准差大约降低 29%（即乘以 1/sqrt(2) ≈ 0.707）』 → **TRUE**

**推导：**

由 `m = 2^p`，p 增加 1 时

```
m' = 2^(p+1) = 2 · 2^p = 2m
```

于是

```
SE' = 1.04 / sqrt(m')
    = 1.04 / sqrt(2m)
    = (1 / sqrt(2)) · (1.04 / sqrt(m))
    = SE / sqrt(2)
    ≈ 0.7071 · SE
```

即 `SE` 变为原来的 `1/sqrt(2) ≈ 0.707`，等价于**降低了约 29.3%**（精确：1 − 1/sqrt(2) ≈ 0.2929）。该主张正确。

---

## 一句话总结

> **『把 SE 减半』需要：`m` 增大为原来的 4 倍（即 `m → 4m`）、总内存同步变为原来的 4 倍（因为内存 ∝ m）、精度参数 `p` 增加 2（因为 `2^p → 2^(p+2) = 4 · 2^p`）。**
>
> 反过来要记清两条容易混淆的缩放关系：
> - m 翻倍 ⇒ SE 缩小 `1/sqrt(2) ≈ 0.707`（仅 ~29%），**不是减半**；
> - p +1 ⇒ 同样使 SE 乘以 `1/sqrt(2) ≈ 0.707`（~29%），**不是减半**；
> - 要使 SE 减半，必须让 m（及内存）翻 4 倍，或等价地让 p 增加 2。
