# FM-index 反向搜索的 LF-mapping 性质 —— 纯逻辑推导

> 材料来源：P. Ferragina, G. Manzini, *"Opportunistic Data Structures with
> Applications"* (FOCS 2000)，即 FM-index 原始论文。本节在不查原文 T 的
> 前提下，仅凭 BWT 的最后一列 L、数组 C、以及对 L 的 rank 查询，
> 解释为何「从右向左扫描」计数模式 P 的出现次数是正确的。

---

## 0. 符号约定

- 文本 `T[1..u]`，其中 `T[u] = #` 是最小字符且全文唯一。
- 记 `T#` 即 `T`（已带 `#` 终止符）。
- 后缀数组 `A[1..u]`：按字典序排列的所有后缀起始位置。
- BWT 矩阵 `M`（u×u）：第 i 行 = `T#(A[i]..u) · T#(1..A[i]-1)`，
  即「T# 的第 A[i] 个循环移位」。M 的行按其内容字典序升序排列。
- `F[i] = M[i][1] = T#(A[i])`（首列）。
- `L[i] = M[i][u] = T#(A[i]-1 mod u)`（末列；`0 ≡ u` 处理 A[i]=1 的环回）。
- `C[c] = #{x ∈ T# : x < c}`，即 `T#` 中**字典序严格小于** `c` 的字符数
  （包含唯一的 `#`）。所以 `C[c]+1` 是 `F` 中**第一个** `c` 出现的位置。
- `rank_c(L, k) = #{j ∈ [1, k] : L[j] = c}`（即 `Occ(c, 1, k)`）。

注意：M 的每一行都是 T# 的一个循环移位；因此每一列都是 T# 的字符的
一个排列。特别地，`L` 与 `F` 含**完全相同**的字符多重集（各字符个数
都等于其在 T# 中的个数）。

---

## 1. LF-mapping 的定义

> 定义（LF-mapping）：对任意行 i，令 `LF[i]` 是使 `F[LF[i]] = L[i]` 的
> 行号；即「以第 i 行末字符为首字符的那一行」。

由 M 的构造直接读出：

```
L[i] = T#(A[i]-1 mod u)              (1)
LF[i] = rank_A(A[i]-1 mod u)         (2)
```

直观理解：第 i 行的循环移位对象是「从位置 A[i] 开始的 T# 后缀」；
把末位 L[i] 移到首位（再做一次循环移位）后得到的就是「从位置 A[i]-1
开始的后缀」对应的行——这正是 `LF[i]`。

---

## 2. 核心命题：i-th c in L ↔ i-th c in F

**命题 P**（rank-preserving bijection）：对任意字符 c，设按行号升序
排列的「`L` 列中字符 c 出现」的位置为

```
i_1 < i_2 < ... < i_k，其中 L[i_r] = c，k = #{j : T#(j) = c}
```

则

```
LF[i_1] < LF[i_2] < ... < LF[i_k]，
且 LF[i_r] = C[c] + r。                                  (3)
```

这正是题目所问的「L 中字符 c 的第 i 次出现 = F 中字符 c 的第 i 次出现」
的严格表述。

### 2.1 证明：双射 + 顺序保持 + 首位置

**(a) 集合上的双射**

对 T# 中每个字符 c 出现的位置 p（即 `T#(p) = c`）：

- 以 c 为末字符的那一行是「后缀 `T#(p+1..u)·T#(1..p)` 所在的行」——
  它的末位是 `T#(p) = c`，故 `L = c`。
- 以 c 为首字符的那一行是「后缀 `T#(p..u)` 所在的行」——
  它的首位是 `T#(p) = c`，故 `F = c`。

这两种行**一一对应**：从后者做一次循环右移就得到前者。
因此「`L = c` 的行集合」与「`F = c` 的行集合」之间存在**双射**，
且该双射正是 `LF`（在 c 上的限制）。

**(b) 双射保持行号顺序**

行号 `i` 由后缀字典序决定：`i < i'` 当且仅当
`T#(A[i]..u) < T#(A[i']..u)`。

设 `L[i] = c`、`L[i'] = c` 且 `i < i'`，则

```
T#(A[i]-1..u) = c · T#(A[i]..u)
T#(A[i']-1..u) = c · T#(A[i']..u)
```

两者首字符同为 c，字典序由后续部分 `T#(A[i]..u)`、`T#(A[i']..u)` 决定。
故 `i < i'` 蕴含 `T#(A[i]..u) < T#(A[i']..u)`，进而

```
T#(A[i]-1..u) < T#(A[i']-1..u)
⇒ LF[i] = rank_A(A[i]-1) < LF[i'] = rank_A(A[i']-1)
```

所以 `i_1 < i_2 < ... < i_k  ⇒  LF[i_1] < LF[i_2] < ... < LF[i_k]`。

**(c) 首位置 = C[c]+1**

F 列是 T# 字符按字典序的排列（每字符出现次数等于在 T# 中出现次数），
首位 `#`，随后是所有非 # 字符升序。因此

```
F[1]               = #                              （位置 1）
F[2..C[c]+1]       = T# 中 < c 的字符                （共 C[c] 个）
F[C[c]+2..]        = T# 中 ≥ c 的字符
⇒ 第一个 c 出现在 F[C[c]+1] 处（1-indexed）
```

（核对：T# 中小于 c 的字符共 C[c] 个，加上首位的 #，占 F 的前 C[c]+1
个位置；故首 c 的位置为 `C[c]+1`。）

综合 (a)(b)(c)，`L` 中第 r 次出现的 c（行 `i_r`）对应 `F` 中第 r 个 c
（行 `C[c]+r`），即 `LF[i_r] = C[c] + r`。命题 P 证毕。

把命题 P 写成更紧凑的算式：

```
LF[i] = C[L[i]] + rank_{L[i]}(L, i)        当 L[i] ∈ Σ 时   (4)
```

这正是论文中 backward BWT 的公式（Section 2 中的 LF-mapping 公式），
也是论文附录 B 中 Step 5/6 所基于的等式。

## 3. 反向搜索（Backward Search）：区间收窄公式

设 `P[1..p]` 是查询模式。维护行区间 `[sp, ep]`，使不变式为：

> **不变式**：在第 i 阶段结束后，`[sp, ep]` 恰好是 M 中所有循环移位以
> `P[i+1..p]` 为结尾的行（即这些行的循环移位内容在尾部包含 `P[i+1..p]`，
> 而不在此区间的行均不包含）。

### 3.1 初始化

令 `c = P[p]`。F 中首字符为 c 的所有行恰好落在 `[C[c]+1, C[c+1]]`
（C[c+1] 是下一个字符在 F 中的首位置，或等价地 C[c]+1+|{j : T#(j)=c}|）。
因此

```
sp = C[c] + 1,    ep = C[c+1]                       (5)
```

就使不变式对 i = p 成立（即「匹配 P[p]」对应于「F = P[p]」）。

### 3.2 归纳步：把字符 c = P[i] 前置于已匹配后缀 P[i+1..p]

要找的行必须同时满足两个条件：

1. 循环移位以 `P[i+1..p]` 结尾 ⇔ 行号在 `[sp, ep]` 中（不变式假设）；
2. 把 `c` 前置 ⇔ 该行的末字符是 `c` ⇔ `L = c`。

**关键洞察**（由命题 P）：`[sp, ep]` 内 L 列中字符 c 的第 r 次出现
（按行号升序），通过 LF 恰好对应到 F 列中字符 c 的第 r 次出现。

设 `rank_c(L, sp-1) = m` 是在 `L[1..sp-1]` 内 c 出现的次数；则

- `[sp, ep]` 内 `L = c` 的首次出现对应 rank = `m+1`（若存在），
  落在 F 的位置 `C[c] + (m+1)`；
- `[sp, ep]` 内 `L = c` 的最末次出现对应 rank = `rank_c(L, ep)`，
  落在 F 的位置 `C[c] + rank_c(L, ep)`。

因此新区间为：

```
sp' = C[c] + rank_c(L, sp-1) + 1
ep' = C[c] + rank_c(L, ep)                              (6)
```

这正是论文 Fig. 1（Algorithm BW_Search）Step 5、6 的公式。

### 3.3 终止条件

若 `ep' < sp'`，新区间为空，说明 `P[i..p]` 不出现；否则
`ep' - sp' + 1` 即为 `P[i..p]` 在 T 中的出现次数。

回到 i = 1 时即得到 `P[1..p]` 完整的出现次数。

## 4. 整个过程没有用过 T 与 F

检查一遍反向搜索的「输入」：

- **L**：BWT 的末列（论文中存为压缩形式 `BW_RLX(T)`）；
- **C**：长度 |Σ| 的数组；
- **rank_c(L, ·) 查询**：在论文中由 `Occ(c, 1, k)` 在 `O(1)` 时间内
  回答（借助存储的 `NO, W, NO', W', MTF, S` 等辅助结构）。

而**完全没有使用**：

- 原文 `T`；
- 首列 `F`（虽然 LF-mapping 在逻辑上把行号映到 `F` 中 c-段的某个位置，
  但这个位置由 `C[c] + rank` 算出，无需实际存 F）；
- 整个矩阵 `M` 或后缀数组 `A`；
- 后缀 `T#(A[i]..u)` 的内容。

这正是 FM-index 的精髓：**L + C + rank 已经隐含了 F 和 M 的全部结构
信息**，可以在「只压缩存储」的代价下完成模式计数。

---

## 5. 几何直觉小结

把 M 想象成一面按字典序排列的「墙」：

- **F 列**是「按字符分段的字典序坐标」：`C[c]+1` 是字符 c 段的起点。
- **L 列**是「原文里的字符在每段内部的字典序排列」。
- **LF-mapping** 是把「段内某字符的次序」折回「该字符对应的新段内次序」
  的双射：L 中字符 c 的第 r 次 → F 中字符 c 的第 r 次。

反向扫描模式时，每向左加一个字符 c，就用一次 LF 把区间从「c 段内
已匹配后缀子段」映射回「已匹配后缀的段内 c 子段」——这就是 (6) 式中
`sp', ep'` 的几何含义。

## 6. 与论文 BW_Search 算法的对应

论文 Figure 1（Section 3.1）：

```
Algorithm BW_Search(P[1,p])
 1.  c = P[p], i = p;
 2.  sp = C[c] + 1, ep = C[c+1];
 3.  while ((sp ≤ ep) and (i ≥ 2)) do
 4.      c = P[i-1];
 5.      sp = C[c] + Occ(c,1,sp-1) + 1;
 6.      ep = C[c] + Occ(c,1,ep);
 7.      i = i-1;
 8.  if (ep < sp) return "not found" else return "found (ep-sp+1) occurrences".
```

其中 `Occ(c,1,k) = rank_c(L, k)` 即 §2 中的 `rank`；第 5、6 行的公式
正是本文 §3.2 推导出的 (6) 式。第 2 行对应 §3.1 的初始化 (5)。

论文 Lemma 1（Appendix B）以更形式化的语言证明：若 sp / ep 指向
M 中首/末行被 `P[i,p]` 前缀的行（即 `M[s] = cP[i,p]α`，
`M[e] = cP[i,p]β`，αβ 任意），则 Step 5/6 正确地把 sp/ep 收窄为
被 `P[i-1,p]` 前缀的首/末行——其证明的关键步骤正是

```
s = C[c] + Occ(c, 1, sp-1) + 1
```

（附录 B 行 "s is equal to C[c] plus the rank of M[s'] among all rows in
M ending with the character c"），这正是命题 P 的直接推论。

## 7. 一句话回答（直接对应题目两问）

1. **为何 L 中字符 c 的第 i 次出现 = F 中字符 c 的第 i 次出现？**
   因为 L=c 的行与 F=c 的行之间存在「循环移位」诱导的双射，且该双射
   保持字典序（由 c·T#(A[i]..u) 与 T#(A[i]..u) 的字典序等价性），
   从而保持「在同一字符段内的次序」。

2. **如何用 `rank(c,·)` 与 `C[c]` 在每步正确收窄行区间？**
   由 (1) 的次序保持性，`[sp, ep]` 内 L 列 c 的 rank 区间
   `[rank_c(L, sp-1)+1, rank_c(L, ep)]` 平移到 F 列 c 段
   `[C[c]+1, C[c+1]]` 后，即为新区间

   ```
   sp' = C[c] + rank_c(L, sp-1) + 1
   ep' = C[c] + rank_c(L, ep)
   ```

   这只需 L 列、C 数组与 rank 查询，无需查阅原文 T，也无需查阅 F 或
   后缀数组。
