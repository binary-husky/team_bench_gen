# Count-Min Sketch: 宽度 `w` 对点查询高估误差的影响

## 1. 实验设置

| 参数 | 值 |
| --- | --- |
| 深度 `d` | **5** (固定) |
| 宽度 `w` | 128, 256, 512, 1024, 2048, 4096 |
| 流长度 `N` | 1,000,000 |
| 不同 item 数 `n` | 100,000 |
| 频率分布 | **均匀** + **Zipfian(s = 1.0)** |
| 哈希种子数 | 3 (独立重复取均值) |
| 哈希族 | mmh3 128-bit, 每行不同 salt `(seed·131 + j) mod 2³²` |
| 更新方式 | `update(item, 1)`, 全 cash-register (非负) |
| 查询方式 | `â_i = min_j count[j, h_j(i)]` (点查询, 仅 d=5 行取 min) |

实现：`cm_experiment.py` / `cm_analyze.py`，CPU，~5 秒/每 (分布, w) 组合。Count-Min 论文 (Cormode & Muthukrishnan 2003) 第 4.1 节为点查询理论依据。

## 4. 结论

### 4.1 高估误差随 `w` 近似 `1/w` 下降 ✅

- 在固定 `d = 5` 下，将 `w` 由 128 加倍到 4096（三次翻倍，即 32×），两种分布的均值高估都下降 ≈ 30×–40×。
- 均匀：7469 → 185，缩放 40.3×，理论 32×。
- Zipfian：4084 → 57，缩放 71.7×，理论 32×（Zipfian 略快于 1/w 因为 min‑over‑rows 对高方差分布更友好，见 4.2）。
- 任何相邻两档 `w` 之间，误差比都在 2.0–2.5 之间，与"翻倍 ≈ 减半"高度一致。
- 与 Theorem 1 的 `â_i ≤ a_i + ε‖a‖₁`（其中 `ε = e/w`）完全自洽：实验观测到 `w · 均值 ≈ ‖a‖₁ = N`，且随 `w` 略微下降（因为 min 操作的方差缩减因子随 `w` 增大）。

### 4.2 Zipfian 偏态分布的高估并未大于均匀——反而更小 ❌（修正朴素直觉）

实验结果：

| `w`  | 均匀均值 | Zipfian 均值 | 均匀 p99 | Zipfian p99 |
| ---: | -------: | -----------: | -------: | ----------: |
|  128 |   7469   |        4084  |   7716   |    5081    |
|  4096|    185   |          57  |    227   |      90    |

Zipfian 的均值/p99/最大值 **全部小于** 均匀分布（同等 `w` 下，Zipfian 误差约为均匀的 1/2–1/3），与任务提示中"高频 item ‖a‖₁ 大、碰撞贡献被放大"的朴素直觉相反。

**原因分析（与定理 1 兼容）：**

1. `‖a‖₁` 两种分布完全相同（均为 N = 1e6），所以定理 1 给出的上界 `ε‖a‖₁` 相同。
2. **每行 per‑row 误差的均值** 都是 `(N − a_i)/w ≈ ‖a‖₁/w`，与分布无关。关键差异在 *方差*：
   - 均匀分布：每个 row `j` 的桶计数 ≈ `a_i + Poisson(N/w)`，方差 ~ `N/w` ≈ 244（w=4096）。
   - Zipfian 分布：少数极重 item（a₁ ≈ 8.3e4）若与查询 item 落到同一桶，会以 `a₁/w ≈ 20` 的量级瞬间拉高该 row 的计数；否则该 row 计数接近 0。这造成 per‑row 计数方差远大于均匀分布。
3. CM 用的是 `min_j count[j, h_j(i)]`，**取 min 对方差高的分布更有利**——只要 d 行里有任一行"幸运地"躲过重 item 的碰撞，那一行的桶计数会非常小，min 就被拉低。Zipfian 的高方差让这种"幸运行"出现的概率远高于均匀。
4. Zipfian 中 *绝大多数* item（>99%）的真实计数 ≤ 10，它们在被查询时几乎不会把任何重 item 拉到自己的桶上；取 min 后误差接近 0。均匀中每个 item 计数都约 10，每个 item 都"均匀地"承受 ≈ 244 的碰撞误差。
5. 因此 Zipfian 的高估是 *集中* 在少数 item 上的（重 item 之间互相碰撞），但平均/分位数被长尾上的零误差主导。最大误差层面 Zipfian (≈6773 at w=128) 仍小于均匀 (≈7964)，因为 Zipfian 重 item 的 a_i ≈ 8e4，本身太大，相对误差远小于均匀 item 的 6×–9×。

### 4.3 与论文理论的一致性

- Theorem 1 的高概率上界 `â_i ≤ a_i + ε‖a‖₁`（其中 `ε = e/w, w = ⌈e/ε⌉`）：上界随 `w` 反比下降，与实验一致。
- 我们的 `w · 均值` 在均匀下基本常数（≈ N），在 Zipfian 下随 `w` 缓慢下降——后者来自 min‑over‑rows 的方差缩减效应，不是分布本身的原因。
- 对于偏态分布，CM 凭借 `min` 操作对重 item 的过估天然 *鲁棒*（重 item 估计误差仅 ≈ 36/82919 ≈ 0.04% at w=4096），这正是 Cormode & Muthukrishnan 设计 CM 的核心动机。
