# Count-Min 点查询的单边误差（"只高估、不低估"）的结构化证明

> 任务：纯粹从更新与查询的结构出发，**不使用任何概率误差界**，严格证明 Count-Min Sketch 的点查询估计 `â[item]` 永远不会低估真实频率 `a[item]`，即 `â[item] ≥ a[item]`。本证明针对只增（cash-register）模型，所有更新量 `c ≥ 0`。

---

## 0. 记号与对象回顾

- 数据流把隐式向量 `a ∈ ℝ^n` 增量地构造出来，每个分量 `a[i]` 表示 item `i` 到目前为止的累计频率。
- CM Sketch 是 `d × w` 的计数器数组 `C[1..d][1..w]`，初始全部为 0；并有 `d` 个两两独立的哈希函数 `h_1, …, h_d : {1,…,n} → {1,…,w}`。
- 更新 `update(i, c)`（`c ≥ 0`）：对每个 `j = 1..d`，`C[j][h_j(i)] ← C[j][h_j(i)] + c`。
- 点查询 `point_query(i)`：返回 `â[i] = min_{j=1..d} C[j][h_j(i)]`。

下面对三个推理环节依次给出严格论证。

---

## 1. 单行视角：固定任意一行 `j`，必有 `C[j][h_j(i)] ≥ a[i]`

固定任意一行 `j ∈ {1,…,d}`，看计数器 `C[j][h_j(i)]` 在整条流结束后的最终值。

**每次更新 `update(k, c)`（`c ≥ 0`）对该计数器的影响只有两种可能：**

- **情形 A（自身更新）：** `k = i`。此时 `h_j(k) = h_j(i)`，于是计数器 `C[j][h_j(i)]` 加上 `c`。这恰好是 `a[i]` 自身累加量中的一次。
- **情形 B（别人"撞"进来）：** `k ≠ i` 但 `h_j(k) = h_j(i)`。此时该 item 的更新 `c` 也被加到同一个桶 `C[j][h_j(i)]` 里。
- **情形 C（不撞）：** `k ≠ i` 且 `h_j(k) ≠ h_j(i)`。此时对 `C[j][h_j(i)]` 没有影响。

把这三类更新对 `C[j][h_j(i)]` 的贡献加起来：

```
C[j][h_j(i)]  =  Σ_{所有更新 (k, c)} c · 1[h_j(k) = h_j(i)]
              =  Σ_{k=i} c · 1                          （情形 A：自身真实计数）
               + Σ_{k≠i 且 h_j(k)=h_j(i)} c · 1         （情形 B：碰撞带来的"噪声"）
```

把"自身真实计数"这部分提出来，记作 `A_j(i)`，把"碰撞噪声"记作 `N_j(i) ≥ 0`，得到：

```
C[j][h_j(i)]  =  A_j(i)  +  N_j(i),      其中  A_j(i) = a[i],   N_j(i) ≥ 0
```

**关键观察**：

1. `A_j(i) = a[i]`：因为每次 `update(i, c)` 都把 `c` 加到 `C[j][h_j(i)]`，而 `a[i]` 正是这些 `c` 的总和。更新规则与 item 自身的位置 `h_j(i)` 在每一行都一致，**没有任何 item 能绕过"自己那一格"**。
2. `N_j(i) ≥ 0`：碰撞项是若干个非负 `c` 的求和，且更新永远只加不减（`c ≥ 0`），所以这部分只会让计数器变大，绝不会让它变小。

**因此，对每一行 `j` 都有**

> **C[j][h_j(i)] = a[i] + N_j(i) ≥ a[i]，其中 N_j(i) 是由其他 item 与 `i` 在第 `j` 行发生哈希碰撞所引入的非负贡献。**

这一步是确定性的、**与哈希函数具体长什么样无关**：哪怕 `h_j` 退化成常数函数、或把所有 item 都映射到同一个桶，等式 `C[j][h_j(i)] = a[i] + N_j(i) ≥ a[i]` 仍然成立（只不过此时 `N_j(i)` 可能很大）。

---

## 2. 取 min 的保号性：`min_j C[j][h_j(i)] ≥ a[i]`

第 1 步告诉我们，对**每一行** `j = 1..d`，都有

```
C[1][h_1(i)] ≥ a[i],
C[2][h_2(i)] ≥ a[i],
   ⋮
C[d][h_d(i)] ≥ a[i].
```

换言之，`{ C[j][h_j(i)] : j = 1..d }` 是一个**每一项都 ≥ `a[i]`** 的数集。

**而任意一个"每一项都 ≥ M"的有限集合，它的最小值也 ≥ M。** 形式化地：

```
设集合 S = {x_1, …, x_d}，若 ∀j, x_j ≥ M，则 min_j x_j ≥ M。
```

证明（反证）：若 `min_j x_j < M`，则存在某个 `j*` 使 `x_{j*} < M`，与前提 "∀j, x_j ≥ M" 矛盾。

把这个事实套到 `S = { C[1][h_1(i)], …, C[d][h_d(i)] }` 和 `M = a[i]` 上，立刻得到：

> **â[i] = min_{j=1..d} C[j][h_j(i)] ≥ a[i].**

注意这一步**只用了"最小值的保号性"**——没有任何关于哈希分布、概率或独立性的假设。它纯粹是初等不等式的事实。

把第 1 步的等式分解 `C[j][h_j(i)] = a[i] + N_j(i)` 代入 min，还可以写得更明白：

```
â[i]  =  min_j ( a[i] + N_j(i) )
       =  a[i] + min_j N_j(i)             （因为 a[i] 与 j 无关，可以提出 min 之外）
       ≥  a[i]                            （因为 min_j N_j(i) ≥ 0）
```

两种写法殊途同归：min 之所以能"消去"碰撞噪声，是因为它挑的是所有行里**碰撞最少**的那一行，而自身真实计数 `a[i]` 是每一行都共有的下界。

---

## 3. 单调性推论：在只增模型下，`â[i]` 永远单调不减

cash-register 模型要求每次更新的增量 `c ≥ 0`，且向量 `a` 只能随时间变长：`a[i]` 本身就是流的**单调不减**函数。

**定理**：在只增流上，`â[i]` 也是单调不减的，即当流从时刻 `t` 推进到时刻 `t+1` 时，`â[i]_(t+1) ≥ â[i]_(t)`。

**证明**：

把第 2 步的结构性等式 `â[i] = a[i] + min_j N_j(i)` 应用到时刻 `t` 与时刻 `t+1`：

- 真实计数部分：`a[i]_(t+1) = a[i]_(t) + c_(t+1) ≥ a[i]_(t)`（新更新的 `c ≥ 0`）。
- 碰撞噪声部分：`N_j(i)_(t+1) = N_j(i)_(t) + Δ_j ≥ N_j(i)_(t)`，其中 `Δ_j` 是新更新在第 `j` 行产生的额外碰撞贡献（也可能为 0）。因为 `c ≥ 0`，新引入的碰撞贡献 `Δ_j ≥ 0`，**绝不会"抹掉"过去的碰撞计数**——计数器只能加，不能减。
- 因此 `min_j N_j(i)_(t+1) ≥ min_j N_j(i)_(t)`。

把两部分加起来：

```
â[i]_(t+1)  =  a[i]_(t+1) + min_j N_j(i)_(t+1)
            ≥  a[i]_(t)    + min_j N_j(i)_(t)
            =  â[i]_(t).
```

**为什么会这样**：CM Sketch 的所有操作都是单调的——更新永远 `+= c`（`c ≥ 0`），计数器永远只增不减；min 又是单调函数（输入每一项都变大，输出也只会变大或不变）。两层单调叠加，就让 `â[i]` 整体上"只能升、不能降"。

这也正是单边误差的几何直觉：**噪声是"累加"上去的，永远不会被清零**。Hash 碰撞造成的多余计数，会让某些行的 `C[j][h_j(i)]` 比真值大；而 min 选最小的那个，仍然夹在真值之上。如果某条 item 后来真的被再次更新，对应行又被加上一个非负量；但已经发生过的碰撞既不会因为"过段时间别的 item 来了"而被扣掉，也不会因为 item 自身没有更新而减小。整张草图只会越来越"满"，不会越来越"空"。

---

## 4. 结论

> **Count-Min 的点查询是"只高估、不低估"的单边误差估计（在 cash-register / 只增模型下）。**

直观地说，原因有三层：

1. **结构上**：`C[j][h_j(i)] = a[i] + N_j(i)`，其中 `N_j(i) ≥ 0`。每个计数器都把"真实频率"和"由哈希碰撞引入的非负噪声"加在了一起，所以单行必然 `≥ 真值`。
2. **算子上**：min 是一个"保号"的算子——一组每个都 `≥ M` 的数，它们的最小值也 `≥ M`。所以跨 `d` 行取 min 不会跌破 `a[i]`。
3. **时间上**：在只增流上，更新只能往计数器里"加东西"（`c ≥ 0`），既有的碰撞噪声不会被冲销，于是 `â[i]` 随时间单调不减；真实频率 `a[i]` 也在单调增，两条曲线永远有 `â[i]` 在上、`a[i]` 在下，且二者之差 `min_j N_j(i)` 也是单调不减的。

三层叠加，就给出了论文 Theorem 1 中那个看上去"显然"、但本次任务要求严格结构化推出来的核心不等式：

> **对任意 item `i`、任意数据流、任意（哪怕是退化、极端对抗的）哈希函数：**
> **`â[i] ≥ a[i]`，恒成立，无例外。**

也正因为这条不等式是**确定性**的而非概率性的，Count-Min 才能被放心地用在"只接受正向偏差"的场景里，比如 heavy hitters 的判别——`â[i]` 只要过了阈值 `φ‖a‖_1`，就**保证** `a[i]` 也已经过了阈值 `(φ−ε)‖a‖_1`；永远不会出现"估计说很大、真实其实很小"的方向性误判。

---

## 5. 与概率误差界的关系（备注）

本次证明**完全没用**到哈希的两两独立性，也没有用到 Markov 不等式——它只用到了"非负加法"和"min 保号"这两个初等事实。这并不与论文 Theorem 1 中的概率上界 `â[i] ≤ a[i] + ε‖a‖_1`（概率 ≥ 1−δ）矛盾，而是与之互补：

- `â[i] ≥ a[i]`：**无条件、无概率**，纯由结构决定。
- `â[i] ≤ a[i] + ε‖a‖_1`：**需要**两两独立性 + Markov 才有 `1 − δ` 的把握。

也就是说，本次任务所证的"下界方向"是 CM Sketch 最便宜也最坚固的性质——这也是论文第 4.1 节末那条"the error bound here is one-sided, as opposed to all previous constructions which gave two-sided errors"的底气来源。
