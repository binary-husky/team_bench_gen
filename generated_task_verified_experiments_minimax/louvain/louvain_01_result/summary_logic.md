# Louvain 第一阶段：把接受规则从 ΔQ > 0 改成 ΔQ ≥ 0 的逻辑推理

## 0. 题目与材料回顾

依据 *Fast unfolding of communities in large networks* (Blondel, Guillaume, Lambiotte, Lefebvre, 2008) 第 2 节（Method）：

- 模块度 Q = (1/2m) Σ_ij [A_ij − k_i k_j / 2m] · δ(c_i, c_j)，其值被夹在 [−1/2, 1] 之间。
- 第一阶段（local moving）依次访问每个节点 i：把 i 从其当前社区取出，再放到使 ΔQ 最大的邻居 j 所属社区；当存在平局时用 breaking rule 决定；但只有当 ΔQ > 0 时才真正接受这次移动。
- 第一阶段在「没有任何单点移动能改进 Q」时停止，即停在 Q 的一个局部极大；然后做第二阶段（社区聚合），并把两阶段合起来称为一个 pass。

题目问：把接受规则从 "ΔQ > 0" 改成 "ΔQ ≥ 0"（平局零增益也接受）后，是否还保留

1. 性质 A：「模块度随每次接受移动严格递增」（严格单调性）；
2. 性质 B：「每个 local-moving pass 必然终止」？

并要求说明：哪条性质会丢失、为什么、以及它会如何导致算法不再终止。

下面是完全通过逻辑推理得到的回答，不依赖任何实验运行。

---

## 1. 原算法下两条性质为什么成立

**性质 A（严格单调性）—— 显然成立。**
按定义，只有 ΔQ > 0 的移动才被接受，所以每一步都把 Q 推到一个比当前更严格大的值，Q 形成一条严格递增的轨迹。

**性质 B（pass 必终止）—— 性质 A 直接蕴含。**
证明链：

- 状态空间有限。节点数 N 固定时，{1,…,N} 上的分拆是有限的（Bell 数 B_N 有限）。
- 由性质 A，每接受一步，Q 都严格大于上一步的 Q；Q 又以 1 为上界（更精细地由 2m 和节点度数决定），因此 Q 的取值只可能出现在一个有限的离散子集上。
- 把"当前分拆"作为状态：每个被接受移动都对应到严格更大的 Q，所以同一个分拆不可能被访问两次。
- 状态有限、又不会被重复访问 ⇒ 接受移动的次数有限 ⇒ 节点被"反复扫若干遍"的过程必结束 ⇒ 一个 pass 必终止。

所以在原算法里，**性质 A 是性质 B 的证明工具**：正是因为有严格单调，才能保证"不可逆地走向一个上界"。

---

## 2. 把规则改成 ΔQ ≥ 0 后，两条性质都丢失

### 2.1 性质 A（严格单调性）首先被破坏

新规则只要求 ΔQ ≥ 0。所以当一次移动的 ΔQ 严格等于 0 时，Q 在这一步**保持不变**。
此时"Q 在每一步都严格递增"这个命题在形式上已经不成立——Q 现在只是「弱单调」(non-decreasing) 而已。

所以性质 A 在改规则的第一时间就被丢掉了。注意：弱单调仍然成立（Q 永远不会下降），但题目问的是"严格递增"，所以这一条性质**不**再保留。

### 2.2 性质 B（pass 必终止）随之不再成立

性质 A 的丢失，把性质 B 的证明链彻底切断：

- 没有严格单调 ⇒ 同一个分拆可以被再次访问（只要再次访问时 Q 仍然是同一个值）。
- 状态空间是有限的。
- 在一个有限状态空间上，如果存在"Q 保持不变"的状态转移边，并且这些边可以组成一条回到起点的有向回路，那么算法就会沿着这条回路无限循环。
- 因此，"每个 pass 必然终止"这个**保证**不再成立。

注意：算法仍然可能终止（例如恰好落进一个没有任何 ΔQ ≥ 0 移动的局部极大点上），但**没有一般性的保证**——存在具体图、具体节点访问顺序、具体平局规则下，算法可以无止境地转。

### 2.3 一句话总结因果链

> 严格单调（A）丢失 ⇒ 同 Q 值状态可被重复访问 ⇒ 有限状态空间上的循环是可能的 ⇒ pass 不再保证终止（B 也丢失）。

所以"哪条性质会丢失"的标准答案是：**性质 A（严格单调性）是被直接丢掉的性质；而性质 B（pass 必终止）的丢失，是性质 A 丢失的直接后果。**

---

## 3. 一个具体的小图，让"循环"在眼前发生

考虑 3 个节点的路径图 1 — 2 — 3，权重均为 1，因此 m = 2、2m = 4。逐项计算 Q：

| 分拆 | Σ_in | Σ_tot | 该社区贡献 Σ_in − (Σ_tot)²/2m | Q |
|---|---|---|---|---|
| {1, 2, 3} | 2 | 4 | 2 − 16/4 = −2 | −1/2 |
| {1, 2}, {3} | 1, 0 | 3, 1 | (1 − 9/4), (0 − 1/4) = −5/4, −1/4 | (1/4)(−6/4) = **−3/8** |
| {1}, {2, 3} | 0, 1 | 1, 3 | −1/4, −5/4 | (1/4)(−6/4) = **−3/8** |
| {1}, {2}, {3} | 0, 0, 0 | 1, 2, 1 | −1/4, −1, −1/4 | (1/4)(−3/2) = **−3/8** |

关键观察：**Q({1,2}, {3}) = Q({1}, {2,3}) = −3/8**——这是"两个不同分拆共享同一个 Q 值"的最简例子。

现在看 ΔQ：

- 把节点 2 从 {1,2} 移到 {3}：k_{2,in} = 1（边 2–3 那一端），目标社区 Σ_in = 0、Σ_tot = 1；
  新分拆 = {1}, {2,3}，Q 新 = −3/8。
  ΔQ = Q新 − Q旧 = 0。
- 把节点 2 从 {1}, {2,3} 移到 {1}：同样 ΔQ = 0。

在原算法下，这两次 ΔQ = 0 的移动都会被拒绝，所以算法在第一个 pass 内很快停住。但在 ΔQ ≥ 0 规则下（且平局规则为「在多个等好选项中仍做出某个移动」），会发生：

- Pass 1：从 {1,2}, {3} 出发，访问节点 2 时 ΔQ = 0，接受，把节点 2 移进 {3}；分拆变成 {1}, {2,3}。
- Pass 2：从 {1}, {2,3} 出发，访问节点 2 时 ΔQ = 0，接受，把节点 2 移进 {1}；分拆变回 {1,2}, {3}。
- Pass 3：又回到 {1}, {2,3}……
- 永不终止。

整个循环里 Q 始终等于 −3/8——也直观地说明，性质 A（严格单调）已经丢失，性质 B（终止保证）随之失效。

---

## 4. 进一步的推论

1. **弱单调仍然成立。** ΔQ ≥ 0 ⇒ Q 不下降，所以 Q 仍然沿着一个不减的轨迹移动，只是它可能长时间停留在同一个 Q 值上而不上升。
2. **任何合理的 breaking rule 都救不了这个循环。** 关键不是 breaking rule 选哪个邻居，而是**只要允许 ΔQ = 0 的移动被执行**，就有可能在 Q 不变的方向上走出一个回到起点的环。要回避循环，就只能在 ΔQ = 0 时拒绝移动——但那恰好等价于把规则改回 ΔQ > 0，新规则名存实亡。
3. **聚合阶段（第二阶段）也救不了。** 第二阶段的作用是把"不再动的社区"压成一个超节点再开新一轮，但只要第一阶段本身在某些图上就不会终止，整个 Louvain 算法在最外层就失去了"有限步内能进入下一阶段"的保证，构造层次结构的整个工作流都不再有终止性结论。
4. **状态空间有限 + 弱单调 ⇒ "终止" 与 "循环" 二选一。** 这是在有限状态空间上做非严格递增搜索的标准困境——这是把"严格"放成"非严格"之后丢失的根本机制。

---

## 5. 结论（直接回答题目三问）

- **会丢失的两条性质**：严格单调性（性质 A）和 pass 必终止性（性质 B）。
- **"哪条性质会丢失"** 这个问题最尖锐的回答是：**性质 A 才是被直接丢掉的性质，性质 B 是它的逻辑后果**。换句话说，原算法用"严格单调"换来了"必终止"，把规则改成 ΔQ ≥ 0 等于先把"严格单调"丢掉了，从而把"必终止"也带走了。
- **它如何导致算法不再终止**：失去严格单调后，Q 不再每步严格上升，Q 值相同的不同分拆之间的零增益移动被允许执行；在一个有限的分拆空间里，这样的零增益移动完全可以组成一条回到原状态的有向环——上面 3-节点路径 1–2–3 在两个分拆 {1,2}, {3} 和 {1}, {2,3} 之间反复横跳就是最简例子。算法于是陷入这个环，每个 pass 永远走不到"没有任何 ΔQ ≥ 0 移动"的那个终止条件。
