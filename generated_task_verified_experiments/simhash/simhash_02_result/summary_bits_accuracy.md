# SimHash 估计余弦相似度的精度随位数 b 的变化

> 任务：考察 SimHash 用汉明距离估计余弦相似度的精度随位数 b 的变化。
> 固定设置：`d=100`，`1000` 对随机向量（unit-normal 归一化），固定种子 `seed=12345`；`b ∈ {16, 64, 256, 1024}`；`sketch = sign(G @ v)`，`G` 为 `b×d` 随机高斯矩阵。唯一自变量为 b。
> 估计式：汉明距离 `hd` ⇒ `θ̂ = π·hd/b` ⇒ `cosθ̂`；与真实 `cosθ` 比较得逐对误差。
> 代码：`code/simhash_bits.py`（纯 numpy/CPU）；原始数值：`results/bits_accuracy.json`。

## 主结果（1000 对，seed=12345）

| b | MAE | RMSE | mean_err(偏差) | std_err | RMSE·√b |
|---:|---:|---:|---:|---:|---:|
| 16 | 0.30519 | 0.37226 | +0.01081 | 0.37211 | 1.489 |
| 64 | 0.15088 | 0.18580 | −0.00252 | 0.18579 | 1.486 |
| 256 | 0.07693 | 0.09749 | +0.00583 | 0.09732 | 1.560 |
| 1024 | 0.03924 | 0.04902 | −0.00157 | 0.04900 | 1.569 |

## 与 O(1/√b) 对比

- **RMSE·√b 近常数**（1.486–1.569），符合 `RMSE = O(1/√b)`。
- 以 b=16 为基准预测各 b 的 RMSE（`pred = RMSE₁₆·√16/√b`），实测比 `actual/pred`：b=16 1.000、b=64 0.998、b=256 1.048、b=1024 1.053——大 b 略偏高（约 +5%），因 `θ̂=π·hd/b` 的非线性（`cos` 在 0 附近近似线性、远离 0 时弯曲）带来微小偏差，但量级吻合。
- **减半误差需约 4 倍位数**：b→4b 时 RMSE 比 = 0.499 / 0.525 / 0.503，均 ≈0.5，与 `std∝1/√b` ⇒ 减半需 `b×4` 一致。

## 偏差（无偏性）

各 b 的 `mean_err` = +0.0108 / −0.0025 / +0.0058 / −0.0016，符号交替、绝对值 ≤0.011（远小于对应 RMSE 0.049–0.372），估计**近似无偏**。

## 结论

1. **估计误差随 b 增大而下降**：RMSE 0.372 → 0.186 → 0.097 → 0.049（b=16→1024，约 7.6× 降）。
2. **量级符合 O(1/√b)**：RMSE·√b 近常数 1.49–1.57；减半误差需约 4 倍位数（b→4b 比例 ≈0.50）。
3. **近似无偏**：各 b 平均误差 |mean_err| ≤ 0.011 ≪ RMSE，符号交替无系统偏移。

## 一句话总结

1000 对 d=100 向量、b∈{16,64,256,1024}：RMSE 随 b 降（0.372→0.049），RMSE·√b≈常数 1.49–1.57 ⇒ O(1/√b)、减半需 4 倍位数（比例 ≈0.50），平均误差 ≤0.011 ≪ RMSE ⇒ 近似无偏。
