# SimHash（余弦）vs MinHash（Jaccard）在同一语料上的估计误差对比

## 实验设置（固定）

| 参数 | 值 |
|---|---|
| 特征词表规模 D | 16384 (2^14) |
| 向量/集合数 N | 500 |
| 每个向量激活特征数 \|A\| | 256 |
| SimHash 位数 b | 256（256 个随机超平面，分量 ~ N(0,1)） |
| MinHash 哈希函数数 | 256（与 b 相同） |
| 随机种子 | 42 |

- 生成 N 个随机稀疏二值向量：每个从 D 中无放回抽取 256 个特征。这些向量同时被视作集合，因此可在同一配对上同时定义余弦相似度与 Jaccard 相似度。
- 自变量仅为**估计器**（SimHash vs MinHash），其余全部固定。
- 评估全部 C(500,2)=**124750** 个无序配对，误差 = |估计值 − 真值|。
- 真值：`cos = |A∩B|/√(|A||B|)`（因 \|A\|=\|B\|=256，即 = |A∩B|/256）；`jac = |A∩B|/|A∪B|`。
- SimHash 余弦估计给出两种：原理型 `cos(π·H/b)`（H=汉明距离）与 Charikar 线性型 `1−2H/b`。
- MinHash：自实现，对每个 set 取 256 个随机排列下的最小 rank，Jaccard 估计 = 签名相同列的比例。

> 完整可复现代码见 `experiment.py`，数值见 `results.json`。运行约 4 秒（CPU only）。

## 真值分布（本语料的相似度天然很低）

| 真值 | 均值 | 最小 | 最大 |
|---|---|---|---|
| 余弦 cos_true | 0.0156 | 0.0000 | 0.0586 |
| Jaccard jac_true | 0.0079 | 0.0000 | 0.0302 |

由于词表大、向量稀疏，配对重叠极小。等规模集合下二者有精确关系 `jac = cos/(2−cos)`（自检误差 0.00e+00），所以 jac ≈ cos/2，量级约为余弦的一半——这一点在比较绝对误差时需留意。

## 估计误差对比（核心结果）

| 估计器（目标量） | MAE 均值 | 中位数 | p95 | 最大 | std |
|---|---|---|---|---|---|
| **SimHash**（余弦，原理 `cos(πH/b)`） | **0.0778** | 0.0658 | 0.191 | 0.424 | 0.0586 |
| SimHash（余弦，线性 `1−2H/b`） | 0.0500 | 0.0430 | 0.125 | 0.281 | 0.0379 |
| **MinHash**（Jaccard） | **0.00417** | 0.00384 | 0.0116 | 0.0368 | 0.00356 |

相对误差（MAE / 真值均值，衡量“把信号估成几倍噪声”）：

- SimHash 原理型：0.0778 / 0.0156 ≈ **4.99×**
- SimHash 线性型：0.0500 / 0.0156 ≈ **3.20×**
- MinHash：0.00417 / 0.0079 ≈ **0.53×**

### 结论一：在本（低相似度）regime 下，SimHash 对余弦的估计误差远大于 MinHash 对 Jaccard 的误差

- 绝对 MAE：SimHash 原理型 (0.0778) 约为 MinHash (0.00417) 的 **18.6 倍**；即便用更稳的线性型 (0.0500) 也是约 **12 倍**。
- 相对误差：SimHash 把真值放大数倍估计，MinHash 误差不到真值均值的六成。
- 原因分析：SimHash 通过“超平面符号”间接估计角度，每位翻转在 `H/b≈0.5`（即余弦≈0）附近对应的角度变化被 `cos(π·H/b)` 强非线性放大——低相似度区域方差极大，估计值常摆到 ±0.2 甚至负值（见下表样本：真值 0.012 被估成 0.267、真值 0.016 被估成 −0.147）。线性型 `1−2H/b` 在 0 附近更平缓，故误差更小但仍远大于 MinHash。
- MinHash 直接对“签名碰撞”计数，是无偏的 Jaccard 抽样估计，方差 ~ O(1/√b)，在 b=256 下精度高、对低 Jaccard 仍稳健。

## 二者在同一配对上数值不同

即便真值层面 `cos` 与 `jac` 一一对应（corr=1.0000），两者的**估计值在同一配对上数值差异显著**：

- `|SimHash余弦估计 − MinHash Jaccard 估计|`：均值 0.0782，中位数 0.0658，p95 0.191，最大 0.420。
- 124750 对中 **114707 对（91.9%）** 两估计值相差 > 0.01。
- 估计值之间的 Pearson 相关仅 **0.0485**（真值之间是 1.0）——即 SimHash 估计的噪声几乎完全掩盖了它与 Jaccard 估计本应共有的同源信号。

### 结论二：同一配对上两估计器给出的数值几乎完全不同

这是“用同一 b 但估不同量、且 SimHash 在低相似度下方差大”共同作用的结果：MinHash 估计紧贴真值（≈0.008 量级），而 SimHash 估计在 ±0.2 量级剧烈摆动，故二者在同一对上极少吻合。

## 方法论说明与可比性声明

- SimHash 与 MinHash 估的是**不同度量**（余弦 vs Jaccard），绝对误差不直接可比；本文同时给出绝对 MAE、相对误差（除以各自真值均值）与样本，以便公平解读。无论哪种口径，在本低相似度语料上 SimHash 误差都显著更大。
- 如需“同度量”对比，可对等规模集合用 `jac = cos/(2−cos)` 互相换算后比；此处遵循任务要求——SimHash 估余弦、MinHash 估 Jaccard——各报各的误差。
- 结果随种子固定可精确复现（seed=42）。regime 改变（更高重叠/更小词表）时 SimHash 相对劣势会缩小，但 MinHash 在 Jaccard 上的低误差特性稳健。

## 一句话总结

在同一组随机稀疏二值向量上（D=16384、N=500、每向量 256 特征、b=256、seed=42，共 124750 对）：**SimHash 对余弦的 MAE≈0.078（线性型 0.050），MinHash 对 Jaccard 的 MAE≈0.0042，前者约为后者的 12–19 倍；且在同一配对上两估计器数值差异显著（91.9% 的对相差 >0.01，估计值相关仅 0.05）**——低相似度 regime 下 SimHash 经非线性放大方差很大，而 MinHash 作为无偏抽样估计精度稳健。
