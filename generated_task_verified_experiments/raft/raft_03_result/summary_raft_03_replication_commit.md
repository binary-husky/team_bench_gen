# Raft 日志复制与提交实验结论

> 任务：用进程内 Raft 仿真器（Python、内存消息队列、逻辑 tick 时钟、确定性种子、无真实网络/socket/Docker）复现 AppendEntries 日志复制与提交（commitIndex 推进），验证「条目在多数节点上复制后即被提交、commitIndex 单调推进」。
> 材料：`raft_material/raft.pdf`（Ongaro & Ousterhout, *In Search of an Understandable Consensus Algorithm*, 扩展版）。

## 1. 实验设置（固定，未改动）

| 参数 | 取值 |
|---|---|
| 客户端命令数 | 1000（整数序列，逐 tick 流式提交给 leader） |
| 节点数对比 | N=3 与 N=5 各一组 |
| 随机种子 | 每个 N 跑 5 个不同种子：1,2,3,4,5 |
| 网络 | 进程内全互联，消息按 tick 投递，无丢包/无分区/无宕机 |
| 每消息投递延迟 | 由种子确定的 jitter ∈ {1,2,3} tick（**非丢包**，仅模拟逐消息网络抖动，使「需更多 follower 确认」的阶统计量差异可观） |
| 选举超时 | 随机 ∈ [20,40] tick（充分随机化，避免 split-vote） |
| 心跳间隔 | 2 tick |
| 选举 | 完整 RequestVote：随机化超时 + 简多数投票 + up-to-date 日志比较 |

记录项：(a) 每个 entry 被复制到的节点数；(b) 是否被提交（commitIndex 是否覆盖到该 index）；(c) 每条 entry 的提交时延（append 到 commit 的逻辑 tick 数）；(d) commitIndex 是否单调不减。

仿真器代码：`raft_sim.py`；绘图脚本：`make_plot.py`；原始聚合结果：`results.json`；逐 entry 时延分布图：`latency_dist.png`；汇总 CSV：`latency_stats.csv`。

## 3. 结论要点

1. **全部提交**：N=3 与 N=5 两组、各 5 个种子下，1000 条客户端命令最终 100% 被提交（final commitIndex = 1000，提交比例 1.00）。无丢包/无故障的全互联网络下，AppendEntries 复制流水线把所有条目最终都拉到了多数节点并提交。

2. **提交前必先在严格多数节点完成复制**：每条被提交的 entry，在 leader 推进 commitIndex 覆盖到它的那一刻，持有该 entry 的节点数都恰好等于「严格多数」——N=3 全部为 2（>3/2），N=5 全部为 3（>5/2），无一例外（`all_strict_majority=True`）。这是 Raft leader 提交规则的直接体现：leader 只在 `match_index >= N` 的节点数达到多数时，才把 current-term 的 entry N（及其之前）提交，因此「已提交 ⇒ 先在严格多数上复制」由机制本身保证，实验中 10000 条提交条目无一违反。

3. **N=5 提交时延大于 N=3**：是。N=3 均值 3.16 tick、N=5 均值 3.41 tick（约 +8%），最大值相近但 N=5 在 4–6 tick 的尾部更厚。原因：提交要求 leader 收到 `majority−1` 个 follower 的 AppendEntries 成功回复——N=3 只需 1 个、N=5 需 2 个；在带 jitter 的并行投递下，N=5 的提交时刻由「4 个 follower 中第 2 快的往返」决定，而 N=3 由「2 个中最快」决定，前者作为更高阶统计量系统性偏大，故 N=5 时延更大。两组中位数同为 3 tick（多数提交在最常见的 1+1+1 tick 往返下完成），差异主要体现在右尾。

4. **commitIndex 全程单调不减**：是。两组全部种子下 `monotonic_violations = 0`，commitIndex 序列严格非递减，从未出现回退。leader 的 `maybe_advance_commit` 只把 commitIndex 向前推进到更大的 N，从不下调，符合 Raft 安全性。

5. **额外观察**：复制节点数恒等于严格多数下限（N=3→2、N=5→3）而非更高，是因为 commit 在达成多数的瞬间即触发，leader 不必等其余慢节点即提交——这正体现「多数复制即可提交、不必全员」。整轮（10 次仿真 + 绘图）< 30 分钟。

## 4. 复现方法

```bash
python3 raft_sim.py        # 跑 N=3/N=5 各 5 种子，打印表并写 results.json
python3 make_plot.py       # 重跑并生成 latency_dist.png / latency_stats.csv
```
