# 随机 3-SAT 相变点附近求解难度实验总结

## 1. 实验设置

- **变量数**：固定 n = 50
- **子句/变量比 α = m/n**：{3.0, 3.5, 4.0, 4.267, 4.5, 5.0, 6.0}
- **每个 α 的实例数**：15 个（随机种子 seed = 0..14）
- **实例生成**：每个子句从 50 个变量中无放回抽取 3 个不同变量，每个文字以 0.5 概率取正/负（标准随机 3-SAT 模型）。对应子句数 m = round(α·n)。
- **求解器**：PySAT 的 `Solver(name='Minisat22')`，`bootstrap_with` 载入公式，`use_timer=True`。
- **记录指标**：每个实例的冲突数 `conflicts`（`accum_stats()`）、求解器内部计时 `time()`、以及包构造+求解的壁钟时间 `perf_counter`。
- **唯一自变量**：α。其余（n、α 取值、随机种子、求解器）全部固定。
- **运行环境**：CPU-only；Python 3.11 + python-sat 1.9.dev5。

## 3. 平均冲突数 / 平均时间 / SAT 占比 随 α 的变化

### 3.1 SAT 占比随 α 的变化
SAT 占比随 α 单调下降，呈现典型的 S 形相变曲线：
- α ≤ 3.5：几乎所有实例可满足（100%）。
- α = 4.0：86.7%，开始有不可满足实例出现。
- α = 4.267：66.7%（理论阈值附近，约 2/3 仍可满足）。
- α = 4.5：40.0%，SAT/UNSAT 大致对半——**50% 临界点落在 α ≈ 4.3–4.5 之间**。
- α = 5.0：仅 6.7%。
- α = 6.0：0%，全部不可满足。

由于 n=50 较小，相变区间被展宽并略微右移；50% 临界点（≈4.4）与渐近理论阈值 α_c ≈ 4.267 基本一致，符合有限尺寸效应预期。

### 3.2 平均冲突数随 α 的变化
冲突数呈现明显的**单峰**形态，在相变区达到最大：
- α=3.0→4.0：随 α 增大缓慢上升（5.3 → 29.2）。
- α=4.267–5.0：达到峰值区间（45.1 / 54.2 / 53.4）。**峰值出现在 α≈4.5**，略高于渐近阈值 4.267。
- α=6.0：回落到 33.6（实例高度过约束，几乎必然 UNSAT，求解器很快得出矛盾证明，冲突数反而下降）。

冲突数峰值与 SAT 占比的 50% 临界点基本重合（都在 α≈4.3–4.5），印证「相变点附近最难」的经典结论。

### 3.3 平均求解时间随 α 的变化
求解时间随 α 的变化趋势与冲突数一致（同为单峰，峰值在 α≈4.5）：
- α=3.0–3.5：~0.00009 s（壁钟），几乎瞬时。
- α=4.0：0.00013 s。
- α=4.5：0.000164 s（峰值）。
- α=6.0：0.000136 s（回落）。

需注意：n=50 对现代 CDCL 求解器（MiniSat22）而言极小，所有实例均在**亚毫秒级**完成，绝对时间差异很小；因此**冲突数是比时间更可靠、更显著的难度指标**。时间曲线的形状与冲突数一致，但量级差异不显著。内部计时与壁钟计时趋势一致。

## 4. 结论

1. **相变存在且可观测**：随机 3-SAT 在 n=50 下，SAT 占比随 α 从 100% 陡降到 0%，50% 临界点约在 α≈4.3–4.5，与渐近理论阈值 α_c≈4.267 吻合（有限尺寸下略有展宽/右移）。
2. **求解难度在相变区最高**：平均冲突数与平均时间均在 α≈4.5（相变点附近）达到峰值，随后在过约束区（α=6.0）回落——因为高 α 下实例几乎必然 UNSAT，矛盾易于发现。
3. **冲突数是更敏感的难度指标**：由于 n=50 实例对 MiniSat22 过于容易（亚毫秒级求解），绝对时间差异被噪声/计时分辨率压低；冲突数的单峰结构清晰得多，更适合刻画相变附近的难度峰值。
4. **材料印证**：MiniSat（Een & Sörensson, 2003）的冲突驱动子句学习（CDCL）机制使得小规模实例在相变点也只需数十次冲突即可定解；峰值冲突数（~54）远低于大规模相变实例的典型量级，正是 n=50 有限尺寸下的体现。

**核心结论**：在固定 n=50、仅改变 α 的条件下，平均冲突数与平均时间随 α 呈单峰变化、峰值位于相变点 α≈4.3–4.5；SAT 占比随 α 单调骤降并在同一点穿越 50%——三者共同刻画了随机 3-SAT 的相变现象，且难度峰值与可满足性临界点重合。
