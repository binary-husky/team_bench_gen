# 为什么学习子句 C 后不会再因同一冲突赋值再次冲突

> 任务要求：仅逻辑推理（不做实验）解释——CDCL first-UIP 学习子句 C 后，为何不会因「当初伪造 C 的那组赋值」再次冲突；明确 (i) C 相对原公式的逻辑地位；(ii) 断言层上哪个机制结构上阻止该冲突赋值再现。
> 依据 Een & Sörensson, *An Extensible SAT-solver* (SAT 2003)。

---

## (i) C 相对原公式的逻辑地位：合法消解式 ⇒ 可靠

学习子句 `C` 是在蕴含图上以 **first-UIP** 为切入点、对冲突子句逐级消解（对每个决策/蕴含变量作超限消解）得到的。每一步消解都是**保可满足性**的推理：消解式被原参与子句逻辑蕴含，加入 `C` 不改变公式的可满足性（`F ⊨ C`，`F` 可满足 ⟺ `F∧C` 可满足）。故 `C` 是原公式的一条**合法、被蕴含**的子句，加入它总是可靠的——这是「学习」之所以安全的根基。

## (ii) 断言层上的机制：C 退化为单元 ⇒ 强制断言文字 ⇒ 剪除冲突子赋值

回跳到**断言层**（assertion level，即 `C` 中第二高决策层）后：

- `C` 中除**断言文字** `a` 外的所有文字，其所对应的变量在该层及以下**都已被赋为使该文字为假的值**（这是断言层的定义：first-UIP 之下、回跳目标层之上其余文字皆假）。
- 于是 `C` 在断言层**退化为单元子句**（仅 `a` 未赋值，其余文字皆假 ⇒ 欲使 `C` 满足必须 `a` 为真）。
- **单元传播**强制断言 `a` 取所需真值。

**结构上为何阻止冲突赋值再现**：当初伪造 `C` 的那组赋值，恰是「`C` 中所有文字皆为假」。回跳后，一旦搜索试图重新建立这组赋值中的任意一个「使非断言文字为假」的部分，`C` 就会**更早地**被触发为单元（或冲突）——因为 `C` 现在是公式的一部分，单元传播会在那组赋值**尚未全部完成**时就强制 `a` 为真（或立即冲突），从而**结构上不可能**再让 `C` 的全部文字同时为假。每条学习子句都**永久**地把「当初伪造它的那组冲突子赋值」从未来搜索中剪除，于是 CDCL 单调推进（不重复同一冲突状态）并最终终止。

## 一句话总结

(i) `C` 是原公式的合法消解式、被原公式蕴含，加入不改可满足性 ⇒ 可靠；(ii) 回跳到断言层后 `C` 中非断言文字皆假、`C` 退化为单元，由单元传播强制断言文字取所需真值——一旦搜索试图重建当初伪造 `C` 的那组赋值，`C` 会更早被触发为单元/冲突，故该冲突子赋值永不再现，这正是 CDCL 单调推进与终止的根本原因。
