# CDCL vs 朴素 DPLL：冲突驱动子句学习的威力

## 任务与材料

考察「冲突驱动子句学习（CDCL）」相对于「无学习的朴素 DPLL」的威力。材料为
MiniSAT 论文（Een & Sörensson, *An Extensible SAT-solver*, 2003/2004），它正是
CDCL 的标杆实现（含 1-UIP 学习子句、VSIDS 决策、非时序回跳、几何重启）。

## 方法

**实例生成**：随机 3-SAT，子句数 m = round(α·n)，α≈4.2（恰在 3-SAT 相变点
α≈4.267 附近，最难区域）。每个 n 取 5 个随机种子（seed=0..4）。所有实例存为
DIMACS `.cnf`（见 `instances/`），可复现。

**朴素 DPLL**（`experiment.py`，自实现）：
- 单元传播（unit propagation）+ 按时间顺序回溯（chronological backtracking）；
- 静态变量序（取下标最小的未赋值变量），先试正再试负；
- **不做**子句学习、**不做** VSIDS、**不做**重启；
- `decisions` = 分支猜测次数（每次对变量赋一个猜测值计 1 次，含翻极性）。

**MiniSAT (CDCL)**（PySAT 的 `minisat22`，即 MiniSAT 2.2）：记录冲突数 `conflicts`、
决策数 `decisions`、传播数、重启数与用时。两者均设单实例超时上限。

**固定设置**：n ∈ {15,20,25}，α≈4.2，5 个种子，单实例 60s 上限。为揭示随 n 的
渐进趋势，另补一组更大 n 的扫描（n∈{30,40,50,60,80,100,130,160}，2–3 个种子，
30s 上限），明确标注为补充实验。CPU-only，无 GPU。

## 结论

1. **决策/冲突数随 n 的变化呈鲜明对比**。朴素 DPLL 的决策数随 n 近似指数增长：
   n=40 约 10²，n=60 约 10³–10⁴，n=100 约 5×10⁴，n=130 已达 1.4×10⁵ 并触发超时，
   n=160 两个实例全部 30s 超时。MiniSAT 的冲突数则始终在 10¹–10³ 量级，n=160 也
   仅约 2×10³ 冲突、用时 <16ms，**全程零超时**。同等精度下，n=100 时 DPLL 决策数
   约为 MiniSAT 决策数的 90–250 倍，用时比约 5000 倍；n 越大差距越悬殊。

2. **时延对比**。在固定的小规模（n≤25）下两者都 <1ms，难分伯仲；但 n≥80 起 DPLL
   用时从 1.4s 飙升至 7s（n=100）再到超时（n≥130），而 MiniSAT 始终在毫秒级。DPLL
   的「可用」边界大致在 n≈100–130，CDCL 则在所测最大规模（n=160）仍轻松。

3. **为什么 CDCL 强这么多**——正对应 MiniSAT 论文的核心机制：
   - **子句学习**：每次冲突导出一条 1-UIP 否决子句（learned clause）并永久加入子句库，
     等价于把「这棵子树为何无解」记录为通用约束；而朴素 DPLL 不学习，回溯后会在
     不同的决策路径上**反复重新发现同一个失败**，因此其决策数随 n 指数膨胀。
   - **非时序回跳**：CDCL 直接跳到回跳层（implication graph 的断言层），跳过无关分支，
     而 DPLL 只能逐层时序回退。
   - **VSIDS + 重启**：聚焦高频冲突变量并周期性重启逃离坏分支，把搜索集中在「硬」区域；
     DPLL 用静态序、无重启，常在易区域浪费并卡在坏分支。
   三者合力使 CDCL 的搜索复杂度近似随问题「真正的难度」（冲突数）增长，而非随决策树
   体积指数增长——这正是 DPLL 的指数灾难被 CDCL 压成准多项式/亚指数的根因。

4. **关于小规模**：在 n∈{15,20,25} 这一固定规格下，随机 3-SAT 实例本身只有几十个
   决策的搜索树，朴素 DPLL 已足够，CDCL 的学习机制「杀鸡用牛刀」，优势仅 ~2× 且
   绝对时间都接近零。CDCL 的威力**随 n 增大而急剧放大**，在 n≈100 之后 DPLL 已不可
   行、CDCL 仍瞬时——这才是「冲突驱动子句学习相对于无学习 DPLL 的威力」的实证。

### 设置说明与局限
- 决策数口径：DPLL 的 `decisions` 为分支猜测次数；MiniSAT 的 `decisions` 为其内部
  计数（含重启后的重决策）。两者口径不完全一致，故横向比较以「数量级与随 n 的
  斜率」为准，并辅以冲突数（CDCL 的核心工作量度量）与时延。
- α=4.2 近相变点，SAT/UNSAT 混合；UNSAT 实例需穷尽证明，是 DPLL 缺乏学习时受害最
  重的场景（n=160 两个超时实例恰好如此）。
- 30s/60s 超时为墙钟；DPLL 超时记录为 `sat=None`、`timed_out=True`。
- 补充扫描（n≥30）超出任务固定规格，仅用于展示随 n 的趋势；结论不依赖其具体规模，
  仅依赖「DPLL 决策数随 n 指数增长、MiniSAT 冲突数与时延近线性/常数」这一单调现象。
