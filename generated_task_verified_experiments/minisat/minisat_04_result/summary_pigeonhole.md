# 鸽巢原理（PHP）结构性难例对 MiniSAT 求解器的影响

## 实验设置（固定项）

- **公式编码**：PHP_n —— 将 n+1 只鸽子放入 n 个洞，恒不可满足。
  - 变量 `x_{i,j}`：鸽子 i（1..n+1）是否进洞 j（1..n），变量编号 `(i-1)*n + j`。
  - **正子句**（每只鸽子至少进一洞）：对每个 i，`x_{i,1} ∨ … ∨ x_{i,n}`。
  - **二文字互斥子句**（每个洞至多一只鸽子）：对每个洞 j 与每对 i1<i2，`¬x_{i1,j} ∨ ¬x_{i2,j}`。
- **n 取值**：`{3, 4, 5, 6, 7}`。
- **求解器**：PySAT 的 MiniSAT，即 `minisat22`（MiniSAT 2.2）。
- **唯一自变量**：n。求解结果在所有 n 下均为 **UNSAT**（符合 PHP 恒不可满足）。

## 冲突数与时间随 n 的增长

**冲突数**（核心指标）随 n 的增长：

| n→n+1 | 冲突数比值 | 时间比值 |
|-------|-----------|----------|
| 3→4 | 4.1× | ~1× |
| 4→5 | 6.9× | ~4× |
| 5→6 | 12.4× | 12.5× |
| 6→7 | 4.6× | 6.8× |

冲突数从 n=3 的 **7** 增长到 n=7 的 **11 530**，约 **1 647 倍**。每增加一阶 n，冲突数大致乘以 5–12 倍，呈**指数级**增长（PHP_n 的最短反驳证明规模本身约为 2^{Θ(n)}，这是 CDCL/分辨率证明系统的已知下界现象）。

**时间**同步指数增长：从 n=3 的约 0.0001 s 升至 n=7 的 0.034 s，跨越约 2.5 个数量级。在 n≤7 范围内 MiniSAT 仍能在 0.04 s 内跑完，但按 5–12×/阶的外推，n=8 预计约 0.2–0.4 s，n=9 约数秒，n=10 约数十至上百秒——增长曲线陡峭，是典型的「结构性难例」特征。

## 结论

- PHP 编码对 MiniSAT 构成**结构性难例**：尽管公式极小（n=7 时仅 56 变量、204 子句），冲突数却高达 1.15 万、传播近 17 万次。规模与难度严重脱节，难度的来源是结构（鸽巢不可能性）而非公式体积。
- 冲突数与时间均随 n **指数增长**，每增一阶放大数倍至十余倍，符合鸽巢原理在基于分辨率的 CDCL 求解器上指数级证明复杂度的理论预测。
- 在 n≤7 区间，求解仍很快（<0.04 s）；但曲线斜率陡峭，n 再增 2–3 阶即可迅速进入「很慢」区间（任务提示的「n=7 附近可能已很慢」在本机本次运行中尚未触顶，但增长趋势已明显显现）。
- 固定编码、取值与求解器后，唯一自变量 n 的效果清晰可辨：**n 越大，冲突数与时间指数级攀升**，验证了鸽巢原理作为 SAT 求解器结构性难例的典型表现。
