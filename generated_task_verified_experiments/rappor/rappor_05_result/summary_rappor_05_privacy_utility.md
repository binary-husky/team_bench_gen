# RAPPOR 隐私-效用权衡：PRR `f` / ε_perm 对频率估计误差的影响

## 实验设置（固定，按 task.md）

- 候选字典 `M = 20` 个字符串（google.com … tumblr.com）；真实频率分布：少数高频 + 长尾（最高 0.206，最低 0.0021，已归一化）。
- 客户端数 `N = 5×10⁴`。
- Bloom：`k = 128`, `h = 4`（md5 派生的 h 个哈希）。
- IRR：`p = 0.5`, `q = 0.75`（固定）。
- 扫描 PRR 参数 **`f ∈ {0.1, 0.25, 0.5, 0.75, 0.9}`**；对应的永久隐私
  `ε_perm = ln((2−f)/f)`，从 `f=0.1` 的 ≈2.94（弱隐私）单调降到 `f=0.9` 的 ≈0.20（强隐私）。
- 解码：IRR 报告按位去偏 → 恢复 Bloom 位概率 → 非负最小二乘 (NNLS) 在候选字典上反解候选频率，归一化。
- 每个 `f` 用 **4 个随机种子**（1,2,3,7）重复，取均值±标准差。
- 基线：**非私有直接计数**（客户端如实上报字符串，服务端 `bincount/N`），同样 4 个种子。
- 实现：`rappor_pipeline_05.py`（纯 NumPy/SciPy，CPU），结果存于 `rappor_results_05.npz`，图见 `privacy_utility_plot.png`。

> PRR 机制（与 rappor_03/04 同款）：每位以概率 `f` 换成新鲜 Bernoulli(½)、否则保留 Bloom 位，`E[B'_i]=(1−f)B_i+f/2`，等价于"以概率 1−f/2 保留、f/2 翻转"。该机制提供 `ε_perm=ln((2−f)/f)` 的本地差分隐私。

## 结论要点

1. **隐私-效用单调权衡成立。** `ε` 越小（`f` 越大、隐私越强）→ 频率估计误差越大。
   从 `f=0.1`（ε≈2.94，弱隐私）到 `f=0.9`（ε≈0.20，强隐私），L1 误差从 0.082 单调升到 0.610（约 **7.5×**），max|err| 从 0.012 升到 0.091（约 **7.7×**）。误差随 `f`/`1/ε` 近似单调膨胀，且在强隐私端方差也显著增大（L1 的 std 从 ~0.010 涨到 ~0.066）——更难稳定估计。

2. **非私有基线误差 ≈ 0。** 直接计数的 L1 ≈ 0.012、max|err| ≈ 0.0018，仅来自 `N=5×10⁴` 的有限抽样噪声（理论 1/√N 量级 ~10⁻³），与"如实上报"应有零误差一致。它就是 RAPPOR 各 `f` 误差所附加的隐私成本的参照零点。

3. **量化"本地隐私所购买的效用损失"。** 相对基线：
   - `f=0.1`（ε≈2.94）：L1 ≈ 0.082，比基线高约 **6.9×**——隐私扰动小、效用损失可接受。
   - `f=0.5`（ε≈1.10）：L1 ≈ 0.144，约 **12×** 基线。
   - `f=0.9`（ε≈0.20）：L1 ≈ 0.610，约 **51×** 基线——强隐私下长尾候选几乎不可分辨，效用大幅牺牲。
   即把 ε 从 ~3 压到 ~0.2，付出的代价是估计误差放大约一个量级。

4. **`f`（等价地 ε）就是隐私-效用旋钮。** 在固定 `N,k,h,p,q` 下，分析师无需改协议、仅调一个参数即可在"低噪声高泄露风险 ↔ 高噪声强本地 DP"之间连续移动。本数据下若要 L1 控制在 ~0.1 附近，需 ε ≳ 2（即 f ≲ 0.25）；要 ε<0.5（f>0.75）则 L1 必然 >0.28。这给出了在给定 `N` 下选择 ε 时效用预算的显式约束。
