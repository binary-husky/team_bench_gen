# Louvain 接受规则 ΔQ>0 改为 ΔQ≥0 会丢失什么

> 任务要求：仅逻辑推理（不做实验）——把局部移动接受规则从 ΔQ>0 改为 ΔQ≥0，是否还保留「模块度严格递增」与「pass 必然终止」？哪条丢失、为何、如何导致不终止。
> 依据 Blondel et al., *Fast unfolding of communities in large networks* (2008)。

---

## 1. 原算法的两条性质及其论证

Louvain 第一阶段：对每个节点 `i`，移入能使 ΔQ 最大的邻居社区，**仅当 ΔQ > 0** 才接受；一个 pass 内无任何 ΔQ>0 移动时结束。原算法依赖：

- **(P1) 严格递增**：每次接受 ΔQ>0 ⇒ 模块度 `Q` 严格上升。
- **(P2) 必然终止**：`Q` 严格递增 + `Q` 有上界（`Q ≤ 1`，模块度定义上界）+ 划分状态有限（节点到社区的映射有限）⇒ 严格递增序列在有界有限状态空间上不可能无限延续 ⇒ pass 必然终止（落到单节点移动意义下的局部最优）。

## 2. 改为 ΔQ≥0 后哪条丢失

接受 ΔQ=0（零增益平局移动）后：

- **(P1) 丢失**：模块度只**非降**（ΔQ≥0），不再**严格递增**——零增益移动不改变 `Q`。
- **(P2) 随之丢失**：原终止论证的三段「严格递增 + 有上界 + 有限状态」中，**严格递增**这一段失效。非降序列在有界空间上**可以无限震荡**而不违反任何一条：零增益移动让节点在两个「吸引力相等」的社区之间无限来回翻转（或搜索在若干等 `Q` 划分间反复游走），`Q` 始终不变 ⇒ 没有上界/有限状态能阻止这种「同 `Q` 水平上的循环」⇒ pass 可能**永不终止**。

## 3. 一句话总结

ΔQ>0 ⇒ `Q` 严格递增 + `Q≤1` + 状态有限 ⇒ pass 必然终止（局部最优）；改为 ΔQ≥0 ⇒ `Q` 仅非降，零增益移动使节点可在等 `Q` 社区间无限来回翻转、`Q` 不变而状态循环 ⇒ 「严格递增」与「必然终止」两条性质均丢失，算法可能不终止。
