问题：Louvain 的第一阶段（局部移动）只在能带来正的模块度增益 ΔQ > 0 时，才把节点 i 移入能使 ΔQ 最大的邻居社区；当一个 pass 内不再存在任何 ΔQ > 0 的移动时，该 pass 结束。假设有人把接受规则从「ΔQ > 0」改成「ΔQ ≥ 0」（即也接受零增益的平局移动）。请推理：这个改动是否还保留原算法「模块度随每次接受移动严格递增」以及「每个局部移动 pass 必然终止」这两条性质？哪条性质会丢失、为什么，以及它会如何导致算法不再终止？

1. 标准答案：两条性质都会丢失（至少「严格递增」丢失，进而殃及「必然终止」）。原因：接受 ΔQ = 0 的移动后，模块度只是非降而非严格递增；由于零增益移动不改变模块度，节点可在两个「吸引力相等」的社区之间无限来回翻转（或搜索在相同划分之间反复游走）而不改变 Q，于是原本「严格递增 + 模块度有上界（≤1）+ 划分状态有限 ⇒ 必然终止」的论证不再成立，修改后的算法可能永不终止。原 ΔQ > 0 规则正是靠严格单调性保证有限终止、并使每个 pass 落到（单节点移动意义下的）局部最优。
