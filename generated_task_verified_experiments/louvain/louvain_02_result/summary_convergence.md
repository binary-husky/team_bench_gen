# Louvain 多轮（pass）迭代模块度收敛分析

## 实验设置

- **算法**：`community_louvain.generate_dendrogram(G, random_state=0)`
  - dendrogram 的每一层 = 一次 pass 之后的划分；用 `partition_at_level` 映射回原始节点。
- **数据**：networkx LFR 基准图
  - `n=1000, tau1=3, tau2=1.5, mu=0.4, average_degree=6, min_community=20, seed=0`
  - 该参数**首次生成即成功**，无需微调。
  - 生成图：1000 个节点，2044 条边。
- **模块度 Q**：对每一层用 `community_louvain.modularity(partition, G)` 计算（基于原始图）。
- **固定项**：上述 LFR 参数与 `random_state=0` 均固定；唯一考察对象是 pass 轮次。

## 实验结果

总轮数（dendrogram 层数）= **4**。每一层把上一层的小社区再聚合一次，induced graph 节点数依次递减：1000 → 323 → 83 → 24。

| Pass（层） | 社区数 | 模块度 Q | ΔQ（相对上一轮） |
|:---:|:---:|:---:|:---:|
| 0 | 323 | 0.360066 | — |
| 1 | 83  | 0.523549 | +0.163483 |
| 2 | 24  | 0.559394 | +0.035845 |
| 3 | 20  | 0.559732 | +0.000338 |

## 收敛趋势

1. **Q 单调上升**：每一轮 pass 都使模块度增加，从未出现回退，符合 Louvain "每轮贪心局部优化只会提升（不降低）Q" 的设计。
2. **增益迅速衰减**：
   - 第 1 轮贡献最大（+0.1635），单 pass 把 323 个原子社区聚合到 83 个，Q 从 0.36 跃升到 0.52；
   - 第 2 轮增益骤减至 +0.0358，社区数 83→24；
   - 第 3 轮增益近乎为零（+0.00034），社区数 24→20，Q 基本封顶。
3. **收敛点**：约在第 2~3 轮之间收敛，最终 Q ≈ **0.5597**，社区数稳定在 **20**（与 LFR `min_community=20` 的先验社区规模量级吻合）。
4. **停止判据**：当 induced graph 已无足够能带来正增益的合并时（本例第 3 轮 ΔQ ≈ 3×10⁻⁴），dendrogram 自然终止，即算法已无法再通过聚合提升 Q —— 典型的"边际收益递减、快速饱和"的收敛曲线。

## 结论

在上述 LFR 图、`random_state=0` 的固定设置下，Louvain 经 **4 轮 pass** 收敛：模块度 Q 从第 0 轮的 0.360 单调上升至最终 0.560，社区数从 323 收缩到 20。绝大部分增益（约 64%）来自首轮 pass，随后边际增益急剧衰减，第 3 轮已基本饱和（ΔQ ≈ 3×10⁻⁴），呈现"先快后慢、单调不减、快速收敛"的典型模式。
