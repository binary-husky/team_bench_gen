# CRDT 强最终一致性（SEC）收敛实验结论

> 任务 `crdt_02`：用 Python 进程内实现 5 种 state-based CRDT（CvRDT），副本为内存对象、消息经内存队列在全互联（full-mesh）拓扑上交换，验证**强最终一致性（SEC）**——所有副本在收到相同的更新集合后状态必然等价。
> 实现与实验脚本：`crdt_sec_experiment.py`。理论依据：Shapiro 等《A comprehensive study of Convergent and Commutative Replicated Data Types》(2011)，`crdt_material/crdt_shapiro_2011.pdf`（Def. 2.4/2.5：`merge(x,y)=x⊔y` 为最小上界/join，满足交换、幂等、结合 ⇒ 单调半格 ⇒ CvRDT ⇒ SEC）。

## 1. 实现的 5 种 CvRDT（每个含本地 `update` 与 `merge`）

| CRDT | payload 状态 | 本地 update | `merge`（对应 join） | value 查询 |
|---|---|---|---|---|
| **G-Counter** | 整数向量 `P[i]`（每副本一项） | `increment(i)`: `P[i]+=1` | 逐分量 `max` | `Σ P[i]` |
| **PN-Counter** | 两个 G-Counter `P`（增）、`N`（减） | `increment`/`decrement` | 各向量逐分量 `max` | `ΣP − ΣN` |
| **G-Set** | 只增集合 `S` | `add(e)` | 集合并 `∪` | `S` |
| **OR-Set** | `(element, unique-tag)` 对集合 | `add` 新增唯一 tag；`remove` 删除**已观测到的**对 | 对集合并 `∪` | 存在 ≥1 tag 的元素 |
| **LWW-Register** | `(value, timestamp)` | `set` 赋更高时间戳 | 取**时间戳更大者** | 当前 value |

时间戳为全序 `(logical_clock, node_id)`（Lamport 风格，node_id 破平局），保证 LWW 的 `merge` 无歧义且构成半格。OR-Set 的 tag 形如 `(node_id, 本地计数器)`，全局唯一。

## 2. 固定实验设置（与任务一致）

- 副本数 **N ∈ {3, 4, 5}**，每组各跑。
- 每副本先执行随机本地操作序列，**混合 5 种 CRDT 的 update**；每次运行总操作数 ∈ **[1000, 10000]**（实测范围 N=3:[1778,9522]、N=4:[1735,9424]、N=5:[2293,9783]）。
- **确定随机种子**（`seed*1009 + N*37`）保证可复现；每 N 用 **8 个种子**（≥5）。
- 交换方式：**全互联（full-mesh）内存拓扑**，反复 `merge` 直到“安静”（一轮内无任何副本状态变化）。
- 收敛判定：所有副本的**规范状态两两相同**（G-Counter/PN-Counter 比向量；G-Set/OR-Set 比 `(元素,tag)` 对集；LWW 比 `(value,timestamp)`），并额外校验**查询 value 两两相同**。

> “轮（round）”定义：一轮 = 每个副本把自己当前完整 payload 推送给其余 N−1 个副本、并 merge 所有入站状态；从开始交换计到（含）**第一个零变化轮**为止的轮数即收敛轮数。5 种 CRDT 共享同一组 merge 轮，故各 CRDT 的收敛轮数相同。

## 4. 结论要点

1. **SEC 成立——所有 CRDT、所有副本数 N、所有种子下均达到 100% 收敛。**
   G-Counter / PN-Counter / G-Set / OR-Set / LWW-Register 共 5 种类型，在 N∈{3,4,5}、每种 8 个随机种子（共 24 组 × 5 类型 = 120 个收敛检查）下，收敛正确率全部 **100%（8/8）**：每组的所有副本在全部 merge 消息投递完毕后，规范状态两两相同、查询 value 也两两相同。这与理论一致——它们的 `merge` 都是其状态半格的最小上界（逐分量 max / 集合并 / 时间戳覆盖），满足交换·幂等·结合，故必然收敛到全局 LUB，SEC 成立。

2. **收敛轮数：全互联广播下为常数 ~2 轮，不随 N 增加（O(1)，远少于 O(N)）。**
   模型 A（任务指定的全互联交换）平均收敛轮数在 N=3/4/5 下均为 **2.00**：第 1 轮每个副本即可收到其余全部副本的初始状态并 merge 到全局 LUB（全互联图直径=1），第 2 轮无变化即“安静”。因此 **O(1) ≤ O(N)**，完全符合“~O(N) 或更少”的预期，且因 full-mesh 直径为 1 而更紧。

3. **带宽受限时轮数才随 N 增加，但仍 ≤ O(N)。**
   模型 B（每轮每副本仅随机交换 1 个对端）平均轮数 2.50 → 3.50 → 3.88，随 N **亚线性增长**（约 O(log N) 流言传播量级），同样 100% 收敛。说明“轮数是否随 N 增加”取决于每轮交换的度数：纯全互联广播是常数轮；受限流言则 ~O(log N) 轮，二者均不超过 O(N)。

**一句话结论：** 在全互联内存拓扑下，5 种 CvRDT 的 merge（join 半格）保证了 100% 强最终一致性；同步全互联广播下收敛轮数恒为 ~2（不随 N 增加，O(1)），带宽受限流言下随 N 亚线性增长（≤ O(N)）。
