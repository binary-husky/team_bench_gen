# JADE：为什么对 `F ≤ 0` 必须 regenerate 而不能 truncate（夹为 0）

> 任务：纯逻辑推理（不做代码实验），从 JADE 算法语义出发，回答"如果把负的 `F_i` 简单夹为 0（`F_i = max(0, F_i_raw)`），相比论文公式 (10) 的'负数重新生成'规则，会产生什么本质不同的失败模式"。
>
> 参照来源：`jade_paper_original.pdf`（§IV.B，公式 4/7/8/10/11/12，表 I 伪代码第 20–23 行）。两个参考实现中，`pypop7_jade.py` 忠实复现了论文规则（`while f_mu[k] <= 0.0: 重采样`，再 `if f_mu[k] > 1.0: 夹为 1`，见 L122–125；CR 直接 `np.minimum(np.maximum(p_cr,0),1)` 截断，见 L135）；`turboJADE.py` 则偏离论文、对 F 也用了 `clip(0,1,…)`（L75），恰好实现了本题所问的"错误截断"规则，可作为反面例证。

---

## 0. 前提：两条规则到底差在哪

- 论文对 **CR**（公式 8，高斯 `randn(μ_CR, 0.1)`）：超出 `[0,1]` 直接**截断到边界**。
- 论文对 **F**（公式 10，柯西 `randc(μ_F, 0.1)`）：`F ≥ 1` 截断为 1；`F ≤ 0` **重新生成**（resample，直到为正），**不**夹为 0。

也就是说，论文在 F 的左尾上刻意回避了"夹为 0"。下面证明这个回避是必须的。

---

## 1. 代入 `F_i = 0`：变异向量退化为父代，trial 退化为父代的完全克隆

JADE 带归档的变异（公式 7）：

```
v_{i,g} = x_{i,g} + F_i·(x_{pbest,g} − x_{i,g}) + F_i·(x_{r1,g} − x̃_{r2,g})
```

其中 `x̃_{r2,g}` 取自种群与归档的并集 `P ∪ A`。代入 `F_i = 0`：

```
v_{i,g} = x_{i,g} + 0 + 0 = x_{i,g}
```

即 **变异向量与父代逐维相等**（与 pbest、r1、r2 选谁都无关——被 F_i=0 整体归零）。

再看公式 (4) 的二项交叉，对每一维 `j`：

```
u_{i,j,g} = v_{i,j,g}          若 rand_j ≤ CR_i  或  j == j_rand
u_{i,j,g} = x_{i,j,g}          否则
```

`j_rand` 是强制从 `v` 取值的那一维，保证 trial 至少在一维上等于变异向量。但因为 `v_{i,g} = x_{i,g}`（**所有**维都相等），无论 `rand_j ≤ CR_i` 是否成立、无论 `j_rand` 取哪一维，都有 `v_{i,j,g} = x_{i,j,g}`，于是

```
u_{i,j,g} = x_{i,j,g}   ∀j   ⇒   u_{i,g} = x_{i,g}
```

**结论 1**：`F_i = 0` 使 trial vector `u_{i,g}` 成为父代 `x_{i,g}` 的**完全克隆**（所有 D 维）。`j_rand` 本应"保住至少一维来自 v"，但在 `F=0` 下 `v=x`，`j_rand` 被彻底中和，救不回来。

---

## 2. 大量 `F_i = 0` 同时发生时的连锁反应（表 I 第 20–23 行）

JADE 是最小化问题。选择步（表 I 第 20 行附近）的语义是：若 `f(u_{i,g}) < f(x_{i,g})`（严格更优）则接受 trial，**同时**：

- `x_{i,g+1} = u_{i,g}`（替换父代）；
- 把**旧父代** `x_{i,g}` 作为劣解压入归档 `A`（第 22 行 `A ∪ {x_{i,g}}`）；
- 把 `F_i` 压入成功 F 集合 `S_F`，`CR_i` 压入 `S_CR`。

由第 1 节，`F_i = 0 ⇒ u_{i,g} = x_{i,g} ⇒ f(u_{i,g}) = f(x_{i,g})`，**严格不等式不成立** ⇒ 不接受。于是对每一个 `F_i = 0` 的个体：

- **归档 `A` 不增长**：没有父代被替换，自然没有劣解可入档；
- **`S_F` 不被填充**（这个 `F_i=0` 不会进入成功 F 集合）；
- **`S_CR` 不被填充**（这个 `CR_i` 也不进入成功 CR 集合）。

当一个世代里**大量**个体 `F_i = 0`（极端情形：整代全部 `F_i = 0`）时，连锁反应为：

1. **整代几乎/完全没有成功试验** ⇒ 这一世代 `S_F = ∅`、`S_CR = ∅`、`A` 无新增。
2. 公式 (11)/(12) 的更新被各自的 `if len(S_·) != 0` 守卫跳过 ⇒ **`μ_F` 与 `μ_CR` 本代不更新**（冻结在上一个值）。
3. `μ_F` 被冻结 → 下一代仍从同一个（偏小的）`μ_F` 处采柯西 → 左尾 `P(F_raw ≤ 0)` 依旧很大 → 下一代又是大量 `F_i = 0` → 再冻结 …… **正反馈**，形成一个**吸收性停滞吸引子**。

这里的关键是 `μ_F` 的漂移方向：公式 (12) 用的是 **Lehmer 均** `ΣF²/ΣF`，它 ≤ 算术均值，且对小成功 F 更敏感，JADE 本就有把 `μ_F` 往下拉（过早收敛）的倾向。`μ_F` 越小，柯西左尾 ≤0 的质量越大（`μ_F=0.5,scale=0.1` 时约 6%；`μ_F→0` 时趋近 50%）。**在截断规则下这是一条单向下行通道**：`μ_F↓ → 零-F 比例↑ → 成功数↓ → S_F 空/瘦 → μ_F 冻结或被更小的成功 F 继续下拉 → μ_F↓ …`，一旦 `μ_F` 跌到"整代多数为 0"的区间，算法就**卡死**：每代大多生成克隆，不搜索、不入档、不更新参数。归档 `A` 也因长期没有替换而停止流入（其作为 `r2` 差异向量、维持搜索方向多样性的作用随之退化）。

> 反观 `turboJADE.py` 的 `clip(0,1,…)` 正是落入了这个陷阱——它把左尾统统钉在 0，制造克隆，是论文要规避的写法。`pypop7_jade.py` 的 `while f_mu[k]<=0: 重采样` 则规避了它。

**结论 2**：截断规则下，`F=0` 个体是"白消耗一次变异+交叉+评估"的克隆——既不推进搜索，也不给 `S_F/S_CR/A` 任何信号；当其在世代中成规模出现时，`μ_F/μ_CR` 更新被跳过、归档停止增长，且与"`μ_F` 下行 → 零-F 增多"形成正反馈，最终陷入吸收性停滞。

---

## 3. 与 CR 对比：为什么 CR 截断到 `[0,1]` 无副作用，而 F 截断就不是

关键在公式 (4) 的 `j_rand`。

**`CR_i = 0` 的情形**（假设 `F_i > 0`，所以 `v_{i,g} ≠ x_{i,g}`）：交叉条件 `rand_j ≤ 0` 几乎必然不成立（`rand∈[0,1)`，取 0 是零测事件），于是除 `j_rand` 那一维外，trial 都取父代值；但 `j_rand` 那一维**强制取自 `v`**：

```
u_{i,g} = x_{i,g}，但第 j_rand 维换成 v_{i,j_rand,g}   ⇒   u_{i,g} ≠ x_{i,g}
```

即 `CR=0` 的 trial 是一个**单点（单维）变异体**，它和父代不同，`f(u)` 与 `f(x)` 一般不相等 ⇒ 仍可能严格更优 ⇒ 仍可被选择接受 ⇒ `CR_i=0` **仍可进入 `S_CR`** ⇒ `μ_CR` 仍被正常更新。所以 `CR=0` 是一个**合法、可被选择、可喂回自适应**的搜索动作，截断产生 0 没有病态。

**`F_i = 0` 的情形**（第 1 节）：`v=x` 使**所有 D 维**都退化，`j_rand` 也被中和 ⇒ `u=x` 的**完全克隆** ⇒ `f(u)=f(x)` 严格不更优 ⇒ 永不被接受 ⇒ 既不入档也不喂 `S_F`。这是**全 D 维**的坍缩，与 `CR=0` 的"只坍缩 D−1 维、`j_rand` 保住 1 维"截然不同。

**结论 3（不对称的根源）**：`j_rand` 为 CR 留了一条"至少一维来自 v"的活路，所以 `CR=0` 仍是真实可选择的 trial，截断无副作用；而 `F=0` 让 `v=x`，`j_rand` 也救不了，trial 沦为完全克隆、必然落选——所以对 F 用同样的截断规则会引入 CR 那条规则里根本不存在的失败模式。

---

## 4. 为什么 regenerate 能避免这个失败模式

regenerate（`F_raw ≤ 0` 时重采样至正）的本质，是把柯西的**左尾整体重映射为正样本**，等价于在 `(0,1]` 上采样——**永远不产生 `F=0`**，因而：

- 每个个体都得到**严格正**的 `F_i` ⇒ `v_{i,g} ≠ x_{i,g}`（除零测情形）⇒ trial 是真实变异体，不是克隆；
- 真实变异体有真实成败 ⇒ `S_F` 始终可能被填充 ⇒ `μ_F` 始终被更新，**没有"整代空 S_F → 冻结"的吸收态**；
- 即便 `μ_F` 漂到很小，左尾重采样仍给出**正**的 F（实质是半柯西），这些正的成功 F 通过 Lehmer 均对 `μ_F` 提供**向上拉力**，形成自校正，阻止 `μ_F` 坍缩到 0-吸引子；
- 父代持续被真实改进替换 ⇒ 归档 `A` 持续流入 ⇒ 差异向量多样性得以维持。

也就是说，regenerate 把"质量堆积在 0"这件事从根上消除了，让变异、选择、归档增长、参数自适应这四条链路始终保持"活的"；而截断恰恰把它们卡死。

---

## 5. 一句话结论

**JADE 必须对 `F ≤ 0` 做 regenerate 而不能 truncate，因为 `F=0` 会让 `v_i = x_i`、经公式 (4) 后 `u_i = x_i` 成为完全克隆（连 `j_rand` 也被中和），这种克隆在最小化的严格选择下永不被接受，因而既不入归档 `A`、也不喂 `S_F/S_CR`，一旦随 `μ_F` 下行而规模化出现就会冻结 `μ_F/μ_CR` 更新、停滞归档，形成自我强化的吸收性停滞；而 `CR=0` 因 `j_rand` 保住一维仍是合法可选择的 trial，截断无此副作用——所以 F 必须重采样以彻底消除"质量堆积在 0"，CR 则可以安全地截断到 `[0,1]`。**
