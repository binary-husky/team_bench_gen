# HyperLogLog: 估计相对误差是否随真实基数 n 变化

## 1. 设置

复现 FlFuGaMe07 的 HyperLogLog，完整实现 §4 的估计流程：

- **原始估计** `E = α_m · m² / Z`，`Z = Σ_j 2^{-M[j]}`，`α_m = 0.7213 / (1 + 1.079/m)`（m≥128）。
- **小范围修正（linear counting）**：当 `E ≤ 5/2·m` 且有空寄存器 `V≠0` 时，`E* = m·ln(m/V)`。
- **中范围**：`5/2·m < E ≤ 2³²/30`，`E* = E`。
- **大范围修正**：`E > 2³²/30`，`E* = -2³²·ln(1 - E/2³²)`。

实现细节：精度 `p=14` → `m=2^14=16384` 寄存器；64-bit 哈希用 `mmh3.hash64`（unsigned，取低 64 位）；寄存器索引取哈希高 p 位，`ρ` 取剩余 `64-p` 位窗口中最左 1 的位置。数据流为合成流：每个 item 是字符串 `"seed:i"`（i=0..n-1），真实基数 `n` 完全可控、无需去重。`n` 网格 `{1e3,5e3,1e4,5e4,1e5,5e5,1e6}`，每个 n 用 **20 个不同随机种子** 独立重复。仅 CPU，单轮约 13 秒。

**正确性校验**：向量化实现与一个纯 Python 暴力参考实现（逐项计算 ρ 与 max）在多组数据上逐位一致（相对差 0.0%），确认向量化 `ρ`/scatter-max 无误。

理论标准误差：`SE_theory = 1.04/√m = 1.04/√16384 ≈ 0.812%`。

## 3. 结论

**相对误差在中段基数范围内大致与 n 无关（常数级相对精度），且经验误差与理论 SE ≈ 1.04/√m ≈ 0.81% 同量级。**

具体地：

1. **中段到大基数（n = 1e4 … 1e6）**：平均绝对相对误差在 0.50%–0.98% 之间波动，带符号标准差在 0.54%–1.08% 之间，没有随 n 单调上升或下降的趋势——这与 HLL “固定 m 下相对误差 ~ 常数（仅依赖 m）”的理论一致。带符号偏差接近 0（最大 +0.83%，多数 <0.3%），说明估计量无系统性偏置，波动纯由随机方差驱动。

2. **经验量级与理论 SE 吻合**：中段带符号标准差（0.5%–1.1%）与理论 0.81% 同量级；|rel| 均值也落在 0.5%–1% 区间，与 0.81% 一致（少量偏离来自仅 20 个种子的抽样波动，以及某些 n 下个别种子出现 ~2% 的尾部样本）。

3. **小基数（n = 1e3, 5e3）**：误差反而更小（~0.39%–0.40%），低于理论 0.81%。原因是此时处于小范围修正区（`E ≤ 2.5·m = 40960`），linear-counting 估计 `m·ln(m/V)` 比原始估计更准、方差更小——这正是 §4 小范围修正的设计目的。这说明算法在各区间都启用了对应修正，估计量在整个 n 网格上可用。

4. **大范围修正**在本实验未触发：64-bit 哈希空间下，n ≤ 1e6 远小于 `2³²/30 ≈ 1.4×10⁸`，始终落在中范围（原始估计直接使用）；代码仍保留了该分支以保持流程完整。

综上，在固定精度 p=14 下，HLL 的相对误差在可用基数中段呈常数级、与 n 无关，量级与理论 `1.04/√m ≈ 0.81%` 一致；仅在很小基数处因 linear-counting 修正而更优。
