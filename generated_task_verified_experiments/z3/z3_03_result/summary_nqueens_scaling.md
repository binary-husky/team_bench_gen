# N-Queens 规模扩展性：z3 求解时间随 N 的变化

## 1. 实验设置

- **求解器**：`z3-solver` (Z3 v4.8.12)，Python API。
- **问题**：N-皇后，**找第一个解**（`Solver.check()` 返回 `sat` 即停，不枚举全部解）。
- **编码方式（固定）**：每行一个 `Int` 列号变量 `c_r`，域 `1..N`。
  - 列互不相同：`Distinct(c_0 … c_{N-1})`
  - 对角线互不相同：令 `d1_r = r - c_r`（主对角线方向），`d2_r = r + c_r`（反对角线方向），分别施加 `Distinct(d1_0 … d1_{N-1})` 与 `Distinct(d2_0 … d2_{N-1})`。
  - 即经典的“行列+两条对角线全部互异”整数编码，未使用布尔位编码。
- **求解器设置（固定）**：
  - `s.set("timeout", 120000)`（120 秒上限）
  - `s.set("random_seed", 1)`（固定随机种子以提升可复现性）
  - 其余设置保持 Z3 默认。
- **唯一自变量**：`N ∈ {8, 10, 12, 15, 20}`。
- **计时**：每个 N 重复求解 5 次，取**中位数**（同一进程内每次新建 Solver 重建模型），报告 wall-clock 秒。环境为 CPU-only。

## 2. 结果

| N  | 状态 | 中位求解时间 (s) | 最小 (s) | 最大 (s) |
|----|------|------------------|----------|----------|
| 8  | sat  | 0.0076           | 0.0063   | 0.0179   |
| 10 | sat  | 0.0103           | 0.0091   | 0.0209   |
| 12 | sat  | 0.0664           | 0.0521   | 0.1341   |
| 15 | sat  | 0.0985           | 0.0436   | 0.2316   |
| 20 | sat  | 0.2324           | 0.1975   | 0.3491   |

全部实例在 120 秒超时上限内求得解（均为 `sat`），未触发超时。

各 N 的一个可行解（列号，1-indexed，按行 0..N-1）：

- N=8 : `[4, 2, 8, 6, 1, 3, 5, 7]`
- N=10: `[4, 7, 9, 2, 6, 1, 3, 5, 8, 10]`
- N=12: `[1, 6, 11, 7, 10, 3, 5, 2, 4, 9, 12, 8]`
- N=15: `[8, 11, 4, 6, 15, 9, 1, 3, 14, 2, 7, 10, 13, 5, 12]`
- N=20: `[11, 4, 6, 1, 9, 13, 18, 2, 17, 8, 16, 12, 19, 3, 14, 7, 5, 10, 20, 15]`

## 3. 求解时间随 N 的变化趋势

- **整体单调上升**：从中位数看，求解时间随 N 增大而增大（0.008 s → 0.23 s），约 30 倍增长跨度。
- **量级整体很小**：即使在 N=20，中位时间也仅约 0.23 s，远低于 120 s 上限。说明在“只求第一个解”的设定下，整数 + `Distinct` 编码对 20×20 规模仍非常轻松；z3 的 CDCL 搜索配合 `Distinct` 全局约束传播能快速定位到一个可行布局。
- **增长形态（亚线性—近似多项式）**：
  - N=8→10：约 1.35×（0.0076→0.0103）
  - N=10→12：约 6.4×（0.0103→0.0664）——这一步跳变最明显，进入约束传播开始需要回溯的规模。
  - N=12→15：约 1.48×（0.0664→0.0985）
  - N=15→20：约 2.36×（0.0985→0.2324）
  - 从 N=12 起到 N=20，时间随 N 的增长大致呈**略超线性的多项式关系**（约 ∝ N^1.5–N^2 量级），而非指数爆炸。这与“找首个解”场景下解密度高、搜索树被及早剪枝一致——N-皇后解的数量随 N 增长极快（解空间稀疏但仍密集到首解易达），所以 z3 不需要深度回溯。
- **方差随 N 增大**：最大/最小时间之比从 N=8 的 ~2.8× 扩大到 N=20 的 ~1.8×（N=15 一度达 ~5.3×）。较大 N 下单次运行受 OS 调度/缓存影响更明显，但中位数稳定可靠。

## 4. 结论

在固定整数编码（每行一个 `Int` 列号 + 列/双对角线 `Distinct`）、固定 `random_seed=1`、`timeout=120s` 的设置下，z3 求解 N-皇后**首个解**的时间随 N 从 8 增至 20 单调上升，从约 **0.008 s（N=8）增至约 0.23 s（N=20）**，呈**多项式级（近似 N^1.5–N^2）的温和增长**，远未触及指数爆炸或超时上限。增长主要发生在 N=10→12 的拐点（约束传播开始触发回溯），其后增速平稳。在该规模区间内，z3 对 N-皇后的规模扩展性良好，瓶颈不在求解器而在编码与首解密度。

## 5. 复现

脚本（同目录）：
- `nqueens_z3.py`：单次运行，打印每个 N 的状态/时间/解。
- `nqueens_z3_repeat.py`：每个 N 重复 5 次取中位数（本报告数据来源）。

运行：`python3 nqueens_z3_repeat.py`
