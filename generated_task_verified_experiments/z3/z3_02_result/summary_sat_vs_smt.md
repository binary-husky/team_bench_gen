# 纯布尔 SAT 编码 vs 原生 SMT 理论编码 —— 规模与时间对比

## 1. 实验设置（固定）

- **z3 版本**：`z3-solver` 4.8.12（pip 安装，CPU-only，Python API 调用）。
- **约束谜题实例（固定）**：3×3 幻方（magic square）。
  - 9 个整数变量 `x0..x8`，排列成 3×3 网格。
  - 每个变量取值域 `{1,2,...,9}`。
  - **全互不等**（all-different，即 1..9 的一个排列）。
  - **8 条线性求和约束**，每条 `== 15`：3 行 + 3 列 + 2 条主对角线。
  - 该实例可满足（有解），用于公平对照。
- **自变量**：编码方式（两种），其余全部固定。
  - **(a) 纯布尔 SAT 编码**（direct + table 编码）：
    - 每个 `(变量, 取值)` 二元组对应一个 `z3.Bool`（direct encoding）。
    - 「每变量恰好一个取值」：ALO（大析取）+ AMO（两两互斥合取范式）。
    - 「全互不等」：对每对变量、每个取值，加 `¬(xi=v ∧ xj=v)` 二元子句。
    - 「`sum==15`」：**表编码（table encoding）**——枚举所有满足 `a+b+c=15` 的有序取值三元组（共 **61** 个/约束），每个三元组引入一个 Tseitin 辅助布尔变量 `t`，加 `t ⇒ (b[i1][a]∧b[i2][b]∧b[i3][c])`（3 子句）+ 每约束一条「至少一个元组成立」的大析取子句。
  - **(b) 原生 SMT 编码**：直接用 `z3.Int` 变量 + `And`/`Or`/算术/关系运算；全互不等用 `z3.Distinct`；求和用 `x[i]+x[j]+x[k] == 15`。
- **计时**：每个编码各跑 7 次（`time.perf_counter()`，仅计 `solver.check()`），取 min/median/max；两次求解之间重建求解器以避免缓存。代码见 `experiment.py`；规模放大对照见 `scale_test.py`。

## 2. 主要结果（固定实例：3×3 幻方）

| 指标 | (a) 纯布尔 SAT | (b) 原生 SMT |
|---|---|---|
| 变量数 | 81 direct + 488 table 辅助 = **569 布尔变量** | **9 个 Int 变量** |
| 约束数 | **2129 条子句**（其中 table 部分 1472 条） | **18 条断言**（9 域界 + 1 `Distinct` + 8 求和） |
| 求解结果 | sat | sat |
| 求得解 | `[2,7,6 / 9,5,1 / 4,3,8]`（合法幻方） | `[2,7,6 / 9,5,1 / 4,3,8]`（同解，合法幻方） |
| 求解时间 min | 7.8 ms | 4.1 ms |
| 求解时间 median | **8.9 ms** | **5.8 ms** |
| 求解时间 max | 21.5 ms | 7.0 ms |

两种编码都判定为 SAT 且还原出同一组合法幻方（行/列/对角线和均为 15，且为 1..9 的排列），验证了两套编码语义一致。

## 3. 规模放大对照（4×4 幻方，作为趋势佐证）

固定实例之外，另跑 4×4 幻方（16 变量、域 1..16、10 条 `sum==34` + 全互不等）以放大差异、看清趋势：

| 指标 | (a) 纯布尔 SAT | (b) 原生 SMT |
|---|---|---|
| 变量数 | **27 616** 布尔变量 | **16** Int 变量 |
| 约束数 | **113 306** 条子句 | **27** 条断言 |
| 求解时间 median | **~2 733 ms** | **~215 ms** |
| 求得解 | 合法 4×4 幻方 | 合法 4×4 幻方 |

## 4. 结论

1. **规模差异巨大且随问题规模膨胀。** 同一实例，纯布尔 SAT 编码的变量/约束数远大于原生 SMT：3×3 时即 569 vs 9 变量、2129 vs 18 子句；放大到 4×4 时差距爆炸式扩大（27 616 vs 16 变量、113 306 vs 27 子句）。原因是 SAT 编码必须把「整数取值」显式展开为 `(变量,取值)` 笛卡尔积个布尔变量，并把每条算术/全不等约束拆解成大量基础命题子句（direct 编码的 AMO/互斥是 O(N²·V)、table 编码随约束元数与取值域指数增长）；而原生 SMT 只需声明少量 Int 变量与高层算术/`Distinct` 断言，理论求解器在内部按需展开。

2. **求解时间 SMT 更快，且随规模扩大优势更明显。** 3×3 上两者都在毫秒级（SAT ~8.9ms / SMT ~5.8ms，SAT 约慢 1.5×，差异尚在测量噪声附近）；4×4 上 SAT ~2.73s 而 SMT ~0.21s，SMT 约快 12.7×。这正符合材料（de Moura & Bjørner, *Z3: An Efficient SMT Solver*, 2008）所述：Z3 用 DPLL 式 SAT 求解器与算术/未解释函数等理论求解器（theory solvers）协同——对原生 SMT 输入，高层约束由专用理论求解器增量传播、按需消解，避免一开始就把整数量化为海量命题。纯布尔输入则等于绕过理论求解器、把全部组合负担直接压给底层 SAT，规模代价由编码者预付。

3. **可读性与建模成本。** 原生 SMT 编码 9 个 Int 变量 + 18 行断言即可，贴近问题本身；SAT 编码需手工处理「恰好一个」「互斥」「表枚举/Tseitin」等机制，代码量与出错面都更大。

4. **何时仍会考虑纯布尔编码。** 当问题天然就是命题结构（如电路可满足性、纯 0/1 约束）、或目标求解器只接受 SAT/CNF 输入、或需要对布尔子句做精细控制（如特定启发表）时，纯布尔编码仍有意义；但对「小范围整数 + 线性/全不等」这类约束谜题，原生 SMT 编码在**规模、速度、可读性**三方面均占优，是首选。

**一句话总结**：在固定的小规模整数约束谜题（幻方）上，纯布尔 SAT 编码相对原生 SMT 编码的变量数/约束数大 1–3 个数量级且随规模快速膨胀，求解时间更慢（3×3 约 1.5×、4×4 约 12.7×），原生 SMT 编码在规模、时间、可读性上全面胜出。
