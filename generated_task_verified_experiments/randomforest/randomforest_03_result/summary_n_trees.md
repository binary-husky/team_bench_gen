# 树的数量（n_estimators）对随机森林误差收敛的影响

## 实验设置

- **数据集**：`sklearn.datasets.load_digits`（手写数字，64 特征，10 类，1797 个样本）。
- **划分**：70/30 训练/测试划分（`train_test_split(test_size=0.30, random_state=42, stratify=y)`，固定种子 + 分层）。
- **模型**：`RandomForestClassifier(oob_score=True, random_state=0)`，其余参数取默认值（`max_features='sqrt'`、`bootstrap=True`、未限深）。`random_state=0` 固定整片森林的随机性。
- **自变量**：仅 `n_estimators ∈ {10, 50, 100, 200, 500, 1000}`。
- **度量**：
  - OOB 误差 = `1 - oob_score_`（袋外估计的泛化误差，仅用训练集）。
  - 留出测试误差 = `1 - score(X_te, y_te)`（30% 独立测试集）。
- 运行环境：CPU，scikit-learn 1.2.1。

## 结果

| n_estimators | OOB 误差 | 测试误差 |
|-------------:|---------:|---------:|
| 10           | 0.1321   | 0.0667   |
| 50           | 0.0398   | 0.0370   |
| 100          | 0.0294   | 0.0352   |
| 200          | 0.0239   | 0.0315   |
| 500          | 0.0239   | 0.0315   |
| 1000         | 0.0231   | 0.0278   |

（OOB 与测试误差均随 n_estimators 上升而下降并趋于平稳。）

## 结论与分析

### 1. 误差随树数增加单调下降并收敛
OOB 误差从 10 棵树时的 0.132 快速下降到 50 棵的 0.040、100 棵的 0.029，200 棵降到 0.024，之后基本走平（500 棵仍为 0.0239，1000 棵仅微降到 0.0231）。测试误差走势一致：0.0667 → 0.0370 → 0.0352 → 0.0315，并在 500/1000 处趋于 0.028–0.031。

这与 Breiman (2001) 随机森林的理论预期一致：森林通过 **bagging + 特征子采样** 降低各树之间的相关性，随着树数增加，平均后的方差持续下降；但树与树间并非完全独立（共享相同分布的训练数据与分裂规则），因此误差存在下限——增加树数只会让估计逼近该下限，而不会无止境降低。本实验中 **200 棵之后 OOB 误差几乎不再改善**，说明在该数据集/参数下，约 200 棵树即已触及收敛平台期，再增到 500、1000 收益边际很小。

### 2. OOB 与测试误差吻合，且 OOB 是可靠的收敛监测量
从 50 棵树起，OOB 误差与独立测试误差数值上非常接近（同一量级、走势一致），说明袋外估计本身就是对泛化误差的良好代理，可在不碰测试集的前提下监测收敛。这印证了 Breiman 提出的 OOB 估计的有效性。

### 3. 树数过少时的两个问题
- **n=10 时 OOB 误差（0.132）异常偏高、甚至远大于测试误差（0.067）**：这并非真实泛化能力这么差，而是因为树数太少导致部分样本从未进入任何一棵树的袋外集合（scikit-learn 同时给出警告 `Some inputs do not have OOB scores`）。这些样本的 OOB 预测缺失，使 `oob_score_` 在少量样本上退化为训练误差的反向量，估计不稳定且偏悲观。因此 **OOB 估计在树数很少时不可靠**，至少需要几十棵树才有意义。
- n=10 时即便测试误差也已 0.067，明显高于收敛后的 ~0.028，说明少量树下方差尚未充分平均掉，模型尚未收敛。

### 4. 稳定性 / 平台期
- 200 → 500：OOB 与测试误差**完全不变**（0.0239 / 0.0315），表明误差曲线已进入平台区，增加树数不改变结果（仅是相同随机源下的更多采样平均）。
- 500 → 1000：OOB 仅再降 0.0008、测试仅再降 0.0037，改善可忽略。
- 即在收敛后，模型对 n_estimators 的进一步变化**不敏感**（稳定），这与“随机森林不会因树过多而过拟合”的特性一致——增加树数只会降低方差而不会增加偏差，不存在像提升方法那样的过拟合风险。

### 5. 实践建议
- 在本设置（digits 数据集）下，**n_estimators ≈ 200 即可达到收敛**，性价比最高；继续加到 1000 几乎不带来误差改善，只增加训练/内存成本。
- 用 **OOB 误差**监测收敛即可，无需预留测试集；但 OOB 估计需要足够多的树（≥~50）才可信，n=10 时的 OOB 值应视为不可靠而被舍弃或单独标注。
- 选定树数时，应观察误差曲线进入平台区的拐点，而非盲目取最大值。
