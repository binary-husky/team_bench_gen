# 为什么给随机森林加树不会过拟合

> 任务：仅逻辑推理——RF 加树不过拟合、误差收敛到极限不回升；给出 B 棵相关树平均的方差公式；为何偏差不变；与 boosting 的本质不同。
> 依据 Breiman, *Random Forests* (2001)。

## 方差公式与极限

RF 预测 = B 棵树的平均/多数投票。设每棵树方差 `σ²`、两两相关系数 `ρ`（同分布、同 mtry 训练 ⇒ 正相关）。B 棵树平均后方差：

> `Var(平均) = σ²·[ (1−ρ)/B + ρ ]`

- `(1−ρ)/B`：树间不相关部分，随 B 增大**消失**。
- `ρ`：相关地板，B 再多也消不掉。
- 故 B→∞ 时方差 → **σ²ρ**（地板），单调下降、不回升。

## 偏差为何不变

每棵树的偏差由分裂策略/mtry 决定，与 B 无关；平均不改变偏差（`E[平均]=E[单树]`）。故加树只降方差、不抬偏差，泛化误差 = 偏差² + 方差 + 噪声 ⇒ 单调降到极限、**永不回升**（不过拟合）。

## 与 boosting 的本质不同

Boosting 是**加法型**集成：每棵新树拟合当前集成的**残差**，`F_B = F_{B-1} + η·h_B`。随 B 增大，模型容量与训练拟合**持续增强**（加法累加容量）⇒ 偏差持续降但方差/容量上升，B 过大即过拟合。RF 是**平均型**（`F_B = (1/B)Σh_i`），加树只稀释方差、不增容量 ⇒ 加树只赚不亏（直到方差地板）。

## 一句话总结

RF 平均 B 棵相关树 ⇒ 方差 `σ²[(1−ρ)/B+ρ]`，B↑ ⇒ 方差降到地板 `σ²ρ`、偏差不变 ⇒ 误差单调趋极限不回升（不过拟合）；boosting 加法型累加容量、B 过大过拟合——「平均」vs「加法」是根本差异。
