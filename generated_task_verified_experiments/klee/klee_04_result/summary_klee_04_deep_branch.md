# KLEE-04: 符号执行 vs 随机 Fuzzing —— 到达深层 / 稀有目标分支能力对比

## 实验设置

- 框架：轻量 Python + `z3-solver` 符号执行 / concolic 框架（未安装 KLEE 本体，仅 CPU、仅 pip）。
- 目标函数 `g(x)`，深层目标分支为累积多 guard 的“针”条件：
  ```
  if x > 10 and x % 7 == 3 and x * 2 < 100:  <TARGET>
  ```
- 输入域：32 位非负整数 `[0, 2^32)`，共 4294967296 个取值。
- 满足条件的真实输入 = [17, 24, 31, 38, 45]，共 5 个；真实命中概率 = 1.164153e-09（5/4294967296），占比极小。
- 随机 fuzzing 重复种子：[1, 2, 3]（每个配置 ≥3 个不同种子）。

## 1. 结果对比表

| 方法 | 是否找到到达 TARGET 的输入 | 关键指标 |
|---|---|---|
| 符号执行 (z3 解 PC) | 找到 | z3 求解次数 = 1；模型 x = 17 |
| 随机 fuzzing (N=1000) | 未找到 | 命中数 = 0 / 3000；命中率 = 0.000000e+00 |
| 随机 fuzzing (N=10000) | 未找到 | 命中数 = 0 / 30000；命中率 = 0.000000e+00 |
| 随机 fuzzing (N=100000) | 未找到 | 命中数 = 0 / 300000；命中率 = 0.000000e+00 |

## 2. 结论要点

1. **符号执行确定性找到**到达 TARGET 的输入：沿路径把 `x>10`、`x%7==3`、`x*2<100` 三个 guard 累积进路径条件 PC，**只需 1 次 z3 求解**即可判定 PC 可满足，模型直接给出满足条件的输入 `x=17`（经验证确为 `[17, 24, 31, 38, 45]` 之一）。约束求解把“在海量输入里找针”的问题归约为可满足性判定，命中是确定性的。

2. **随机 fuzzing 在合理 N 下命中率极低甚至为 0**：在 1e3 / 1e4 / 1e5 三个量级、每个量级 3 个种子下，总命中数均为 0（命中率 ≈ 0.000000e+00），与真实命中概率 1.164153e-09 一致——满足累积多 guard 条件的输入测度极小，纯随机采样在可承受的 N 内几乎不可能命中。

3. **对比验证**：符号执行的约束求解能力可定向“反推”出满足深层累积约束的输入，成本是少量（此处仅 1 次）z3 求解；而随机 fuzzing 受限于概率，对测度极小的稀有目标分支本质上无能为力——这正是符号执行（如 KLEE）相对于盲目 fuzzing 的核心优势：以约束求解换取对深层 / 稀有路径的可达性。

---
参考材料：Cadar, Dunbar, Engler (2008), *KLEE: Unassisted and Automatic Generation of High-Coverage Tests for Complex Systems Programs*（见 `klee_material/`）。KLEE 的核心即符号执行 + 约束求解以驱动覆盖深层分支，本实验以最小复现印证了其相对随机 fuzzing 的优势。
