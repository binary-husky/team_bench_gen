# 符号执行 vs 随机测试：含难达分支小函数的分支覆盖率对比

## 1. 实验设置

- **不安装 KLEE 本体**（其 LLVM 工具链 30 分钟内装不完）。用 Python + `z3-solver` 自研一个轻量符号执行框架，复现 KLEE 的核心方法：把输入当作 32 位符号位向量（`z3.BitVec`），沿执行路径在每个分支处把**所取条件**累积为路径约束，用 z3 判断每条路径约束的可满足性，为每条可行路径解出一个具体输入（模型），并**系统枚举所有可行路径**（DFS）。

- **目标函数** `f(x)`（32 位有符号整数）：

  ```python
  if x == 12345: return "rare"   # 难达分支：需输入恰为 magic 值
  if x < 0:     return "neg"
  if x == 0:    return "zero"
  return "pos"
  ```

- 分支边共 **6 条** = 3 个 if × (true/false)：`E1t(x==12345)`、`E1f`、`E2t(x<0)`、`E2f`、`E3t(x==0)`、`E3f`。难达分支 = `E1t`（magic=12345）。

- (a) 符号执行：枚举所有可行路径，每条发一个测试输入，统计分支覆盖率。结果由方法本身决定（确定性），仍用 3 个内部种子重复以确认稳定。

- (b) 随机测试：每种子 **1000** 个随机输入（取自 32 位有符号全域 [-2^31, 2^31)），种子 = [1, 2, 3, 7, 42]。

- 平台：仅 CPU；依赖：`z3-solver`（已可用，4.8.12）。


## 2. 结果表

| 方法 | 种子 | 分支覆盖率 | 覆盖边数 | 命中难达分支 (E1t=12345) |
|------|------|-----------|----------|--------------------------|
| 符号执行 (z3) | 确定性 | **100.0%** | 6/6 | **是 ✓** |
| 随机测试 | 1 | 66.7% | 4/6 | 否 ✗ |
| 随机测试 | 2 | 66.7% | 4/6 | 否 ✗ |
| 随机测试 | 3 | 66.7% | 4/6 | 否 ✗ |
| 随机测试 | 7 | 66.7% | 4/6 | 否 ✗ |
| 随机测试 | 42 | 66.7% | 4/6 | 否 ✗ |
| 随机测试 (汇总) | [1, 2, 3, 7, 42] | min 66.7% / mean 66.7% / max 66.7% | — | 否 ✗（所有种子均未命中） |

### 符号执行枚举出的可行路径

| 解出的输入 | 返回结果 | 路径约束项数 |
|-----------|---------|-------------|
| 12345 | rare | 1 |
| -2147483648 | neg | 2 |
| 0 | zero | 3 |
| 1073741824 | pos | 3 |

### 符号执行覆盖的边

`['E1f(x==12345)', 'E1t(x==12345)', 'E2f(x<0)', 'E2t(x<0)', 'E3f(x==0)', 'E3t(x==0)']` —— 含难达边 `E1t`，**6/6 = 100%**。


### 随机测试典型覆盖边（seed=1）

`['E1f(x==12345)', 'E2f(x<0)', 'E2t(x<0)', 'E3f(x==0)']` —— 缺 `E1t`（难达），通常也缺 `E3t`（x 恰为 0 概率 ~1/2³²）。


## 3. 分支覆盖率对比（柱状示意）

```
符号执行 : [##########] 100.0%   命中难达: 是
随机测试 : [######    ] 66.7% (均值) 命中难达: 否
          : [######    ] 66.7% (最好) 命中难达: 否
```

## 4. 结论要点

1. **符号执行达到完全分支覆盖**：100.0%（6/6），系统枚举出全部 4 条可行路径，并为每条解出具体输入，其中一条输入恰为 magic 值 12345，**命中难达分支** `E1t`。这正体现了 KLEE 核心方法的优势——通过路径约束求解而非盲目采样，能精确‘算出’触发难达分支的输入。

2. **随机测试在有限样本下漏掉难达分支**：在 32 位全域中每次取到 恰 12345 的概率约 1/2³² ≈ 2.3e-10，1000 次采样几乎不可能命中；5 个种子**无一命中**难达分支 `E1t`。覆盖率均值仅 66.7%（最好 66.7%），明显低于符号执行的 100%。

3. 随机测试还常漏掉 `E3t`（x 恰为 0 同样概率极低），故其覆盖的多为‘普通’分支（负/正/各 false 边）。符号执行则对这些边界分支也一并覆盖。

4. **方法学对照**：本实验复现了 KLEE 论文（Cadar et al., OSDI 2008）的核心论点——符号执行通过约束求解系统探索路径空间，能在随机测试难以触及的难达分支上取得（近）完全覆盖并自动生成触发输入；随机测试受样本量与输入域限制，对‘针尖式’难达分支基本无能为力。
