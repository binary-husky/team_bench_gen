# 符号执行：生成测试数 ≈ 可行路径数（覆盖率饱和验证）

## 1. 实验设置

**框架**：轻量级 z3 符号执行 / 树状路径枚举器（Python + `z3-solver`，**未安装 KLEE 本体**，纯 CPU / 纯 pip，< 30 分钟）。引擎解析 toy 函数的 Python AST，用 z3 `BitVec(32)` 作为符号输入；在每个 `If` 处按 True/False 双臂 fork，把分支条件累加进路径约束，用 z3 `check()` 判定每条臂是否**可行**（infeasible 即剪枝）；可行路径走到 `return` 时用 z3 `model()` 求出一个具体输入 = 一条生成的测试。这复现了 KLEE（Cadar et al., 2008）“每条可行路径恰好一个测试输入”的核心关系，但不依赖 KLEE 二进制。

**复现**：每个函数用 **3 个种子**（`1, 7, 42`）重复（种子只影响 fork 顺序，不影响完整枚举的计数结果——见下）。

**toy 函数**（路径复杂度递增）：
- `get_sign(x)`：`x<0` / `x==0` / 否则 → 3 条路径。
- `classify_triangle(a,b,c)`：非法(0) / 等边(3) / 等腰(2) / 不等边(1) → 4 条路径。
- `k_independent_ifs_k(x1..xk)`：`k` 个相互独立的 `if x_i>0`，可行路径数 = `2^k`，`k ∈ {2,3,4,5}` → 4/8/16/32 条路径。

## 2. 结果表（3 个种子取值一致；列示每个种子）

| 函数 | 生成测试数 | 可行路径数 | 分支覆盖 | 分支覆盖率 |
|---|---|---|---|---|
| `get_sign` | 3 (3/3/3) | 3 | 4/4 | **100.0%** |
| `classify_triangle` | 4 (4/4/4) | 4 | 6/6 | **100.0%** |
| `k_independent_ifs_2` | 4 | 4 (=2²) | 4/4 | **100.0%** |
| `k_independent_ifs_3` | 8 | 8 (=2³) | 6/6 | **100.0%** |
| `k_independent_ifs_4` | 16 | 16 (=2⁴) | 8/8 | **100.0%** |
| `k_independent_ifs_5` | 32 | 32 (=2⁵) | 10/10 | **100.0%** |

> 括号内为 3 个种子的逐一种子值（`seed=1/7/42`），三者完全相同：完整路径枚举是确定性的，种子扰动不改变计数——这本身就是“枚举全部可行路径”的强证据。原始逐次运行见 `experiment_results.json`，引擎见 `symexec.py`，驱动见 `run_experiment.py`。

## 3. 结论要点

1. **生成测试数 ≈ 可行路径数（每个可行路径恰好一个输入）。**
   所有 6 个函数、3 个种子下，`生成测试数 == 可行路径数` 严格成立（3=3、4=4、4/8/16/32=2^k）。引擎对每条经 z3 判定可行的路径求解出恰好一个具体模型；不可行路径在 fork 时被剪枝，因此不会产生多余或缺失的测试。生成的输入语义可验证，例如 `classify_triangle` 给出：非法三角形、等腰（`a==c`）、等边（`a==b==c`）、不等边四类各一；`k_independent_ifs` 给出全部 `2^k` 种 0/1 组合。

2. **路径爆炸：`k_independent_ifs` 的可行路径数随 `k` 按 `2^k` 增长。**
   `k=2→4, k=3→8, k=4→16, k=5→32`，精确符合 `2^k`。这是因为 `k` 个 `if` 互相独立、每个分支两侧都可行，可行路径数 = 各 `if` 分支数的乘积 = `2^k`。这是符号执行面临的指数路径爆炸的最小可复现缩影：分支数线性增长（`k` 个 `if`），而可行路径数指数增长。

3. **分支覆盖率饱和到（近乎）100%。**
   一旦所有可行路径被枚举，每个 `If` 的 True/False 两条边只要可行就都被至少一条路径覆盖，6 个函数全部达到 `100.0%`（4/4、6/6、…、10/10）。注意分母是“全部 if 边”而非“可行 if 边”：本组 toy 函数中每个 `if` 的两侧均可行，故覆盖率直接饱和到 100%。若存在恒真/恒假分支（某侧不可行），覆盖率会略低于 100%——饱和上限即“可行分支覆盖率”，这正是符号执行“枚举可行路径 ⇒ 覆盖率饱和”的体现。

4. **种子稳定性**：3 个种子的测试数 / 路径数 / 覆盖率三指标逐字一致，说明在完整枚举（非随机搜索）设定下，结论不依赖随机性；种子只改变路径被访问的先后顺序。

## 附：方法学说明
- 符号整数用 32 位 `BitVec`；toy 函数无溢出、无循环、无调用，AST 子集（算术/比较/布尔短路/`if`-`elif`-`else`/`return`）足以表达。
- “路径”定义为 `If` 节点上的 `T/F` 决策序列（按 AST 节点 id 去重）；“分支覆盖率” = 被至少一条可行路径覆盖的 `If` 边数 / 全部 `If` 边数。
- 文件：`symexec.py`（引擎）、`run_experiment.py`（驱动）、`experiment_results.json`（原始逐次结果）。
