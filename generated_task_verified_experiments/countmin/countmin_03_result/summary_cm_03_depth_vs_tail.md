# Count-Min Sketch：深度 `d` 如何压低尾部失败概率

> 自行复现 Count-Min Sketch（Cormode & Muthukrishnan, *An Improved Data Stream Summary: The Count-Min Sketch and its Applications*，定理 1），用 Python/numpy 从零实现 `d` 行两两独立（Carter–Wegman）哈希，含更新与点查询，实验验证：**固定宽度 `w` 下，点查询高估超过 `ε‖a‖₁` 的概率随深度 `d` 指数下降**。
>
> 代码：`cm_experiment.py`；原始逐种子数据：`cm_results_raw.csv`；汇总：`cm_results_summary.json`；图：`cm_depth_vs_tail.png`。

---

## 1. 实验设置（严格按 task 固定参数）

| 参数 | 取值 | 说明 |
|---|---|---|
| 宽度 `w` | **1024** | 固定 |
| `ε = e/w` | **0.002655** | 误差阈值系数 |
| 深度 `d` | {1, 2, 3, 4, 5, 8} | 网格 |
| 数据流 | 1e6 次更新，全 `+1`（cash-register / 非负） | |
| item 宇宙 | n = 1e5（ranks 1…1e5），**Zipfian s = 1.0** 逆 CDF 采样 | |
| 真实频率 `a[i]` | 由流计数得到 | |
| `‖a‖₁` | **1,000,000** | 80622 个 item 实际出现 |
| 阈值 `T = ε‖a‖₁` | **2654.57** | |
| 哈希 | `h_j(x)=((a_j·x+b_j) mod p) mod w`，`p=2³¹−1`；**每行独立抽取** `(a_j,b_j)` → 行间独立 | 两两独立 |
| 重复 | **3000 个不同哈希种子**（≥20），取 item 失败占比的均值 | |
| 单次实验耗时 | ≈27 s（CPU，64 核机但单线程 numpy） | 全程 < 30 min |

**实现要点**：CM sketch 对 `+1` 更新是**线性**的，`count[j,h(i)] = Σ_{h(i)=k} a[i]`，故每行用加权直方图 `np.bincount(bkt, weights=a)` 一次性建表（与逐条更新数学等价）；点查询 `â_i = min_j count[j,h_j(i)]`，对全部 item 一次性 gather + 逐行取 min。失败判定：`â_i > a_i + T`（严格超过）。

**理论参考（定理 1）**：`a_i ≤ â_i` 恒成立；且
`Pr[â_i > a_i + ε‖a‖₁] = Pr[∀j X_{i,j} > e·E[X]] ≤ (1/e)^d = e^{−d}`，
其中 `X_{i,j}` 为第 `j` 行的碰撞噪声，`E[X] ≤ ‖a‖₁/w`，阈值 `T = ε‖a‖₁ = e·(‖a‖₁/w) = e·E[X]`。关键：**行间独立**使总概率 = 各行概率之积；Markov 给出**每行 ≤ 1/e** 这个（很松的）上界。

## 3. 结论要点

1. **尾部失败概率确实随 `d` 指数下降。** 相邻深度的乘子高度恒定（≈0.038–0.048，拟合 0.0407），对数纵轴下经验点呈一条直线 → 经典的「每多一行把失败概率乘以一个常数因子」的指数衰减。**指数性（exponential-in-`d`）得到强验证。**

2. **但实测的逐行因子 ≈ 0.04，远小于理论的 1/e ≈ 0.368**——即衰减比 `e^{−d}` **快约一个量级/行**。原因：CM 证明里唯一用的工具是 Markov 不等式 `Pr[X > e·E[X]] ≤ 1/e`，这是一个非常松的上界。对本 Zipfian 流，单行碰撞噪声 `X` 很少超过 `e` 倍均值，实测每行越阈概率只有 ~3.9%（而非最坏 36.8%）。
   - 直观解释：只有少数重频 item（`a[k] > T=2654`，约 31 个）能与某 item 在单行碰撞并使其越阈；典型 item 单行撞上它们的概率 ≈ 31/1024 ≈ 0.030，加上中等 item 的累积约到 0.039，正是 `p₁`。

3. **与理论比较：实测处处 ≤ `e^{−d}`，符合定理 1 的上界。** task 要求「实测应与 `e^{−d}` 同量级或在其之下」——本结果**在其之下**（小 d 处差约 1 个量级，随 `d` 增大差距拉大），完全满足 `Pr[â>a+ε‖a‖₁] ≤ (1/e)^d` 的保证。d=8 处实测 0（分辨不出，< 3.3e−9），比上界 `e^{−8}=3.35e−4` 至少低 5 个量级，亦一致。

4. **行间独立的乘积结构得到直接验证。** 由于各行哈希独立，应有 `rate[d] = ∏_j Pr[第 j 行越阈] ≈ p₁^d`。实测 `rate[d]/p₁^d` 在 d=1..4 几乎恒为 1.00（1.00, 0.99, 1.02, 1.02），说明 `(1/e)^d` 这个**形式**（每行贡献一个独立因子）是精确成立的——只是 CM 把每行的真实因子 `p₁` 用其 Markov 上界 `1/e` 替换了，故上界偏松。

5. **cash-register 单侧性核实**：3000 种子 × 1e5 item 共 3e8 次点查询中，`â_i < a_i`（低估）发生 **0 次**，符合定理 1 的 `a_i ≤ â_i`（CM 误差是单侧的，只高估）。

### 一句话总结
> 固定 `w=1024`：点查询越阈（`â_i > a_i + ε‖a‖₁`）的比例随深度 `d` **干净地指数下降**，实测逐行乘子 ≈ **0.04**（拟合 `ln rate ≈ −3.20·d`），**远快于**理论 Markov 上界给出的 `1/e≈0.368`/行；实测值在每个 `d` 都**满足且严格低于** `e^{−d}` 上界，并精确符合 `rate[d] ≈ p₁^d` 的行间独立乘积结构。
