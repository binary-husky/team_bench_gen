# Count-Min Sketch：Heavy-Hitter / Top-k 识别的 Precision & Recall 随 sketch 大小的变化

> 复现自材料中的论文 *Cormode & Muthukrishnan, "An Improved Data Stream Summary: The Count-Min Sketch and its Applications"*（`countmin_material/cm-full.pdf`）。
> 代码：`cm_experiment.py`（主实验）、`cm_diagnostics.py`（假阳性诊断）；原始数值：`cm_results.json`。

## 1. 实现要点

- **Sketch**：`d` 个两两独立的哈希 `h_j : {1..n}→{1..w}`，采用 Carter–Wegman 族
  `h_j(x) = ((a_j·x + b_j) mod p) mod w`，素数 `p = 2^31−1`，`a_j≠0`、`b_j` 随机（与论文 §3、§6 "pairwise-independent, 2 words 存储" 一致）。
- **更新**（cash register，`c_t=+1`）：`count[j, h_j(i_t)] += 1`。
  利用 `c_t≡+1`，第 `j` 行可直接用 `np.bincount(h_j(整条流))` 一次性累加得到，等价于逐次更新但快得多。
- **点查询**：`â_i = min_j count[j, h_j(i)]`（论文 §4.1，非负情形取 min，保证 `a_i ≤ â_i`，一侧误差）。
- **Heavy-hitter / top-k**：流结束后对所有出现过的 item 估计 `â[i]`，按 `â[i]` 降序取 top-`k=100`，与真实 top-`k` 比对。
  另外也实现了论文 §5.2 的 φ-阈值规则（`â[i] ≥ φ·‖a‖₁`）作为对照（见下文 §3）。

## 4. 结论要点

1. **整体趋势：sketch 越大（`w,d` 越大），precision 与 recall 越高并趋近 1。**
   - `(512,3)`→`(4096,8)`：P@100 从 **0.422 单调上升到 1.000**，recall 同步。
   - 与之同步，点查询误差 `E[â−a]` 与最大高估 `max(â−a)` 随 `w`、`d` 增大而**单调下降**（902→351→145→49.5→17.7；最大高估 8902→…→58）。这与论文 Theorem 1 的保证 `â_i ≤ a_i + ε‖a‖₁`（`ε=e/w`）一致：`w` 翻倍 → ε 减半 → 每点误差上界减半；`d` 增大 → 失效概率 `δ=e^{-d}` 指数下降，min 估计更可靠。
   - 误差度量单调改善也独立佐证了"识别精度随 sketch 增大而提升"。

2. **小 sketch 下出现明显假阳性——长尾 item 因被高估而挤进 top-k（precision 下降）。**
   - `(512,3)` 平均每个种子有 54–67 个假阳性（即 top-100 里近六成是错的）。
   - 这些假阳性的**真实排名深埋于长尾**：诊断显示其真实频率分布在 1～169 之间（最低仅 1 次！），而真实 top-100 的边界频率为 836。也就是说，真实频率仅 1 次的长尾 item 的 `â` 被高估到足以挤进 top-100（最大高估达 ~8902）。
   - 机制（论文 §4.1）：`count[j,h_j(i)] = a_i + Σ_{k≠i, h_j(k)=h_j(i)} a_k`。`w` 小 ⇒ bucket 内碰撞多 ⇒ 少数 heavy hitter 的巨大计数"溢出"到与之同桶的长尾 item 上，`â_i = min_j` 仍偏高。`d` 小 ⇒ 多行之间取 min 的"去噪"机会少。结果长尾 item 的 `â` 反超真实 rank≈80–100 的中等高频 item，造成**precision 大幅下降（0.422）**，同时一批真实频率 836–2465 的 heavy hitter 被挤出 top-100（**recall 同步下降**）。

3. **关于最大配置 `(8192,10)` 的 recall 略降（1.000→0.996）：这是有限种子的统计噪声，不是真实退化。**
   - 逐种子看：`{1.000, 0.990, 1.000, 0.990, 1.000}`——两个种子各只有 **1 个** 假阳性，且其真实排名恰为 **100**（即与真实第 101 名在估计值上近乎并列、被排序切到分界线另一侧）。
   - 该配置的点查询误差（mae 17.7、最大高估 58）远优于 `(4096,8)`（49.5 / 145），证明 sketch 本身严格更优；在期望意义上 precision/recall 关于 `(w,d)` 是单调不减的，0.996<1.000 纯属 5 种子均值的小幅抖动。

4. **达到 P、R ≥ 0.95 的最小 sketch（近似）：`(w=1024, d=5)`**，即 ~5120 个计数（`w·d`）。
   注意到再小一档的 `(512,3)`（1536 个计数）只有 0.422，差距悬殊——说明在 Zipf(s≈1) 这种"重头 + 长尾"分布上存在一个相当 sharp 的精度拐点：当 `ε‖a‖₁ ≈ (e/w)·1e6` 降到与真实 top-100 边界频率（836）同量级或更小时（`w≈1024` 时 ε‖a‖₁≈2678 < 边界附近的频率差），识别质量才跨过 0.95。

### 附：φ-阈值 heavy-hitter 规则（论文 §5.2，对照）
按 `â[i] ≥ φ·‖a‖₁` 报告 heavy hitter 时，由于 CM 一侧误差 `â≥a`，**真 heavy hitter 一定会被报告（recall≈1）**，代价是可能多报告长尾假阳性（precision<1），随 `w,d` 增大假阳性减少——与上面 top-k 结论一致；本任务规定的主指标为 top-`k=100` 的 precision/recall，已列于 §3 表。
