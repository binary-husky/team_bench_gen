# Count-Min Sketch：点查询 vs 自连接 F2 / 内积 的相对误差

> 自实现 CM Sketch（numpy，`d=8` 个两两独立哈希 `h_j(x)=((a_j·x+b_j) mod p) mod w`，`p=2³¹−1`）。
> 固定设置：`w=2048, d=8`；流 `1e6` 次更新、`1e5` 个不同 item、Zipfian `s=1.0`；
> `‖a‖₁=1,000,000`，`F₂=Σa_i²=9.13×10⁹`，`‖a‖₁²/F₂≈109.5`；`≥10`（实际 20）个不同哈希种子，取跨种子均值。
> 实现/数据见 `cm_experiment.py`、`results.json`。

实现正确性自检：每行 F2 高估的测量值 = **5.32%**，与理论期望
`(‖a‖₁²−F₂)/w / F₂ = 4.84×10⁸ / 9.13×10⁹ = 5.30%` 几乎完全吻合 → 哈希两两独立性与 sketch 实现无误。

## 1. 两类查询的相对误差（跨 20 种子均值）

| 指标 | 数值（均值 ± std） |
|---|---|
| **点查询相对高估**，`(â_i−a_i)/max(a_i,1)` 对**所有** item 取均值 | **78.12  ± 9.59**　（≈ **7812 %**） |
| **F2 相对误差**，`(F̂₂−F₂)/F₂`（`F̂₂=min_j Σ_l C[j,l]²`） | **0.0353 ± 0.0003**　（≈ **3.53 %**） |

- 两者比值：**点查询(全 item 均值) / F2 ≈ 2213×**（即 F2_rel / 点查询_rel ≈ 0.00045）。
- F2 的"取 min 前"每行相对误差 = **5.32 %**（理论 5.30 %）；取 `min` 跨 8 行降到 **3.53 %**。

点查询相对高估按真实频率分桶（解释 7812% 从何而来）：

| 真实频率 `a_i` | item 占比 | 点查询相对高估 |
|---|---|---|
| 1 | 22.0 % | **16848 %** |
| 2–4 | 53.6 % | 6700 % |
| 5–9 | 15.1 % | 2855 % |
| 10–99 | 8.6 % | 1053 % |
| 100–999 | 0.7 % | 94.8 % |
| ≥1000 | 0.1 % | 8.8 % |

按 item 排名看头部：top-100 item 平均 11.8 %，top-10 平均 **1.27 %**，#1 item 仅 **0.24 %**。

## 2. 结论：F2/内积的相对误差是否显著大于点查询？

**按题目给定的字面指标（点查询对"所有 item"取均值），答案是「否」——恰恰相反：点查询的相对高估（≈7812%）比 F2 的相对误差（≈3.5%）大了约 2200 倍。**

但这**不是**因为 F2 更准，而是两类查询的**分母/归一化方式不同**造成的：

- **点查询**除以**逐 item** 的 `a_i`。Zipfian 流里约 75 % 的 item 极稀有（`a_i∈{1,…,4}`），而它们所在桶的碰撞质量（加性噪声）平均 ~170、对 `a_i=1` 的 item 高达数百。除以 `a_i=1…4` 后，相对误差爆炸到数千倍，并主导了"全 item 均值"。
- **F2** 是单一标量，分母 `F₂=Σa_i²` 被**少数头部 item 的大平方项**支配（#1 item 占 F₂ 的 ~60 %），这些大平方项本身稳定，于是同样规模的碰撞噪声相对 `F₂` 反而很小（~3.5 %）。

**机理（题目所指的"乘积叠加"是真实存在的，且已被定量验证）：**

对自连接，展开某行：
`Σ_l C[j,l]² = Σ_l (Σ_{i:h_j(i)=l} a_i)² = F₂ + Σ_{p≠q, h_j(p)=h_j(q)} a_p·a_q`

即 F2 的高估 = **同桶不同 item 计数的两两乘积之和**（碰撞噪声以**平方/乘积**形式叠加），这正是题目描述的机理。实测该乘积项每行 ≈ **4.84×10⁸**（相对 F₂ = 5.32 %，与理论 5.30 % 吻合）。对点查询而言，`â_i=min_j C[j,h_j(i)]` 取 min 后只剩**加性**碰撞噪声 `X_{i,j}=Σ_{k≠i:h_j(k)=h_j(i)} a_k`，没有"乘积放大"。**所以在「同一个桶的噪声」这一层面上，F2 确实把噪声放大成乘积/平方**——这也是论文 §7 明确指出 CM「不适合估计 L₂ 范数 / norm」的根本原因。

**该机理在题目隐含的"重 item / 信号自连接"区间里确实成立**：把点查询限制到**真正占主导的头部 item**（其计数远大于所在桶的加性噪声，点查询很准），F2 的相对误差（3.53 %）就**大于**这些头部 item 的点查询相对误差（top-10 ≈ 1.27 %，#1 item ≈ 0.24 %）。换句话说：

- 当被估量本身由"大且稳定"的成分主导时（头部 item 的点查询），CM 的加性碰撞噪声被分母压得很小 → 相对误差小；
- 而 F2 的乘积碰撞项 `Σ a_p·a_q` 规模由 `‖a‖₁²`（≈1e12）而非 `F₂`（≈9e9）决定，二者相差 ~110×，因此即便取 min，F2 仍残留 ~3.5 % 的相对误差，**大于头部 item 的点查询误差**。

**一句话总结**：碰撞噪声在 F2 里确实以乘积/平方形式叠加、量级很大（论文也因此判定 CM 不擅 norm）；但题目指定的"点查询对所有 item 取相对误差均值"指标被海量稀有 item（`a_i≈1`）的分母拉爆到 ~7800 %，**反超**了 F2 的 ~3.5 %。若改在"头部/高 频 item"或"绝对高估量"层面比较，则 F2/内积相对误差更大的结论成立——这正是乘积叠加机理起作用的有效区间。
