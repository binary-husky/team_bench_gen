背景：给定材料中 Brandes 算法对每个源 `s` 做一次最短路遍历（无权图即 BFS），得到 `σ[t]`（`s→t` 最短路条数）、前驱集合 `P[w]`（满足 `d(s,w)=d(s,v)+1` 的 `v`）以及一个按"到 `s` 距离"排序的节点栈。随后做**一次反向扫描**（按距离从远到近），用递推
```
δ[v] += (σ[v] / σ[w]) · (1 + δ[w])      （对每个 w 的每个前驱 v∈P[w]）
```
累加依赖 `δ_s•(v)`，并在 `w ≠ s` 时把 `BC[w] += δ[w]`。`n` 个源后即得所有节点的介数（无向图最后除以 2）。

---

**问题**（唯一一题）：

**为什么 Brandes 只需对每个源 `s` 做"一次反向扫描"就能得到 `s` 对所有中间节点的依赖 `δ_s•(v)`，而不必像朴素方法那样对每一对 `(s,t)` 重新枚举最短路并逐节点标记？** 请从最短路计数的角度逻辑推理回答，覆盖以下推理链（给定材料给出了复杂度定理，但需要你自己把这条"为何单次反向扫描就够"的直觉与递推链讲清楚）：

1. **`σ[v]/σ[w]` 这个比例代表什么**：当 `v ∈ P[w]`（即 `v` 是 `w` 的前驱、`d(s,w)=d(s,v)+1`）时，证明 `σ[v]/σ[w]` 恰等于"**所有 `s→w` 最短路中、经过 `v` 的那部分所占比例**"。（提示：每条 `s→v` 最路接上边 `(v,w)` 都唯一对应一条 `s→w` 最短路，反之 `s→w` 最短路必经某个前驱。）
2. **`(1 + δ[w])` 中 `1` 与 `δ[w]` 分别代表什么**：解释 `1` 对应"`w` 自己作为终点 `t=w`"时 `v` 的贡献（占比 `σ[v]/σ[w]`）；`δ[w]` 对应"`w` 下游更远的每个终点 `t` 经由 `w` 的依赖"，并说明为什么这些下游依赖可以**已经累加好**地拿来直接乘以 `σ[v]/σ[w]`（路径可拼接：`s→v→w→…→t` 的占比 = `(σ[v]/σ[w])·(σ[w]/σ[t])`）。
3. **为什么必须反向扫描（按到 `s` 距离从远到近）**：累加 `δ[v]` 时需要 `δ[w]` 已是**终值**；而 `w` 比 `v` 更远（`d(s,w)=d(s,v)+1`），所以先弹出（累加）更远的 `w`、再累加其前驱 `v`。说明 BFS 的栈恰好按距离排序，逆序弹出即天然满足该依赖顺序。

最后给出一句结论：**为什么这条反向依赖累加使 Brandes 在每个源 `O(m)`（一次遍历 + 一次反向累加）、总共 `O(nm)` 内算出所有节点介数，而朴素方法需要对 `(s,t)` 对逐个枚举最短路、无法在 `O(nm)` 内完成。**
