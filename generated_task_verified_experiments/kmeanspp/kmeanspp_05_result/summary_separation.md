# 簇分离度对 k-means++ 优势的影响

## 实验设置（固定）
- 数据集：`make_blobs(n_samples=5000, centers=10)`，数据集种子固定 `random_state=42`（每个 cluster_std 用同一份生成的数据）。
- 聚类：`KMeans(n_clusters=10, n_init=1, max_iter=300)`，自变量为 `cluster_std ∈ {0.5, 1.0, 1.5, 2.5, 4.0}`（从分离良好到高度重叠）。
- 对每个 `cluster_std`，对 `init='k-means++'` 与 `init='random'` 各跑 20 个 `random_state ∈ {0..19}`，记录平均 inertia。
- 比值 = `inertia_random / inertia_kmeanspp`（>1 表示 k-means++ 更优；=1 表示两者持平；<1 表示 random 反而更优）。

## 结果

| cluster_std | inertia (k-means++) | inertia (random) | ratio (random / k-means++) |
|------------:|--------------------:|-----------------:|---------------------------:|
| 0.5         | 2540.95             | 8965.82          | **3.5285** |
| 1.0         | 9545.92             | 13001.24         | **1.3620** |
| 1.5         | 18885.42            | 20007.05         | **1.0594** |
| 2.5         | 39089.47            | 38865.99         | 0.9943 |
| 4.0         | 65163.44            | 65334.14         | 1.0026 |

## 结论

**k-means++ 相对 random 的优势随簇重叠度增加而单调衰减，最终趋于消失（比值 → 1）。**

1. **分离良好（std=0.5）**：比值高达 **3.53**。此时每个簇是一个紧凑、互不交叠的团块，真实的全局最优把每个点分到其本属簇。`init='random'` 一次性随机选 k 个点极易出现多个中心扎堆在同一大簇、而某些簇没有任何初始中心（"空选"问题），落入远离真值的局部最优，inertia 大幅偏高。k-means++ 的 D² 加权播种强制让初始中心在空间上分散开，几乎一定能覆盖到每个簇，因此 inertia 接近最优、优势巨大。这正是 Arthur & Vassilvitskii (2007) 所证明的 O(log k) 期望近似比发挥作用、且实践中收益最显著的场景。

2. **中等重叠（std=1.0–1.5）**：比值从 1.36 降到 1.06，优势迅速收窄。簇开始交叠后，存在大量近乎等价的划分，inertia 曲面的局部最优之间差距变小；即便是较差的随机初始化，迭代后也能爬到一个相当不错的解，k-means++ 的"分散播种"收益递减。

3. **高度重叠（std=2.5–4.0）**：比值 ≈ **1.00**（std=2.5 时 0.9943，std=4.0 时 1.0026，两者已无统计意义上的差异，<1 处纯属随机抖动）。当 std 远大于簇间距，数据实质上退化成单一的高斯团，10 个中心几乎任意摆放都能达到相近的 inertia；初始化策略的选择对最终结果几乎没有影响。

**核心机理**：k-means++ 的优势本质上来自"避免初始中心扎堆、保证空间覆盖"。这一价值与簇间的可分性正相关——簇越分明，覆盖每个簇就越关键，扎堆的代价（漏掉整个簇）就越大；簇越重叠，划分本身越模糊、inertia 曲面越平坦，任何合理的初始化都收敛到接近的解，k-means++ 的边际收益也就趋于零。因此 k-means++ 的实际增益是**簇分离度的递减函数**：在分离良好的数据上是数量级的改进，在高度重叠的数据上与 random 难分伯仲。
