# 收敛迭代数：k-means++ vs random

## 任务

考察 Lloyd 迭代收敛所需的迭代数（`.n_iter_`）。固定数据集与参数，仅改变选种方式
（`init='k-means++'` vs `init='random'`，均 `n_init=1`），各跑约 30 个
`random_state`，比较平均迭代数、标准差与最差值。

## 固定设置（控制变量）

| 项目 | 值 |
|---|---|
| 数据集 | `make_blobs(n_samples=5000, centers=10, n_features=10, cluster_std=1.5)` |
| 数据集种子 | `random_state=42`（固定） |
| k | 10 |
| n_init | 1 |
| tol | 1e-4（sklearn 默认） |
| max_iter | 300 |
| algorithm | 'lloyd' |
| random_state 集合 | 0..29（共 30 个） |
| **自变量** | **选种方式：`'k-means++'` vs `'random'`** |

数据生成与全部运行脚本见 `run_experiment.py`，原始数据见 `results.npz`。

## 结果

| 指标 | `init='k-means++'` | `init='random'` |
|---|---|---|
| 平均迭代数 | **4.27** | **15.27** |
| 标准差（ddof=1） | 4.48 | 4.88 |
| 最差值（max） | **16** | **26** |
| 最优值（min） | 2 | 4 |
| 惯性 inertia 均值 | 125300.8 | 281412.6 |
| inertia 最差（max） | 193562.7 | 474763.9 |

各次 `.n_iter_` 原始记录：

- **k-means++**: `[3, 2, 2, 2, 4, 2, 13, 2, 2, 2, 10, 15, 16, 2, 2, 2, 2, 2, 2, 2, 2, 12, 11, 2, 2, 2, 2, 2, 2, 2]`
- **random**:    `[9, 18, 14, 11, 14, 9, 20, 15, 11, 10, 4, 19, 14, 19, 18, 23, 20, 26, 16, 20, 11, 13, 21, 14, 14, 18, 18, 14, 8, 17]`

## 结论

在固定数据集、k、`n_init=1`、收敛容差与 `random_state` 集合的前提下，**k-means++
选种显著减少了 Lloyd 收敛所需的迭代数**：

- **平均迭代数**：k-means++ 约 4.3 次，random 约 15.3 次——k-means++ 仅为 random 的
  ~28%，即平均节省约 11 次迭代（~72% 的下降）。
- **最差值**：k-means++ 最差 16 次，random 最差 26 次，k-means++ 的最坏情况也更优。
- **标准差**：两者量级接近（4.5 vs 4.9），但 k-means++ 的绝对波动更小，且其多 run
  仅为 2 次迭代（已接近下界，单步即收敛）；random 的迭代数则在 4–26 之间分散。
- 作为旁证，最终 inertia（簇内平方和）k-means++ 也明显更小且更稳定（均值 12.5 万 vs
  28.1 万，最差 19.4 万 vs 47.5 万），与 Arthur & Vassilvitskii (2007) 的结论一致：
  k-means++ 给出更好的初始点，使 Lloyd 过程既更快收敛又收敛到更低的目标值。

**一句话**：收敛迭代数 k-means++ vs random ≈ **4.3 vs 15.3**（mean），最差 **16 vs 26**，
k-means++ 全面优于随机选种。

## 备注 / 局限

- 数据集为合成数据（10 个 well-separated 球状簇），结构相对友好；对真实/重叠簇数据，
  绝对数值会变化，但 k-means++ 在迭代数与目标值上的相对优势通常仍成立。
- `n_init=1` 故意放大了选种方式的影响（无多次取最优兜底）；生产中常用 `n_init=10`
  会平滑差异，但单次 init 的对比更能体现选种质量本身。
- 本机 sklearn 1.2.1，CPU-only。
