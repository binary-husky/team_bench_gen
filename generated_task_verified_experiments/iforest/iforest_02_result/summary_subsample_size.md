# 子样本规模 ψ（max_samples）对 IsolationForest 异常检测 AUC 的影响

## 实验设置（固定项）

- **数据集**：`sklearn.datasets.make_blobs` 生成 **5000 个正常点**，2 维，5 个簇（`cluster_std=1.0`，`center_box=(-10,10)`，`random_state=42`）。
- **异常注入**：约 2% 的**均匀分布**离群点（`rng.uniform(-15,15,size=(100,2))`，同一种子 42）。最终 5100 个点，异常占 1.96%。
- **已知标签**：正常=+1，异常=-1（`y`），全程固定。
- **模型**：`sklearn.ensemble.IsolationForest(n_estimators=100, random_state=0)`。
- **唯一自变量**：`max_samples = ψ ∈ {16, 32, 64, 128, 256, 512, None(全量=5100)}`。
- **评估**：异常分取 `-clf.decision_function(X)`（分越大越异常），用 `roc_auc_score` 以「异常为正类」计算检测 AUC。

## 结果（检测 AUC 随 ψ 变化）

| ψ        | max_samples | 检测 AUC |
|----------|-------------|----------|
| 16       | 16          | 0.8726   |
| 32       | 32          | 0.9230   |
| 64       | 64          | 0.9471   |
| 128      | 128         | 0.9624   |
| 256      | 256         | 0.9653   |
| 512      | 512         | 0.9701   |
| None(全量) | 5100       | 0.9736   |

## 结论

1. **AUC 随 ψ 单调上升，但收益迅速递减。** 从 ψ=16 到 256，AUC 从 0.873 提升到 0.965（+0.092）；而从 ψ=256 到全量(5100)，仅再提升 +0.008，却需要约 20 倍的子样本量。
2. **拐点出现在 ψ≈128–256**：256 之后曲线近乎平台。这正契合 Liu et al. (2008) 论文的核心洞察与默认推荐 ψ=256——小样本已足以捕获绝大多数检测能力，继续增大 ψ 边际收益极小，而训练成本随 ψ 线性增长，故小 ψ 是更优的“性能/成本”折中点。
3. **未观察到论文中“大 ψ 导致掩蔽(masking)而使 AUC 下降”的现象**：本数据集异常仅 ~2% 且为远离簇心的均匀点，异常彼此稀疏，掩蔽/淹没效应微弱，故更多样本带来的是更精细的决策边界、AUC 略升而非下降。论文所述“小样本更好”的下降现象主要出现在异常密度更高、更易互相掩蔽的数据上——本实验的稀疏离群设置不满足该触发条件，因此表现为“单调上升 + 早期饱和”。
4. **实践含义**：在低异常率、异常空间稀疏的常见场景下，ψ 在 128–256 即可达到接近最优的检测 AUC；继续增大 ψ 几乎不再提升检测质量，徒增开销。论文“小样本即足够”的洞察在此以“早期饱和”的形式得到验证。
