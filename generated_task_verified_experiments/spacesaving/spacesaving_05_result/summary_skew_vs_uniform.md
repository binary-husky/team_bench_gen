# Space-Saving: Zipfian (skewed) vs Uniform — precision@k / recall@k

## Setup (fixed)

| 参数 | 值 |
|---|---|
| 流长 N | 1,000,000 |
| 计数器数 / top-k  k | 100 |
| 域基数 cardinality | 100,000 |
| 随机种子 | 42 |
| Zipf 指数 s | 2.0（强偏斜）|

自变量：数据分布（Zipfian s=2.0 vs 均匀）。其余全部固定。
基准：对全量流做精确计数，取真实 top-k。Space-Saving 实现 Metwally et al. 2005：被监控项计数 +1；未监控且桶未满则新建 (count=1, err=0)；桶满则驱逐最小计数项 `m`，新项 `count=m+1, err=m`。返回估计计数最大的 k 项。

注意：当真实 top-k 与估计 top-k 都恰好各 k 项时，`precision@k == recall@k`（二者分母同为 k），故本实验中两指标恒相等，下文合并报告。

## 结果

| 指标 | Zipfian (s=2.0) | Uniform |
|---|---|---|
| **precision@k** | **0.73** | **0.00** |
| **recall@k**    | **0.73** | **0.00** |
| 流中实际出现的不重复项数 | 1,373 | 99,997 |
| 真实 top-100 频率占比 | 0.9940 | 0.0022 |
| 最大真实计数 | 608,072 | 26 |
| 最大估计计数 | 608,072 | 10,000 |
| 监控项平均 \|估−真\| | 83.0 | 9,988.8 |
| 监控项最大 \|估−真\| | 241 | 9,996 |
| top-100 重叠项数 | 73 / 100 | 0 / 100 |

分布形状诊断：Zipfian 头部极度集中——rank-1 占 60.8% 质量，top-100 占 99.4%，top-1000 占 99.96%，整流只暴露 1,373 个不同项。均匀分布则平坦——最大项仅 26 次（期望 N/card=10 附近），top-100 仅占 0.22% 质量，几乎全部 100,000 项都出现。

## 结论：分布对 Space-Saving 效果的影响

**1. 强偏斜（Zipfian）下效果显著好，但仍非完美：precision@k = recall@k = 0.73。**
偏斜使头部项的真实计数远大于驱逐误差。例如 rank-1 真值 608,072，远超任何驱逐带来的误差（最大 241），头部项被牢牢钉在桶里、零误判。所以"绝对头部"完全正确。损失全部发生在 top-k 边界附近：rank 97–105 的真实计数彼此仅差 2–4（56–60 之间），而此时桶内最小计数（即驱逐误差上界）也在 ~58 量级——误差与边界项的真实计数同阶。于是边界附近的项被频繁换入换出，27 个真实 top-100 项被同等量级的"邻居"挤出。这解释了 0.73 而非 1.0：误差正比于最小计数 `m`，偏斜把头部抬高但不抬高边界，边界依然是模糊的。

**2. 均匀分布下效果崩溃：precision@k = recall@k = 0.00，top-100 重叠 0/100。**
均匀分布没有可被"节省"的重频项：所有项计数都在 10 上下，top-100 与 rank-101–200 在统计上不可区分，真实 top-k 几乎是抽签。Space-Saving 的桶只能容纳 100/100,000 项；每次驱逐把最小计数（很快爬到 ~10,000）累加进新项，导致返回的 100 项估计计数被严重高估（最大估计 10,000 vs 真实 26，平均误差 9,989），却和真实 top-100 毫无交集。`est_topk_freq_share=1.0` 是退化假象——Space-SAVING 保持"桶内计数之和 = N"恒成立，故返回的 k 项之和恒为 N，与分布无关，不能用作准确度证据。

**3. 机理总结。** Space-Saving 的误差界 `err ≤ (N − 监控项真计数) / k`（最坏 `≤ N/k = 10,000`）在两种分布下都成立，但**有效误差取决于"被驱逐的最小计数 m 相对于真实 top-k 计数的比值"**：
- Zipfian：头部真实计数 ≫ m，头部无误差；只在 m ≈ 边界项计数的模糊带产生部分误判 → 0.73。
- Uniform：真实 top-k 计数 ≈ m（都在 ~10–26 量级，而 m 漂到 ~10^4），误差完全淹没信号 → 0.00。

一句话：**Space-Saving 是偏斜敏感的——数据越偏斜、头部越突出，top-k 越准；分布越平坦，top-k 越退化为随机抽样，精确度趋近于 0。** 这与 Space-Saving 的设计前提（重击者 / heavy-hitter 场景）完全一致：它在"少数项占多数质量"的流上才真正"节省"。
