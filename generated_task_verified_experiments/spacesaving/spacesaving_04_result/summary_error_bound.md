# 验证 Space-Saving 的误差保证

**算法**：Metwally, Agrawal, El Abbadi, *Efficient Computation of Frequent and Top-k Elements in Data Streams* (2005) 中的 Space-Saving（k 个计数器，Stream-Summary）。

**实验设置（固定）**
- Zipfian 流，参数 `s = 1.1`，字母表大小 `|A| = 10 000`
- 主run：`N = 1×10⁶`，`k = 100`
- 随机种子 `seed = 42`（`numpy.random.default_rng(42).zipf(1.1)`）
- 自变量（用于界收紧检查）：`k`（固定 N）与 `N`（固定 k）
- 实现：`spacesaving_verify.py`（逐元素对照真实频率 `Counter(stream)` 断言；CPU-only，约 12 s）

被验证的三条保证对应论文 Lemma 2 / Lemma 3 / Theorem 1-2：

| # | 保证 | 论文依据 |
|---|------|----------|
| G1 | 每个被监控元素的估计 `f̂ = count_i` 都 ≥ 真实 `f_i`（无低估） | Lemma 3：`0 ≤ ε_i`，`count_i = f_i + ε_i ≥ f_i` |
| G2 | 最大高估误差 `max(count_i − f_i) ≤ 当前最小计数器值 min` | Lemma 3：`ε_i ≤ min`，而 `count_i − f_i ≤ ε_i` |
| G3 | `min ≤ N/k` | Lemma 2：`min ≤ ⌊N/m⌋`，`m = k` |

## 1. 主run结果（N=1×10⁶，k=100）

| 指标 | 值 |
|------|----|
| N/k | 10 000.0 |
| 当前最小计数器 `min` | 8 279 |
| 最大高估误差 `max(count−f)` | 8 279 |
| 最大存储 `ε_i` | 8 279 |
| **G1（无低估）** | **成立** — 0 个违反 |
| **G2（max高估 ≤ min）** | **成立** — 8 279 ≤ 8 279（紧贴） |
| **G3（min ≤ N/k）** | **成立** — 8 279 ≤ 10 000（余量 1 721） |

头部高频元素 `ε = 0`（从未被驱逐，估计即精确）；越靠尾部、被驱逐过越多次的元素 `ε` 越大，但每一行的 `count − f` 都为非负且 ≤ `ε` ≤ `min`，与 Lemma 3 一致。最大高估恰好等于 `min`（出现在某个刚被换入最小桶、真实频率很小的元素上），使 G2 取到等号——界的紧致性得到体现。

**三条保证在主run中全部成立。**

## 2. 界随 N/k 的变化（收紧检查）

### 2a. 固定 N=1×10⁶，增大 k（N/k 缩小）

| k | N/k | min | max高估 | ε_max | G1 | G2 | G3 |
|---:|----:|----:|--------:|------:|:--:|:--:|:--:|
| 50 | 20 000 | 17 653 | 17 653 | 17 653 | ✓ | ✓ | ✓ |
| 100 | 10 000 | 8 279 | 8 279 | 8 279 | ✓ | ✓ | ✓ |
| 200 | 5 000 | 3 869 | 3 869 | 3 869 | ✓ | ✓ | ✓ |
| 400 | 2 500 | 1 806 | 1 806 | 1 806 | ✓ | ✓ | ✓ |
| 800 | 1 250 | 842 | 842 | 842 | ✓ | ✓ | ✓ |
| 1 600 | 625 | 392 | 392 | 392 | ✓ | ✓ | ✓ |

### 2b. 固定 k=100，增大 N（N/k 放大）

| N | N/k | min | max高估 | ε_max | G1 | G2 | G3 |
|----:|----:|----:|--------:|------:|:--:|:--:|:--:|
| 100 000 | 1 000 | 826 | 826 | 826 | ✓ | ✓ | ✓ |
| 250 000 | 2 500 | 2 070 | 2 070 | 2 070 | ✓ | ✓ | ✓ |
| 500 000 | 5 000 | 4 138 | 4 138 | 4 138 | ✓ | ✓ | ✓ |
| 1 000 000 | 10 000 | 8 279 | 8 279 | 8 279 | ✓ | ✓ | ✓ |
| 2 000 000 | 20 000 | 16 561 | 16 561 | 16 561 | ✓ | ✓ | ✓ |

**观察**：
- `min`（以及随之 `max高估`、`ε_max`）随 `N/k` 近似线性变化——`k` 翻倍则 `min` 大致减半（2a：17 653 → 8 279 → 3 869 → 1 806 → 842 → 392），`N` 翻倍则 `min` 大致翻倍（2b：826 → 2 070 → 4 138 → 8 279 → 16 561）。
- 在 Zipfian(s=1.1) 下 `min ≈ 0.66–0.83·(N/k)`，始终严格小于 `N/k`，故 G3 恒成立且有常数级余量；随 `N/k` 减小，绝对误差界（`min`）同比例收紧。
- 三条保证在全部 11 组配置（1 主run + 6 + 4）中**无一违反**——G1 零低估、G2 `max高估 ≤ min`（多处取等）、G3 `min ≤ N/k`。

---

## 3. 结论

在固定种子、固定 Zipfian(s=1.1, |A|=10⁴) 流上的实验验证了 Space-Saving 的三条误差保证**全部成立**：

1. **无低估（G1）**：所有被监控元素的 `count_i ≥ f_i`，0 例违反——算法只向正方向偏（benefit of the doubt）。
2. **高估上界（G2）**：最大高估误差 `max(count_i − f_i)` 恒等于/不超过当前最小计数器 `min`（与 `ε_i ≤ min` 一致），且在多种配置下取等，说明界紧致。
3. **min 上界（G3）**：`min ≤ N/k`（Lemma 2），实验中 `min ≈ 0.7·(N/k)`。

**界随 N/k 收紧**：固定 N 增大 k、或固定 k 减小 N，`N/k` 减小时 `min`（及最大高估误差）按同比例下降；反之按同比例上升。即误差界由 `N/k` 主导，Space-Saving 的精度随计数器数 `k` 增多、随流长 `N`（相对 k）减小而单调收紧——与论文理论分析一致。
