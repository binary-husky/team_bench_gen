# Space-Saving: 计数器之和恒 = N ⟹ 误差 ≤ N/k

> 任务：仅逻辑推理——(1) k 计数器之和为何恒 = N；(2) 最小计数器 ≤ N/k；(3) 高估误差确定性上界。
> 依据 Metwally-Agrawal-El-Abbadi, *Efficient Computation of Frequent and Top-k Elements* (ICDT 2005)。

## (1) 之和恒 = N

每条流到达都恰好让某一个计数器 +1，三种情形：
- 元素已被监控：其自身计数器 +1（净 +1）。
- 未监控且表已满：替换最小计数器为新元素，值由 `min` 变 `min+1`（净 +1）。
- 未监控且有空闲槽：新建计数器取值 1（净 +1）。

三情形总和都 +1 ⟹ N 条到达后总和 = N。

## (2) 最小计数器 ≤ N/k

k 个计数器之和 = N。若每个都 > N/k，则和 > k·(N/k) = N，矛盾。故**最小计数器 ≤ N/k**。

## (3) 高估误差上界

Space-Saving 只增不减、被替换元素以 `min` 为初值 ⇒ 任何被监控元素频率估计 `f̂ ≥ f`（只高估）。高估量 `f̂ − f ≤ min ≤ N/k`（被监控元素的计数器初值至多是它被插入时的 min，后续 min 只增）。故**任意被监控元素频率高估误差 ≤ ε ≤ N/k**，一个与分布/随机性无关的**确定性硬上界**。

## 一句话总结

每条到达恰让某计数器 +1（三情形皆净 +1）⟹ k 计数器和 = N ⟹ min ≤ N/k；Space-Saving 只增不减 ⇒ 被监控元素高估误差 ≤ min ≤ N/k，确定性硬上界（与分布无关）。
