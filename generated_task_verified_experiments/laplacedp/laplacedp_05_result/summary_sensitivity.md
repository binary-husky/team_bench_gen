# 查询敏感度 Δf 对 Laplace 机制噪声/效用的影响

## 材料

Dwork, McSherry, Nissim, Smith (2006), *"Calibrating Noise to Sensitivity in
Private Data Analysis"*。该文的核心结论：对于一个查询函数 `f`，定义其
**L1 敏感度** `S(f) = Δf` 为「任意改变数据库中一行时，`f` 输出所能变化的最大
幅度」。为保证 `ε`-不可区分性（即差分隐私），只需对真实答案加
**Laplace 噪声**，其尺度（scale）

```
b = Δf / ε      即密度 ∝ exp(−|y| / b) = exp(−ε|y| / Δf)
```

噪声尺度**只取决于 ε 与查询本身的敏感度 Δf，与数据库内容及规模无关**。
本实验固定 ε，改变查询类型（即 Δf），测量各查询的平均绝对误差（MAE），
验证「误差随 Δf 的变化」。

## 实验设置（固定）

| 项 | 值 |
|---|---|
| 隐私参数 ε | 1.0 |
| 数据规模 N | 1000 条记录 |
| 求和值域上界 B | 10（取值 ∈ {0,…,10}） |
| 每查询重复试验次数 | 200 000 |
| 随机种子 | 20260626 |
| 数据库 | 一次性生成后固定不变 |

三种查询及其敏感度（自变量为查询类型 ⇒ Δf）：

| 查询 | 真值 | 敏感度 Δf | 说明 |
|---|---|---|---|
| (a) 计数查询 `#{xᵢ ≥ 1}` | 900 | **1** | 增删一行最多改变计数 1 |
| (b) 区间 [0,B] 求和 `Σxᵢ` | 4933 | **B = 10** | 单个值最多变 B，求和最多变 B |
| (c) 均值查询 `mean(xᵢ)` | 4.933 | **1/N = 0.001** | 单个值最多变 B，除以 N 后变 B/N… 但按任务口径取 Δf=1/n |

> 注：均值查询严格敏感度为 B/N。任务给定口径为 Δf=1/n（B=1 时的归一化形式），
> 本实验按任务口径取 Δf=1/N=0.001，结论不受影响——MAE 仍由 `Δf/ε` 线性决定。

噪声机制：`release = true_answer + Laplace(0, Δf/ε)`，每次独立采样。

## 结果

| 查询 | Δf | 噪声尺度 b=Δf/ε | 实测 MAE | 理论 MAE |
|---|---:|---:|---:|---:|
| 计数 | 1.0000 | 1.0000 | **0.9992** | 1.0000 |
| 求和 [0,B] | 10.0000 | 10.0000 | **10.0054** | 10.0000 |
| 均值 | 0.001000 | 0.001000 | **0.000996** | 0.001000 |

比值检验：
- `MAE_sum / MAE_count = 10.01`，与 `Δf_sum / Δf_count = 10` 吻合；
- `MAE_count / MAE_mean = 1003.4`，与 `Δf_count / Δf_mean = 1000` 吻合；
- 以三点 (Δf, MAE) 做线性拟合，斜率 = **1.0006**，理论斜率 = 1/ε = 1.0。

（完整数值另存于 `results.json`，脚本见 `experiment.py`。）

## 结论：误差随查询敏感度 Δf 的变化

1. **线性正比关系。** 在固定 ε 下，Laplace 机制的平均绝对误差
   `MAE = E|Lap(0, Δf/ε)| = Δf/ε`，与 Δf **严格线性**。实验三点几乎完美落在
   过原点、斜率 1/ε 的直线上（拟合斜率 1.0006，R²≈1）。这正印证了论文标题
   “Calibrating Noise to Sensitivity”——**噪声是按敏感度校准的**：敏感度翻几倍，
   误差就翻几倍。

2. **与数据库规模无关、只与查询函数的固有性质有关。**
   - 计数查询 Δf=1，MAE≈1；
   - 求和查询 Δf=B=10，MAE≈10，比计数差 **10 倍**——值域越宽，单行贡献越大，
     必须加越多噪声；
   - 均值查询 Δf=1/N，MAE≈0.001，比计数小 **1000 倍**——分母 N 把单行影响
     “摊薄”了，敏感度随 N 线性下降，误差也随之下降。

3. **隐私-效用权衡的杠杆点在 Δf，而非 ε。** 固定 ε 时，效用完全由查询设计决定：
   把“宽域求和”换成“计数”或“均值”，可在**同等隐私**（同 ε）下把误差降低 1～3 个
   数量级。反之，若查询本身敏感度高（如求和 Δf=B），即便 ε 很宽松，误差仍以
   B 量级存在。论文据此强调：对直方图、列联表、协方差等高维输出，其敏感度
   **与维度无关**（为 O(1)），因此可用 O(1) 量级噪声而非按维度线性增长的噪声——
   这正是“按敏感度而非按输出维度校准”带来的效用红利。

4. **实践含义。** 想在固定 ε 下提升效用，优先选择/构造**低敏感度**的查询
   （计数、均值、直方图），或对高敏感度查询做变换（如求和→均值、裁剪值域
   以降低 B、或用稍大的 N 摊薄）。ε 是全局隐私预算杠杆；Δf 是单查询效用杠杆，
   二者共同以乘积形式决定误差 `Δf/ε`。
