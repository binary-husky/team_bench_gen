# Chord 节点加入与稳定化（stabilization）实验结论

> 任务：在一个**进程内虚拟 Chord 环**上，从已稳定的 N0=500 节点环出发，一次性批量加入 Δ ∈ {50, 100, 200} 个新节点，随后逐轮执行 `stabilize()` / `fix_fingers()`，每轮发起约 1×10⁴ 次随机键查找，统计查找正确率随稳定化轮数的变化，并记录达到 ≈100% 正确率所需轮数。
>
> 实现依据：Stoica et al., *Chord: A Scalable Peer-to-peer Lookup Service for Internet Applications*, SIGCOMM 2001（即本目录 `chord_material/` 所给论文）。代码见 `chord_sim.py`，原始运行输出见 `run_output.txt`。

---

## 一、结论速览（TL;DR）

1. **查找正确率随轮数单调上升，并在很少几轮内收敛到 100%。** 在主种子下，Δ=50 与 Δ=100 在 **第 4 轮** 达到 100%；Δ=200 在 **第 5 轮** 达到 100%。
2. **Δ 越大，起始正确率越低、收敛所需轮数越多**，但即使一次性涌入 200 个新节点（占环 28.6%），也只需约 **4–7 轮**（多种子中位数 4.5）即可完全恢复。
3. **加入瞬间（第 0 轮，尚未 stabilize）的正确率近似等于 N0/(N0+Δ)**：被新节点“截走”的那段键空间，在老节点后继指针被修正之前会查找失败。
4. **关键交叉验证**：每一轮的“查找正确率”与“后继指针正确率”高度吻合（绝大多数轮相差 ≤0.01；主种子下最大相差约 0.04，出现在 Δ=100 第 0 轮，属 10⁴ 次查找的抽样噪声与具体路由路径差异）。这从实测角度印证了论文 §5.1 的论断——**指表（finger table）只影响查找速度、不影响正确性；正确性完全由后继指针是否收敛决定**（Theorem 5：最后一次加入完成后，所有后继指针最终都会变正确）。

## 二、方法

### 2.1 协议实现（严格对照论文伪代码）

- **查找**（论文 Figure 4）：`find_successor` ← `find_predecessor` ← `closest_preceding_finger`，迭代式（与论文模拟器一致，§6.1）。
  - `find_predecessor` 终止条件 `id ∈ (n', n'.successor]`（左开右闭）；
  - `closest_preceding_finger` 取落在 `(n, id)`（开区间）内、下标最大的 finger。
- **稳定化方案**（论文 Figure 7，用于并发加入，替换 Figure 6）：
  - `join(n')`：`predecessor = nil`，`successor = n'.find_successor(n)`；
  - `stabilize()`：读后继的前驱 `x`，若 `x ∈ (n, successor)` 则把后继改为 `x`，再 `successor.notify(n)`；
  - `notify(n')`：若 `predecessor is nil` 或 `n' ∈ (predecessor, n)` 则更新前驱；
  - `fix_fingers()`：随机选一个 finger 下标，`finger[i] = find_successor(finger[i].start)`。

### 2.2 实验流程（对应任务“固定设置 / 自变量”）

| 项 | 取值 | 说明 |
|---|---|---|
| N0 | 500 | 初始已稳定环 |
| Δ | {50, 100, 200} | **自变量之一** |
| 标识长度 m | 24（环 2²⁴） | 700 节点下足够稀疏；m 不影响**正确性**动力学，只影响 finger 路由速度 |
| 每轮查询数 | 10 000（≈1×10⁴） | 随机键、随机源节点 |
| 随机种子 | 20240601（主） | 可复现；另跑 8 个种子做稳健性检验 |
| 轮数 | 自变量之一 | 每轮：所有节点各执行一次 `stabilize()` 与一次 `fix_fingers()` |
| 稳定化例程 | Figure 7 | 固定不变 |

- **“一次性加入 Δ 个”的实现**：Δ 个新节点**同时**以原 500 节点稳定环为接入点计算各自初始后继（`batch_join` 中用**旧环快照** `old_sorted` 求 `true_successor`），因此它们**看不到彼此**——这是真实的“并发加入”语义。新节点初始 `predecessor=nil`、finger 表全部指向其（旧环）后继；老节点对加入者一无所知（后继/前驱/指表均未更新）。
- **真值后继**：相对“加入后真实环成员”（全部 N0+Δ 个节点）的即时后继，用排序数组 + 二分直接求得，作为查找正确与否的判据。
- **每轮内执行顺序**：每轮用（种子化的）随机置换顺序逐节点跑 `stabilize()`（每次可见本轮内此前已更新的状态），再逐节点跑一次 `fix_fingers()`；随后发起 1×10⁴ 次随机查找统计正确率。该“轮”模型可复现。

> 说明：正确性只取决于后继指针，故“轮内顺序 / 同步 vs 异步”只会轻微改变收敛轮数（±1~2 轮），不改变定性结论。

## 四、稳健性（8 个额外种子，相同流程）

| Δ | 第 0 轮查准率（均值 / 范围） | 达到 ≈100%（≥0.9999）所需轮数 | 达到 ≥0.999 所需轮数 |
|---:|---|---|---|
| 50  | 0.914 / 0.899–0.933 | 中位数 3（范围 2–4） | 2–4 |
| 100 | 0.838 / 0.817–0.860 | 中位数 3.5（范围 3–5） | 3–4 |
| 200 | 0.706 / 0.670–0.744 | 中位数 4.5（范围 4–7） | 3–7 |

主种子（Δ=200 得 5 轮）落在该范围内。结论稳健：**Δ 增大单调地拉低起始正确率并增加收敛轮数；Δ=200 的最坏种子约需 7 轮**。

## 五、分析与解读

1. **起始正确率 ≈ N0/(N0+Δ) 的来源。** 加入瞬间老节点对加入者毫无感知：凡真值后继为**新节点**的键，老节点路由到该区段时其（旧的）后继指针会“越过”新节点，从而返回错误后继。被新节点截走的键空间比例约为 Δ/(N0+Δ)，故起始查准率 ≈ 1 − Δ/(N0+Δ) = N0/(N0+Δ)：

   | Δ | N0/(N0+Δ) 预测 | 多种子实测均值 |
   |---|---|---|
   | 50  | 0.909 | 0.914 |
   | 100 | 0.833 | 0.838 |
   | 200 | 0.714 | 0.706 |

   吻合良好，说明第 0 轮的失败几乎全部来自“尚未被链入环的新节点区段”。

2. **为何只需几轮。** `stabilize()` 每轮把“后继前驱”这一信息沿环传播一格、并把新节点逐个链入。落入同一旧区间的新节点形成短链，每轮被剥离一两个；落入同一区间的并发加入者越多（Δ 越大、生日碰撞越重），最长链越长，收敛轮数越多——这解释了 Δ=200 比 Δ=50 多 1–3 轮。500 个旧区间中，Δ=200 时最拥挤区间的新节点数仍是个位数，故收敛只需个位数轮。

3. **指表与正确性解耦（实测验证论文 §5.1）。** 每一轮“查找正确率”与“后继指针正确率”高度重合（主种子下逐轮差绝大多数 ≤0.01、最大约 0.04，多种子同样成立）。原因是 `closest_preceding_finger` 只会跳到 `(n, id)` 严格前方的节点，单调逼近、绝不会越过 id；因此只要后继指针正确，`find_predecessor` 必终止于真前驱。`fix_fingers()` 改善的只是跳数（速度），不改变能否查对。这与论文“correctness 仅依赖后继指针；指表只让查找又快又对”完全一致，也是 Theorem 5（最后一次加入后所有后继指针终将正确）的直接体现。

4. **与论文实验定性一致。** 论文 §6.5 指出加入/失效引发的不一致是**瞬态**的，stabilize 会很快修正后继指针；本实验把“一次性大批量加入”这一极端情形量化为：**即便 Δ=200（占环 28.6%），查准率也在约 5 轮（最坏 ~7 轮）内回到 100%**，且修复过程平滑单调、无振荡。
