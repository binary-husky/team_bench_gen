# Chord「后继列表（successor list）」在节点失效下维持路由正确性的机制

> 任务要求：仅凭逻辑推理（不做实验）回答——后继列表如何让路由在失效下仍正确工作？直接后继失效时，是什么性质使节点仍能向正确方向转发？路由在什么具体条件下才真正断裂（节点何时「搁浅 / stranded」）？并据此说明为何系统能容忍 ≈½ 的节点失效，而非更小或更大。
>
> 本文全部推导依据所给材料 *Chord (Stoica et al., 2001)*，关键佐证来自 §4.3、§5.2、定理 7/8 与 §6.4。

---

## 0. 结论速览（TL;DR）

1. **路由的正确性只依赖后继指针**（finger table 只是加速）。`find_predecessor` 在最坏情况下**仅靠后继指针**就能沿环单调向前走。
2. **后继列表 = 每个节点维护环上最近的 r 个连续后继**。它给那唯一的、决定正确性的「后继指针」加上了**冗余**，且这些冗余条目是**环上连续无空洞**的。
3. 直接后继失效时，节点**落到列表里第一个存活条目**即可继续转发——该条目仍在前方（方向正确），且因为是连续列表，它恰好是环上下一个**存活**节点，**不跳过任何存活节点**，故既保持单调前进又保持正确性（不会越过目标）。
4. **节点「搁浅」的充要条件**：它的 r 个后继**全部**失效 ⟺ 它在环上之后存在一段长度 **≥ r 的连续失效**。等价地，**全网路由断裂 ⟺ 失效模式中存在长度 ≥ r 的连续失效段**。
5. 取 **r = log₂ N**，每节点独立以概率 f 失效时，搁浅节点的期望数 ≈ N·f^r = N^(1+log₂ f)。该量在 **f = ½** 处恰好从「趋于 0」翻转为「趋于 ∞」——这就是相变临界点。**½ 正是 r = log₂ N 这一设计标定出的失效容忍阈值**：更小则冗余浪费、更脆弱；更大需付出指数级更大的每节点状态。

---

## 1. 出发点：正确性只取决于后继指针（后继列表的「地基」）

材料反复强调一条根本性质——**只要后继指针正确，路由就（即便很慢也）正确**：

- §4.3：「Each node need only be aware of its successor node on the circle. Queries … can be passed around the circle via these successor pointers … This additional information [finger table] is not essential for correctness, which is achieved as long as the successor information is maintained correctly.」
- 引言：「Only one piece information per node need be correct in order for Chord to guarantee correct (though slow) routing of queries.」
- §5.2：「The key step in failure recovery is maintaining correct successor pointers, since in the worst case `find_predecessor` can make progress using only successors.」

**为什么后继指针足够？** 因为 `find_predecessor` 的每一步都把请求交给「严格更接近目标、且不越过目标」的节点；退化到极端，它就是沿后继指针绕环线性前进（clockwise）。后继指针永远指向环上**前方**的下一个节点，于是这条链**单调向前、不会回退**，环有限，故必然到达目标的前驱——这就是「正确但慢」的路由。finger table 只是把这条线性走法加速成 O(log N) 跳。

> **推论**：失效恢复的全部要害，就是**让那根唯一的、决定正确性的后继指针不断**。后继列表正是为此而设。

---

## 2. 后继列表提供的两个性质

§5.2：「each Chord node maintains a ‘successor-list’ of its **r nearest successors** on the Chord ring. … If node n notices that its successor has failed, it **replaces it with the first live entry** in its successor list.」

由「最近的 r 个后继」可抽出两条性质，缺一不可：

- **(R) 冗余（Redundancy）**：n 手里不止一根后继指针，而是 r 根。丢掉第一根（直接后继），还有 r−1 根候选。
- **(C) 连续无空洞（Contiguity）**：这 r 个条目是环上**紧挨着 n 之后**的 r 个连续节点，**中间没有缺口**。

单有冗余不够（条目若随机散布，落到某条目可能越过目标或落在目标后方）；单有连续不够（只有一个后继时，一次失效即搁浅）。**两条性质合起来**，才让「后继指针」变得可容错。

---

## 3. 直接后继失效时，凭什么仍能向「正确方向」转发

设 n 的直接后继失效。n 改用后继列表中**第一个存活条目** m。要论证 m 仍是合法的转发目标：

**(a) 方向正确**：列表每一项都是 n 在环上的后继，故 m 一定在 n **前方**（clockwise）。转发仍在向目标方向走，**单调前进**没有破坏。

**(b) 不跳过存活节点（关键，由连续性保证）**：因为列表是连续无空洞的 r 个最近后继，从 n 往后**逐个排查**，第一个存活的就是环上紧邻 n 的**下一个存活节点**——没有任何存活节点被遗漏在 n 与 m 之间。这带来两点：

- **正确性保持**：转发链永远不会「跳过」本该负责 key 的存活节点，所以最终落到的就是目标的**最近存活后继**（定理 7 所断言）。
- **不会越过目标**：`find_predecessor` 的循环条件是「当 `n'.successor` 越过 key id 时停止」。由于 m 是「最近存活后继」，当 n 的有效后继首次越过 id 时，n 恰为 id 的前驱、其有效后继恰为答案。若条目不连续，就可能一步跳过 id 的真正后继、落到更远处而返回错误节点——连续性排除了这一可能。

> **小结**：使失效下仍能正确转发的「性质」= **「连续、有序、冗余的前向指针集合」**；它让那根决定正确性的后继指针被复制 r 份，且每份都是环上无空洞的连续邻居，因此掉一根仍能落到「前方且最近」的存活节点上。finger 表失效或失准都不影响这条底线（§5.2：失效的 finger 可用其前序条目或后继列表条目替代，但**底线正确性只靠后继链**）。

## 4. 路由何时真正断裂：节点何时「搁浅（stranded）」

**定义**：一个**存活**节点 n 处于「搁浅」状态，指它**没有任何一个存活的前向指针可用**——即它无法把请求继续向目标方向交出去。

在后继列表机制下，n 唯一**可靠**的前向信息就是这 r 个后继。所以：

> **n 被搁浅 ⟺ n 的 r 个后继列表条目全部失效 ⟺ n 在环上之后存在一段长度 ≥ r 的连续失效节点。**

注意「搁浅」不是「环上没有存活节点」——失败节点的远端往往仍有大量存活节点——而是 **n 的视野（长度 r 的列表）被这段死亡间隙耗尽**，n 不知道如何跨过这段间隙。连续性使得「跨不过去」与「间隙长度 ≥ r」严格等价。

于是有干净的全局判据：

> **全网路由断裂（存在某个 lookup 失败） ⟺ 失效模式中存在一段长度 ≥ r 的连续失效。**
> 反过来：**后继列表长度 r 可容忍任何不含 r 连续失效的失效模式。**

（若每个存活节点都至少有一个存活后继条目，则每个存活节点都能单调向前，环有限故每条 walk 都到达目标区域；连续性保证不跳过存活节点，故不错答。）

## 5. 为什么恰好是 ≈ ½（核心推导）

设 N 个节点，初始稳定，随后**每个节点独立**以概率 f 失效，取 **r = log₂ N**（材料中 r = O(log₂ N)，且定理 7/8 的直觉段写明 r = log₂ N）。

**第 1 步：单节点被搁浅的概率。** n 的 r 个后继**全部**失效的概率 = f^r。

**第 2 步：用 union bound 估全网搁浅节点数。** 期望搁浅节点数 ≈ N · f^r（存活节点数是 (1−f)N，仅常数因子，不影响阈值）。
代入 r = log₂ N：

  f^r = f^(log₂ N) = 2^(log₂ f · log₂ N) = N^(log₂ f)

所以

  **期望搁浅数 ≈ N · N^(log₂ f) = N^(1 + log₂ f)**

**第 3 步：阈值在 f = ½。** 该量的指数是 `1 + log₂ f`：

| 失效比例 f | 1 + log₂ f | 期望搁浅数（随 N） | 含义 |
|---|---|---|---|
| f < ½（如 ¼） | < 0（= −1） | → 0 | **过度安全**：冗余远超所需，浪费每节点状态 |
| **f = ½** | **= 0** | **= O(1)** | **临界**：whp **不存在**搁浅节点 → 路由存活 |
| f > ½（如 ¾） | > 0（≈ 0.585） | → ∞ | **环碎裂**：大量节点搁浅，路由大面积断裂 |

f = ½ 时：f^r = (½)^(log₂ N) = **1/N**，全网期望搁浅 = N · (1/N) = O(1)；由 union bound，whp **零**搁浅节点。这**正是材料定理 7/8 直觉段的原话**——「a node's r successors all fail with probability 2^(−r) = 1/N, so with high probability a node will be aware of … its closest living successor」——于是定理 7（返回最近存活后继，正确）与定理 8（期望 O(log N)，高效）同时成立。§6.4 的模拟也证实：失效查找比例几乎恰好等于 f（仅丢失负责节点本身持有的 key），没有出现「对半分区则一半请求失败」那种碎裂迹象。

> **所以「½」不是随便挑的数字**：它是 `1 + log₂ f = 0` 的解，是搁浅期望数从「趋于 0」到「趋于 ∞」的**相变临界点**；而把临界点恰好定在 ½ 的，正是 `r = log₂ N` 这一标定。

**为什么不能「更大」？** 想容忍更高比例 f\*，须让 N·f\*^(c log₂ N) = O(1)，解得 **f\* = 2^(−1/c)**，其中 r = c·log₂ N。c = 1 → ½；要 f\* = 0.707 需 c = 2（每节点状态翻倍）；f\* 越大，c 越大，每节点状态开销线性增长，且会越过相变进入「微小扰动即级联碎裂」的脆弱区。换言之，½ 是**给定对数级每节点状态**下、能稳定站住的最高失效比例。

**为什么不能「更小」？** 选更小的容忍比例（等价于更小的 c）等于**主动降低**冗余，系统在更轻的失效压力下就崩；而把目标定在远低于 ½ 则意味着冗余严重过剩、每节点状态被浪费——½ 恰是「每单位状态换得的最大容忍失效比例」的甜点。

**为何是「≈」而非精确 ½：** ① 结论是概率性的（「with high probability」），非硬保证；② r 是 ⌈log₂ N⌉ 这样的整数，带常数因子；③ 真实失效若有相关性（如整段环区域同时宕机），连续失效段会远比独立模型容易出现，½ 阈值便不再成立（见下）。

## 6. 成立前提与边界（推理中用到的全部假设）

上述 ½ 结论依赖以下条件，缺一则会偏移：

- **失效相互独立、随机**（非对抗性）。相关性失效会显著抬高「连续失效段」的出现概率，使容忍比例**低于** ½。
- **初始稳定**：后继列表在失效前是正确、连续的（定理 7/8 显式假设「initially stable」）。
- **r = Θ(log N)**，且为使阈值恰为 ½ 取 r = log₂ N。r 增大→可容忍更高比例（代价是状态）；r 减小→更脆弱。
- **「容忍」= 既正确又高效**：½ 失效下不仅查找成功（定理 7），且期望仍为 O(log N)（定理 8），并非「勉强能跑」。
- **机制不修复已发生的环分区**：§7 明确把「healing partitioned rings」列为未来工作——后继列表防的是随机失效下的搁浅，不能自愈由分区/多重环绕造成的病态拓扑。

---

## 7. 一句话总结

后继列表把 Chord 中**唯一决定正确性的后继指针**复制成 **r 个连续邻居**；连续性使掉一个后继仍落到「前方最近的存活节点」（方向正确、不跳过、不越界），冗余性使单点失效不致搁浅；**当且仅当环上出现 ≥ r 连续失效**时节点才搁浅、路由才断。取 **r = log₂ N** 时，全网搁浅期望数 = N^(1+log₂ f)，恰在 **f = ½** 处由趋于 0 翻转为趋于 ∞——故 ½ 是该设计下天然且最优的失效容忍阈值：更小则浪费冗余、更脆弱，更大则需付出更大状态且已越相变、易级联碎裂。
