# FM-index 的「机会式 / 熵有界」体积特性

## 实验结论（一句话）

FM-index 的核心体积（BWT 的压缩表示）**随原文可压缩性的提高而单调下降**：对几乎不可压缩的随机 DNA，索引/原文比趋近甚至超过 1（付出编码开销）；对高度可压缩的重复文本，比值可降至 ~0.5%；普通英文居中。这与 Ferragina–Manzini (2000) 的熵有界结论 `|BW_RLX(T)| ≤ 5·|T|·H_k(T) + g_k·log|T|` 一致——索引体积被文本的 k 阶经验熵（而非原文长度）所界。

---

## 1. 理论依据（材料）

`fmindex_material/fmindex_ferragina_manzini_2000.pdf` 提出 *opportunistic* 数据结构：对文本 `T[1,u]`，先做 Burrows–Wheeler 变换得 `L = bwt(T)`，再用 **mtf + 零游程 RLE + 前缀码 PC**（合称 `BW_RLX`）压缩 `L`。论文证明（Eq. 1，对任意固定 k ≥ 0）：

> `|BW_RLX(T)| ≤ 5·|T|·H_k(T) + g_k·log|T|`

其中 `H_k(T)` 是 k 阶经验熵，表示“用前 k 个字符作上下文编码每个字符所能达到的最佳压缩”。关键含义：

- **不可压缩文本**：`H_k ≈ log₂σ`，索引体积 `O(|T|·logσ)` —— 与原文同阶（不再有亚线性优势，但仍是“已知的最佳界”）。
- **可压缩文本**：`H_k` 远小于 `log₂σ`，索引体积随熵 *正比缩小*，可达到 **亚线性** 于原文的体积。
- 直觉：BWT 把相同上下文的后缀聚集在一起，使 `L` 中相同字符形成长游程；游程数 `r` 越少，RLE 越省。故 **BWT 游程数 `r` 是“索引体积↔可压缩性”的中介量**。

## 2. 实验设置（固定项 / 自变量）

- **文本长度**：固定 `N = 65 536`（所有文本等长）。
- **文本类型**（4 种，可压缩性递增 / 不同）：
  1. 随机 DNA（`ACGT` 均匀，几乎不可压缩）
  2. 半重复 DNA（16 个长度 8 的 motif 随机拼接，中度可压缩）
  3. 重复 DNA（单一长度 32 的 motif 重复，高度可压缩）
  4. 英文文本（Project Gutenberg《爱丽丝漫游奇境》真实文本前 65 536 字符）
- **随机种子**：`np.random.default_rng(12345)`，固定。
- **度量方式**：
  - 文本可压缩性：0/2/3 阶经验熵 `H_0, H_2, H_3`；
  - BWT 游程数 `r` 及 `r/n`；
  - BWT 自身的 0 阶熵 `H_0(L)`；
  - **索引体积** = `BW_RLX` 流水线输出位数（bwt→mtf→零游程RLE→PC，严格按论文实现）。
  - 原文体积以两种口径给出：`n·log₂σ`（字母表最优定长编码）与 `n·8`（原始字节）。
- **自变量**：文本可压缩性（由文本类型 / `H_k` / `r` 刻画）。其余固定。
- 代码：`experiment.py`；后缀数组用 numpy 前缀倍增 `O(n log n)`，CPU-only，全表 1.2 秒跑完。

## 4. 关系分析：索引/原文体积比 ↔ 可压缩性

**（a）随熵下降而下降（同字母表内最清晰，3 个 DNA 文本 σ=4）：**

```
可压缩性递增 →  H₂: 2.000 → 1.720 → 1.024
                索引/原文(n·log₂σ): 1.574 → 0.451 → 0.0052
                索引 bits/符号:      3.147 → 0.903 → 0.011
```

完全单调：可压缩性越高（`H_k` 越低），索引/原文比越小。重复 DNA 把 65 536 个符号压成 688 位（约 86 字节），相比原文 2 比特/符号的定长编码节省 **99.5%**。

**（b）随 BWT 游程数下降而下降（`r` 是直接中介量）：**

```
r/n: 0.749(随机) → 0.492(英文) → 0.206(半重复) → 0.00043(重复)
索引 bits/符号: 3.147 → 2.892 → 0.903 → 0.011
```

`r` 与索引体积强正相关——这正是 opportunistic 特性的机制：BWT 聚集同上下文后，游程越少，RLE/mtf 压得越狠，索引越小。`H₀(BWT) ≈ H₀(原文)`（BWT 是字符重排，不改字符频率，故 0 阶熵不变）；体积的节省完全来自**高阶结构（游程）**，与论文用 `H_k (k≥1)` 而非 `H₀` 作界一致。

**（c）不可压缩情形的“开销”诚实体观：** 随机 DNA 的 `索引/原文(n·log₂σ) = 1.574 > 1`——对不可压缩文本，`BW_RLX` 的 mtf/RLE/PC 编码有约 1.57× 熵的开销，索引并未小于 2 比特/符号的定长原文。这与 `5·H_k` 的宽松常数界相容（实测 ~1.57×，远低于理论上界 5×）。注意若以**字节原文**(8 bit/符号)为口径，比值仅 0.39，索引仍胜出——因为 BWT 流水线把 4 字母随机序列也压到了 ~3.15 bit/符号。这一对比恰好说明“机会式”二字的含义：**优势只在原文可压缩时显现**，对不可压缩数据则退化为有开销的线性体积。

**（d）英文文本：** σ=69，`H₀=4.39`、`H₂=2.57`、`H₃=1.78`——真实英文具显著高阶冗余。BWT 游程 `r/n=0.49`，索引/原文(n·log₂σ)=0.47、bits/符号=2.89，介于随机与半重复之间，且 `2.89 < H₀=4.39`（亚 0 阶熵），验证了索引对“普通但可压缩”文本同样有效。

## 5. 结论

1. **单调关系成立**：固定长度、固定度量下，索引/原文体积比随文本可压缩性提高（`H_k`↓、BWT 游程 `r`↓）而**单调下降**，从随机 DNA 的 ~1.57（相对定长原文，>1，付开销）到重复 DNA 的 ~0.005（节省 99.5%）。
2. **机制**：体积由 BWT 游程数 `r` 主导；BWT 不改 0 阶熵（`H₀(L)=H₀(T)`），节省全部来自高阶结构，故论文用 `H_k` 作界是必要的、且与实测吻合。
3. **熵有界性验证**：实测 `BW_RLX` 位数 / `|T|·H_k` 比值约为 0.01–1.6，始终 `≤ 5`，落在论文 Eq. 1 的 `5·|T|·H_k + g_k·log|T|` 界内；不可压缩时退化到线性、可压缩时趋亚线性——正是 opportunistic / 熵有界特性的体现。
