# 为什么 FM-index 的 LF-mapping 使「反向搜索」成立

> 任务要求：仅凭逻辑推理（不做实验）解释 LF-mapping 性质，并据此说明反向搜索如何用 `C` 与 `rank` 在每步收窄行区间、从不查阅原文即可计数模式出现次数。
> 依据为给定材料 Ferragina & Manzini, *Opportunistic Data Structures with Applications* (FOCS 2000)。

---

## 0. 记号（与论文一致）

- 文本 `T` 末尾追加哨兵 `$`（字典序最小且唯一），得到 `T$`，长度 `n`。
- **BWT 矩阵** `M`：`T$` 的 `n` 个循环旋转，按字典序排序后得到的 `n×n` 矩阵。
  - `F` = 第一列 = `T$` 所有字符排序后的序列（即排序后各旋转的首字符）。
  - `L` = 最后一列 = **BWT(T$)**。
  - 每一行是一条循环旋转；把某行的末字符 `L[i]` 旋转到最前，就得到「下一行」（在循环意义上）——这是 LF 变换的几何来源。
- `C[c]` = 字典序小于 `c` 的字符总数（即在 `F` 中 `c` 段的起始下标）。
- `rank(c, i)` = `L[0..i-1]` 中字符 `c` 的出现次数（`L` 前 `i` 个位置里 `c` 的个数）。
- 目标：仅用 `L`、`C`、`rank`，**不存原文**，计数模式 `P` 在 `T$` 中的出现次数。

---

## 1. LF 性质：`L` 中 `c` 的第 `i` 次出现 ↔ `F` 中 `c` 的第 `i` 次出现

### 1.1 旋转的「稳定性」

把任意一行（一条循环旋转）的末字符移到最前，得到另一条循环旋转；这一操作称为对该行做 **LF 旋转**。关键观察：

> **在「以相同字符结尾」的行之间，LF 旋转保持它们的相对顺序。**

证明。设两行 `R_a < R_b`（字典序）且都**以同一个字符 `c` 结尾**，即 `L` 在这两行位置都是 `c`。写
`R_a = α_a · c`、`R_b = α_b · c`（`·` 为拼接，`α_a, α_b` 是去掉末字符 `c` 后的前缀）。

- 因为 `R_a < R_b` 且二者末字符相同（都是 `c`），其字典序完全由前缀决定：`α_a < α_b`。
  （末字符相同 ⇒ 比较退化为前缀比较；这是「稳定」的核心。）
- LF 旋转把 `R_a → c · α_a`、`R_b → c · α_b`。两者现在都**以 `c` 开头**，其相对顺序由 `c` 之后的部分决定，即由 `α_a` vs `α_b` 决定：`c·α_a < c·α_b ⇔ α_a < α_b`。

故 `α_a < α_b ⇒ c·α_a < c·α_b`：**LF 旋转把「以 `c` 结尾的行」按原顺序映到「以 `c` 开头的行」**，顺序不乱。这就是旋转稳定性。

### 1.2 推论：第 `i` 个 `c` 对应第 `i` 个 `c`

`M` 的行是排序后的旋转。

- 「`L` 中 `c` 的第 `i` 次出现」= 排序后**以 `c` 结尾**的第 `i` 行。
- 「`F` 中 `c` 的第 `i` 次出现」= 排序后**以 `c` 开头**的第 `i` 行。

由 §1.1，LF 旋转把「以 `c` 结尾的行」**按相同顺序**映到「以 `c` 开头的行」。而 `M` 的每一行经 LF 旋转后仍是 `M` 的某一行（循环旋转的集合在旋转下封闭），所以「以 `c` 结尾的第 `i` 行」恰映到「以 `c` 开头的第 `i` 行」。

> **LF 性质**：`L` 中 `c` 的第 `i` 次出现 ↔ `F` 中 `c` 的第 `i` 次出现。

### 1.3 LF 映射的闭式

对 `L` 中位置 `i`（字符 `c = L[i]`），设它是 `L` 中 `c` 的第 `r` 次出现（`r = rank(c, i)`，按「前 `i` 个位置含 `c` 的个数」计，`i` 本身算第 `r` 个）。由 LF 性质，它对应 `F` 中 `c` 的第 `r` 次出现。`F` 是排序后的字符，`c` 段从下标 `C[c]` 开始，故 `c` 的第 `r` 次出现在 `F` 的下标 `C[c] + r − 1`（0 基），即行号

> **LF(i) = C[L[i]] + rank(L[i], i)**   （按「`rank(c,i)` = `L[0..i-1]` 中 `c` 的个数」的约定，恰为 `C[c] + rank(c,i)`，给出 `c` 段内第 `rank(c,i)` 个的行号；不同实现仅差一个 0/1 基约定，本质相同。）

LF 把「末字符在 `L[i]` 的行」映到「该行旋转后的行」（即原文中 `L[i]` 的前一字符所在行）。这是「不存原文也能向前回溯」的根基。

## 2. 反向搜索：用 `C` 与 `rank` 逐字符收窄行区间

### 2.1 区间的含义

反向搜索维护一个**行区间 `[top, bot)`**（左闭右开）。其不变式：

> `[top, bot)` 中的行 = `M` 中所有「循环旋转的**后缀**等于 `P` 的已处理后缀」的行。

初始（已处理后缀为空）：`[0, n)`，全部行。

### 2.2 前置字符 `c` 时如何收窄

设当前区间 `[top, bot)` 已匹配后缀 `P[j+1..]`。要前置字符 `c = P[j]`，即找「旋转 = `c · P[j+1..] · …`」的行，也就是 **`F` 以 `c` 开头、且 `c` 之后的部分落在当前区间对应位置**的行。

由 LF 性质，`F` 中 `c` 开头的行与 `L` 中 `c` 结尾的行**一一保序对应**。因此「`c` 之后部分落在当前区间」等价于「该 `c` 在 `L` 中出现的位置落在当前区间 `[top, bot)` 内」。`L` 中 `c` 在 `[top, bot)` 内的出现，恰好是 `c` 的第 `rank(c, top)` 到第 `rank(c, bot) − 1` 次（0 基，左闭右开）；这些映射回 `F` 的 `c` 段，就是第 `rank(c, top)` 到第 `rank(c, bot) − 1` 个 `c`，对应行号

> **新区间 = `[ C[c] + rank(c, top),  C[c] + rank(c, bot) )`**

每步只需**两个 `rank` 查询**（每边界一个）+ 一次 `C[c]` 查表。`|P|` 步后，区间大小 `bot − top` = 旋转后缀恰为 `P` 的行数 = `P` 在 `T$` 中的出现次数。

### 2.3 为什么不需原文

每步只读 `L`（取 `rank`）和 `C`（取段起点）。`rank` 是对 `L` 的查询、`C` 是对字母表的预计算——**两者都不依赖原文 `T`**。LF 性质保证了「`L` 中 `c` 的第 `i` 个」与「`F` 中 `c` 的第 `i` 个」的保序对应，所以仅凭 `L`+`C`+`rank` 就能逐字符向**前**重建匹配，最终得到出现次数。这正是 FM-index「opportunistic」（体积被原文熵上界约束）且「succinct」（不存原文）的根本机制。

---

## 3. 一句话总结

BWT 矩阵的行是 `T$` 的循环旋转排序；**LF 旋转在「以相同字符结尾的行」间保序**（因末字符相同时字典序退化为前缀比较，旋转后前缀变后缀、首字符同为 `c`，顺序不变），故 `L` 中 `c` 的第 `i` 次出现恰对应 `F` 中 `c` 的第 `i` 次出现，给出 `LF(i)=C[L[i]]+rank(L[i],i)`。反向搜索维护行区间 `[top,bot)`（其旋转后缀已匹配 `P` 的已处理后缀），前置字符 `c` 时由 LF 对应把区间收窄为 `[C[c]+rank(c,top), C[c]+rank(c,bot))`，每步两个 `rank`，`|P|` 步后区间大小即为 `P` 的出现次数——全程仅用 `L`、`C`、`rank`，从不查阅原文。
