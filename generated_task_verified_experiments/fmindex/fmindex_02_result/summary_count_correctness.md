# FM-index 反向搜索 count(pattern) 正确性验证

## 任务

考察 FM-index（Ferragina–Manzini, 2000）反向搜索 `count(pattern)` 的正确性：从零实现 FM-index（BWT、C 数组、对 L 的 rank/Occ 支持、backward-search 计数），与暴力子串计数（brute-force，重叠计数）逐一比对，统计一致性（匹配率 / 差异）。

材料为 `fmindex_material/fmindex_ferragina_manzini_2000.pdf`。实现依据论文 Figure 1 的 *BW Search* 算法：

> `sp = C[c] + 1, ep = C[c+1]`；循环 `sp = C[c] + Occ(c,1,sp-1) + 1`，`ep = C[c] + Occ(c,1,ep)`；`ep - sp + 1` 即出现次数。

本实现采用等价的 0-基、半开区间 `[l, r)` 写法：`l = C[c] + Occ(c,l)`，`r = C[c] + Occ(c,r)`，`count = r - l`。

## 实现（`fmindex.py`，CPU-only，numpy 向量化）

1. **后缀数组（SA）**：前缀倍增（prefix doubling），O(n log n)，全程 numpy 向量化。关键点：每轮按 (rank[i], rank[i+k]) 合并键排序后，必须对**相等键赋相同 rank**（rank compression）；若误给不同 rank 会破坏倍增不变量（早期版本曾因此产生系统性漏数，修复后 SA 与朴素后缀排序在小例上完全一致）。
2. **BWT（L 列）**：`L[i] = T[(SA[i]-1) mod n]`，文本末尾追加唯一最小哨兵 `\x00`（小于任何可打印字节），保证后缀互异且循环唯一。
3. **C 数组**：`C[code] = 文本中字节值严格小于该字符的字符总数`（即首列 F 中该字符的起始位置）。
4. **Occ/rank**：对 L 的每个出现字符预计算前缀计数矩阵 `occ[code, i] = #该字符在 L[0:i]`（i=0..n），即论文中 `Occ(c,1,k)` 的 O(1) 查询结构（计数表实现，未用小波树，正确性等价）。
5. **反向搜索计数**：半开区间 backward search，见上。
6. **暴力基线**：`str.find` 循环，**重叠**计数（对 DNA 等重复场景必需）。

固定设置：文本、模式集合、随机种子、实现均固定（见 `fmindex.py`：DNA seed=12345/777；English seed=99999/888；sanity seed=111）。本题为正确性验证，无超参自变量。

## 实验结果

### 主实验（三段文本 × 335 模式/段 = 1005 次比对）

| 文本 | 长度 | 模式数 | 一致 | 不一致 | 文本中 absent 模式 | 暴力总出现 | FM 总出现 | 匹配率 |
|---|---|---|---|---|---|---|---|---|
| 随机 DNA | 200,000 | 335 | 335 | 0 | 50 | 3,079,211 | 3,079,211 | **1.000000** |
| 伪英文 | 200,000 | 335 | 335 | 0 | 104 | 691,791 | 691,791 | **1.000000** |
| sanity (banana+mississippi) | 170 | 335 | 335 | 0 | 105 | 3,106 | 3,106 | **1.000000** |

模式覆盖：长度 1–40 的文本子串（保证 ≥1 次出现）、随机字母模式（多数 absent）、边界情形（首/尾字符、前/后缀、整段文本、均聚物 `AAAA`/`A*10`、常见词 `the`、不可能三元组 `qzx`、含文本外字节的必 absent 模式）。

### 附加压力测试（612 次比对，全部一致）

- **BWT 可逆性**（LF-mapping 反向重建文本）：`banana`、`mississippi*3` 重建均与原文逐字节相等 —— 佐证 BWT / C / Occ 三者内部自洽。
- 高重复 / 大规模文本，零不一致：

| 文本 | 模式数 | 不一致 |
|---|---|---|
| `A*5000`（极端重复） | 204 | 0 |
| `AB*5000`（周期） | 204 | 0 |
| 随机 DNA 400KB | 204 | 0 |

### 汇总

- 模式比对总数：**1617**（主 1005 + 压力 612）
- 不一致总数：**0**
- 总匹配率：**100.000%**
- 暴力总出现数 == FM 总出现数（逐文本相等），无系统性偏移。

## 结论

从零实现的 FM-index（后缀数组构造 BWT + C 数组 + 计数表 Occ + 半开区间 backward search）的 `count(pattern)` 与暴力重叠子串计数在全部测试场景下**完全一致**：

- 跨两种文本类型（小字母 DNA、大字母英文）一致；
- 跨模式长度（1–40）与出现/不出现/边界/均聚物/整段文本等情形一致；
- 跨极端重复与 400KB 规模一致；
- 且 BWT 经 LF-mapping 可无损还原原文，进一步佐证索引结构正确。

**反向搜索计数与暴力计数一致性 = 100%（匹配率 1.000000，差异 0）。** 在固定文本/模式/种子/实现下，FM-index backward-search 的计数正确性得到验证。

> 实现注意：正确性对两个细节敏感——(1) 前缀倍增构造 SA 时必须对相等 2k-gram 赋相同 rank，否则会系统性漏数；(2) 暴力基线必须重叠计数，否则对重复文本会人为产生“差异”。两者均在本实现中正确处理。
