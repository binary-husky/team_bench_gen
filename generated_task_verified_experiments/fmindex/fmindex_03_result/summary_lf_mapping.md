# LF-mapping 可逆性验证结论

## 任务
在固定文本（约 100–500 KB，固定随机种子）上构建 BWT，**仅用** BWT 的最后一列 `L`、`C` 数组与 `rank/Occ`，通过反复 LF 映射从 BWT 逐字符反向重建文本，再与原文逐字节比较，把「重建文本与原文的一致性（是否完全相等、首个不匹配位置）」记录下来。

## 实验设置（固定）
- **实现**：见 `lf_mapping_experiment.py`（纯 Python，CPU-only）。
- **文本**：长度 `N = 262144` 字节（约 256 KB，落在 100–500 KB 区间），由 `random.seed(20240626)` 生成的伪随机字节。
- **字母表**：字节 `1..255`；字节 `0` 保留为唯一哨兵 `SENTINEL`（充当 FM-index 中的 `$`）。哨兵严格小于任何文本符号且全表唯一，故构成合法的 BWT 终结符，且不会与正文冲突。
- **BWT 构建**：对 `T' = text + SENTINEL` 用 `pydivsufsort` 计算后缀数组 `SA`，再取 `L[i] = T'[SA[i]-1]`（`SA[i]==0` 时 `L[i] = T'[-1] = SENTINEL`）。SA 仅用于**构造** BWT；重建阶段完全不接触 SA，只用 `L / C / rank`。
- **索引结构**（均仅由 `L` 导出，符合 FM-index 范式）：
  - `C[c] = # { L 中符号 < c }`；
  - `positions[c] = L 中 c 出现的位置升序列表`，于是 `Occ(c, i) = bisect_left(positions[c], i)`，单次查询 O(log n)。
- **LF 映射**：`LF(i) = C[ L[i] ] + Occ( L[i], i )`（Ferragina & Manzini 2000，见材料 PDF）。
- **反向重建**：从第 0 行出发（排序后轮转矩阵第 0 行以哨兵开头，其 `L[0]` 即正文最后一个字符），反复 `i ← LF(i)` 并收集 `L[i]`，得到正文逆序；反转即得原文。

## 自检（算法正确性，独立于大文本）
对 `banana / mississippi / abracadabra / aaaa / abcabcabc` 等经典串做同样流程，BWT 与人工结果一致（如 `mississippi$` → `ipssm$pissii`，`banana$` → `annb$aa`，`abracadabra$` → `ardrcaaaabb`），重建均与原文逐字节相等。算法逻辑本身得到验证。

## 主实验结果（256 KB 随机文本）
| 量 | 值 |
|---|---|
| 文本长度 `N` | 262144 字节 |
| 含哨兵长度 `n` | 262145 |
| 随机种子 | 20240626 |
| **重建文本与原文是否完全相等** | **是（完全相等）** |
| **首个不匹配位置** | **无（无任何不匹配）** |
| 哨兵在 BWT 中的行号 | 113905 |
| `C[SENTINEL]` | 0（最小符号，符合预期） |
| BWT 构造耗时 | ≈ 0.51 s |
| 反向重建耗时 | ≈ 0.13 s |

## 结论
在固定的 256 KB 文本、固定种子、固定实现下：

- 仅凭 `L`、`C` 数组与 `rank/Occ`，通过反复 LF 映射反向重建出的文本**与原文逐字节完全相等**；
- **不存在首个不匹配位置**（比对在全部 262144 字节上一致）；
- 该结果与小样本经典串自检一致，且重建耗时远低于构造耗时，证明流程正确且高效。

这经验性地验证了 LF-mapping 的**可逆性**：给定 BWT 及其 `C`/`rank` 索引，文本可被无损还原。可逆性的根源在于 LF 映射是一个双射——`LF(i) = C[L[i]] + Occ(L[i], i)` 将“以 `L[i]` 结尾的行”一一对应到“以 `L[i]` 开头的行”，从而沿着哨兵行反复应用 LF 即按逆序遍历原文的每一个字符，不丢失、不重排信息，故重建必然与原文一致。本实验未引入任何超参自变量，是一次纯粹的可逆性确认。
