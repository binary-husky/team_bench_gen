**问题**（唯一一题）：

SHADE Algorithm 1 中对历史记忆 `M_CR, M_F`（各有 H 个槽位）使用了两套**结构相同但选择机制不同**的索引：

- **行 9（参数采样侧）**：`r_i = Select from [1, H] randomly`——每个个体 i 在每一代都从 [1, H] 中**均匀随机**抽一个槽位 `r_i`，然后用 `CR_i ~ N(M_CR[r_i], 0.1)`、`F_i ~ C(M_F[r_i], 0.1)` 采样该个体的控制参数。
- **行 29–30（记忆更新侧）**：`k++; if k > H, k = 1`——每代（在 `S_CR, S_F` 非空时）**循环递增**一次槽位 `k`，然后把 `mean_WA(S_CR)` / `mean_WL(S_F)` 写到 `M_CR[k], M_F[k]`。

两个索引都从 [1, H] 中取值，看起来很相似。论文 §V.D 只在文字层面说"H 越小 → 旧值被快速覆写 → 控制参数快速收敛；H 越大 → 旧值影响持续更久 → 收敛变慢"，但**没有**显式解释为什么更新侧用循环 k、采样侧用随机 r_i，也没有讨论如果交换两者的选择机制会发生什么。

请纯粹从算法结构出发（不跑实验），逻辑推理回答下面 3 个子问题：

**子问题 1（更新侧换成随机）**：如果把行 29 改成"`k = uniform_random(1, H)` 每代独立抽一次"（其它不变），请推导：在 `S_CR ≠ ∅` 每代都成立的前提下，G = H 代后**从未被写过**的槽位期望数量是多少（给出闭式表达式与 H=100 时的数值）。然后解释：这一改动如何破坏论文 §V.D 所述的"recency bias / rolling window"性质？

**子问题 2（采样侧换成确定性）**：如果把行 9 改成"`r_i = ((i - 1) mod H) + 1`"（按个体下标确定性映射，其它不变），在 N = H = 100 时：

**子问题 3（综合结论）**：用 1–2 段话总结——为什么 SHADE **必须**同时在更新侧用循环 k、在采样侧用随机 r_i？单独保留任一侧的"原版机制"而把另一侧换成"另一种"会引入什么不同方向的失效模式？

[Judge (IQ requirement: medium-IQ)]

阅读 `./summary_cec_09_dual_index.md`，检查答案是否覆盖以下 1 个核心评价维度（**唯一一题，必须全对**）：

1. **子问题 1 的标准答案（必须出现）**：
    - 闭式：从未被写过的槽位期望数 = `H · (1 - 1/H)^G`；当 G = H 时 ≈ `H/e`（H=100 时约 36.8 个槽位）。
    - 论文 §V.D 的 "recency bias / rolling window" 依赖循环 k 保证"每个槽位每 H 代至少被覆写一次"。换成随机后，约 37% 的槽位**永久停留在初始值 0.5**（或更早期写入的随机值），永远不被刷新——finite memory 退化成"被随机命中的槽位频繁刷新 + 一批死槽位"的混合体，与 JADE 单 μ 自适应相比失去结构性优势。

2. **子问题 2 的标准答案（必须出现）**：
    - (a) 改动版（`r_i = i`）每代**恰好** min(N, H) = 100 个不同槽位被采样；原版（随机）期望 ≈ `H · (1 - (1-1/H)^N)` = `100 · (1 - 1/e)` ≈ 63.4 个不同槽位（coupon-collector 类估计）。
    - (b) 改动版下，个体 i 永远只能从槽位 i 取参数；若 M_CR[j]、M_F[j] 对该问题不合适，则个体 j 在**所有代**都拿到差参数。种群实质上被分裂为"运气好的血脉（绑定好槽位）"和"运气差的血脉（绑定差槽位）"，选择压力作用在**参数运气**而非**解质量**上，导致过早收敛 / 极化。原版（随机 r_i）保证每个个体在统计上都从同一分布采样，**个体间统计对称**——任何个体都有同等机会碰到任何槽位，没有"血脉隔离"。

3. **子问题 3 的综合结论（必须出现）**：
    - 循环 k 保证**记忆侧的公平周转**（每个槽位周期性被刷新，没有死槽位）；随机 r_i 保证**采样侧的统计对称**（个体之间无永久绑定）。
    - 只换更新侧（随机 k）→ 死槽位 + recency bias 丧失 → 退化成"带噪声的 JADE 单 μ"。
    - 只换采样侧（确定性 r_i）→ 血脉隔离 + 选择压力错位 → 退化成"多个独立子种群各自跑自适应"，失去 SHADE 单一协调搜索的意义。
    - 两个机制方向不同、缺一不可，是 SHADE 相对 JADE / SaDE 的核心结构创新。
