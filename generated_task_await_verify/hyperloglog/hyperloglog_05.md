自行复现 HyperLogLog 算法（从给定材料中的论文出发用 Python/numpy 实现；较好的 64-bit 哈希）。本次研究 HyperLogLog 的**可合并性（mergeability）**：两张同精度的 sketch 可通过**逐寄存器取 max** 合并，等价于在两流并集上重建的 sketch。

研究目标：**验证 `M[j] = max(A[j], B[j])` 的合并确实给出并集基数（不是两流基数之和），且与"直接在并集流上建 sketch"在寄存器层完全等价。**

固定实验设置（不要更改）：
- 精度 **p = 14**（`m = 16384`）。
- 两条**不相交**合成流 `A`、`B`，各有 **n_A = 5e4** 与 **n_B = 5e4** 个不同 item（`A ∩ B = ∅`），故真实并集基数 = `1e5`。
- 合并：`S_merge[j] = max(S_A[j], S_B[j])`，再估计 `Ê_merge`。
- 对照 1（直接流）：把 `A ∪ B` 整体喂给一张新 sketch 得 `S_direct`、`Ê_direct`。
- 对照 2（幂等/去重语义）：把 `A` 与 `A` 的副本合并得 `S_dup`，估计 `Ê_dup`（真实并集仍为 `n_A = 5e4`，**不是** `1e5`）。
- **≥ 5 个不同随机种子**独立重复（每种子重新生成不相交流）。
- **仅 CPU**，单机；整轮 **< 30 分钟**。

需要记录/报告的指标：
- 合并精度：`|Ê_merge - (n_A+n_B)| / (n_A+n_B)`（跨种子均值±标准差）。
- 寄存器层等价性：`S_merge` 是否与 `S_direct` **逐寄存器相等**（是/否，或两者估计是否完全相同）。
- 幂等语义：`Ê_dup` 是否 ≈ `n_A`（而非 `2·n_A`）。

把以下内容写到 `./summary_hll_05_merge.md`：上述三类指标的数值结果（表/图）；并一句结论——**逐寄存器 max 合并给出的是并集基数、与直接并集流在寄存器层完全等价、且对重复元素幂等（去重）**。
