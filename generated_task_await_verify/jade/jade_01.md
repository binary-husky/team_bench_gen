**问题**（唯一一题）：

在 JADE 的算法描述中（论文 §IV.B，公式 8 与公式 10）：

- 公式 (8)：`CR_i = randn_i(μ_CR, 0.1)`，**截断到 [0, 1]**（即超出范围的值被夹到最近的边界）。
- 公式 (10)：`F_i = randc_i(μ_F, 0.1)`，**若 F_i ≥ 1 则截断为 1，若 F_i ≤ 0 则重新生成**（regenerate，不是夹到 0）。

也就是说，论文对 CR 用了简单的"截断到边界"规则，对 F 却用了不对称的"负数重新生成"规则。论文本身**没有**直接解释为什么对 F 不能用同样的截断规则（即把负的 F_i 直接夹为 0）。

请纯粹从算法语义出发，逻辑推理回答：

**如果允许 `F_i = max(0, F_i_raw)`（即把负的 F_i 简单夹为 0），相比论文采用的"负数重新生成"规则，会在 JADE 的运行中产生什么本质不同的失败模式？**

具体地，你的回答需要覆盖以下推理链（论文本身没有给出，需要你自己推）：

1. 在公式 (7) 中代入 `F_i = 0`，得到的变异向量 `v_i,g` 与父代 `x_i,g` 的关系是什么？由此推出的 trial vector `u_i,g` 与 `x_i,g` 的关系又是什么（考虑公式 4 的 binomial crossover 中 `j_rand` 的作用）？
2. 在 JADE 表 I 伪代码第 20–23 行的选择 / 归档步骤中，如果大量个体的 `F_i = 0` 同时发生，会触发什么连锁反应？特别地，归档集 A 是否还会增长？SF（成功 F 集合）是否还会被填充？μ_F 和 μ_CR 是否还会被更新？
3. 把上述推理与 CR 的情况对比：在公式 (4) 中 `CR_i = 0` 时 trial vector 与父代的关系是什么？为什么对 CR 用截断到 [0,1] 是无副作用的，而对 F 用截断到 [0,1] 就不是？

最后给出一句清晰的结论：**JADE 为什么必须对 F ≤ 0 做 regenerate 而不能 truncate**。

[Judge]

阅读 `./summary_f_cr_handling.md`，检查答案是否覆盖以下 1 个核心评价维度（**唯一一题，必须全对**）：

1. **核心论断（必须出现）**：当 `F_i = 0` 时，公式 (7) 退化为 `v_i,g = x_i,g`；再经过公式 (4) 的 binomial crossover 后，trial vector `u_i,g` 在 `j = j_rand` 这一个分量上等于 `v_i,g`（= `x_i,g`），其余分量继承父代 `x_i,g`，因此 **`u_i,g ≡ x_i,g`**——即 trial 与 parent 完全相同。这导致：
    - 选择步骤 (5) 中 `f(u_i,g) = f(x_i,g)`，按照 `f(u_i,g) < f(x_i,g)` 的严格小于判定，**不会发生成功更新**；
    - 因此 `x_i,g` 不会进入 archive A，`F_i` 也不会进入 SF；
    - 当大量个体同时撞上 `F_i = 0`（例如 Cauchy 分布有重尾，概率不低）时，A 不增长、SF 为空或几乎为空、μ_F 在公式 (11) 中 `mean_L(SF)` 无意义或保持不变——**整个种群的探索停滞，参数自适应失效**。
2. **CR 的对比（必须出现）**：当 `CR_i = 0` 时，公式 (4) 中除 `j = j_rand` 外全部继承父代，因此 trial vector 与父代仅在 `j_rand` 一个分量上不同——**trial 与 parent 仍然有差别**，仍可能发生成功更新、仍会进入 SF/SCR、archive 仍会增长。所以对 CR 截断到 [0,1] 不会触发 F 那种"整体冻结"的级联失败。
3. **结论（必须出现）**：JADE 对 `F_i ≤ 0` 必须 regenerate，因为 F 是变异向量的**乘性标量**（其符号决定差分向量的方向，其零值导致整个差分项消失），而 CR 是分量级的**概率**（其符号在 [0,1] 之外无物理意义，截断只是把概率压到合法区间）。两者在算法语义上根本不同，不能用同一套截断规则。
