**问题**（唯一一题）：

**为什么符号执行能在小函数上系统性地达到（近乎）完全的分支覆盖，而随机测试无法可靠到达深层 / 稀有分支？** 请从路径与约束的角度逻辑推理，覆盖以下推理链（给定材料描述了 KLEE 的覆盖能力，但需要你自己把"为何符号执行系统性覆盖、而随机测试受概率限制"这条对比论证讲清楚）：

1. **一条执行路径 = 一组合取的分支约束**：解释为什么走完一条特定路径等价于同时满足一串条件 `c1 ∧ c2 ∧ … ∧ ck`（每个 `c_i` 是路径上某个 `if` 取 true 或 false 的编码）。符号执行在每个分支处如何决定探索哪边？（用求解器判断 `PC ∧ c_i` 与 `PC ∧ ¬c_i` 各自是否可满足；两边都可能时**复制状态 fork** 分别探索。）

3. **符号执行为何能系统到达**：在分支处 fork 后，只要 `PC ∧ (c1∧…∧ck)` 可满足，求解器就能**解出一个**具体输入到达该深层路径——不依赖"运气/概率"，而是逐条可行路径地系统枚举。故在小函数（可行路径数有限）上能近完全覆盖。

最后给出一句结论：**为什么符号执行在小函数上近完全分支覆盖、而随机测试受限于指数衰减的命中概率（在稀有/深层分支上尤其乏力）**。
